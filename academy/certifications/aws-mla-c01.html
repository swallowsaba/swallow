AWS MLA-C01Machine Learning Engineer – Associate

2024年に登場した「実装側」の AWS ML 資格。データ準備、モデル開発、デプロイ、監視・保守・セキュリティの 4 ドメインで、本番の機械学習を成立させるアーキテクト力を測ります。MLOps の標準語彙としても価値が大きい。

AWS の機械学習資格は、ML Specialty（MLS-C01）と、本資格 MLA-C01 の 2 系統があります。MLA-C01 は 「実装と運用」 側に焦点を当てた Associate 級。SageMaker、Glue、Feature Store、Pipelines、Model Monitor、Bedrock といった構成要素を、本番の文脈でどう組むかを問います。

§02 / D1ドメイン1: データ準備（28%）

取り込み・保存

  • S3 をデータレイクの中心に、Parquet/ORC など列指向で保存
  • Kinesis Data Streams（低レイテンシ） vs Firehose（バッファ配信、ニアリアルタイム）
  • AWS Glue: サーバレス ETL ＋ データカタログ（Crawler、Studio、DataBrew）
  • EMR: Spark / Hadoop マネージド、大規模分散処理
  • Lake Formation: アクセス制御を集中管理、行・列レベルのきめ細かい権限

変換・特徴量エンジニアリング

  • SageMaker Data Wrangler: GUI で 300+ の組み込み変換
  • SageMaker Processing Job: コンテナベースの前処理／後処理
  • SageMaker Feature Store: オンライン／オフラインの統一管理で train/serving skew を防止
  • Glue DataBrew: ノーコード前処理。Wrangler との使い分けに注意

クエリ・カタログ

  • Athena: S3 への SQL（サーバレス、スキャン量課金）
  • Redshift / Redshift Spectrum: DWH、S3 直クエリ
  • Glue Data Catalog: Athena・EMR・Redshift Spectrum 共通のメタデータ層

データ品質・バイアス

  • SageMaker Clarify: 学習前バイアス、推論バイアス、説明可能性（SHAP）
  • Glue Data Quality: DQDL で品質ルール
  • Macie: S3 内 PII の自動検出

§03 / D2ドメイン2: モデル開発（26%）

学習サービス

  • SageMaker Training: Estimator API（PyTorch/TF/HF/独自BYOC）でジョブ管理
  • SageMaker Training Compiler: XLAで計算グラフ最適化、学習高速化
  • SageMaker Distributed Training: Data Parallel / Model Parallel（テンソル、パイプライン）
  • SageMaker HyperPod: 大規模 FM 学習向け耐障害クラスタ、自動復旧
  • Managed Spot Training: 最大70%コスト削減＋チェックポイント必須

自動化と事前学習活用

  • SageMaker Autopilot: 表形式 AutoML、Notebook も生成
  • SageMaker Canvas: ノーコード ML（ビジネスユーザ向け）
  • SageMaker JumpStart: 事前学習済みモデルのワンクリック展開・微調整
  • Hyperparameter Tuner: Random / Bayesian / Hyperband / Grid、Warm Start

アクセラレータ

  • EC2 G5/P5、ml.g/p インスタンス: NVIDIA GPU
  • AWS Trainium（Trn1/Trn2）: 学習に最適化されたカスタムシリコン
  • AWS Inferentia（Inf1/Inf2）: 推論最適化チップ。Neuron SDK 経由

§04 / D3ドメイン3: デプロイとオーケストレーション（22%）

推論オプションの選択

Real-time Endpoint常時稼働、低レイテンシ。Auto Scaling、Multi-Model、Multi-Container 対応
Serverless Inference断続的トラフィック、従量課金。冷起動あり、Provisioned Concurrency で緩和
Async Inference大きなペイロード（最大1GB）、長時間処理、S3 経由のキュー。Auto Scaling to zero
Batch TransformS3 → S3 のオフライン推論、エンドポイント不要

オーケストレーション

  • SageMaker Pipelines: ステップを DAG で連結（Processing/Training/CreateModel/Endpoint）
  • Model Registry: モデルパッケージのバージョン管理＋承認フロー
  • Step Functions / EventBridge / Lambda: イベント駆動の自動化
  • Inference Recommender: 構成のベンチマークと推奨
  • Deployment Guardrails: Canary / Linear / BlueGreen、Auto-Rollback

大規模・効率化

  • Multi-Model Endpoint（MME）: 数千のモデルを1エンドポイントで共有
  • Multi-Container Endpoint（MCE）: 異種コンテナを並列ホスト
  • Inference Components: 大規模 FM のためのきめ細かいリソース割当

§05 / D4ドメイン4: 監視・保守・セキュリティ（24%）

監視

  • SageMaker Model Monitor: データ品質／モデル品質／バイアスドリフト／特徴量寄与ドリフト
  • CloudWatch: Metrics / Logs / Alarms。InvocationsPerInstance, ModelLatency など
  • CloudTrail: API監査ログ、Security Hub で集約

セキュリティ

  • IAM: 最小権限、Permission Boundary、SCP
  • KMS: 暗号鍵管理（S3、EBS、SageMaker、Bedrock との統合）
  • VPC Endpoint（Gateway/Interface）／PrivateLink: AWS API への閉域接続
  • SageMaker Network Isolation: コンテナをインターネットから切り離す
  • Block Public Access、Bucket Policy: S3 公開リスクの遮断
  • Macie / GuardDuty / Inspector / Detective: 脅威検知・脆弱性・PII検出

運用

  • Model Card: 用途・学習データ・性能・限界・倫理を構造化
  • SageMaker Lineage Tracking: データ・特徴量・モデル・成果物の系譜
  • Auto Scaling: Target Tracking / Step / Scheduled

§06 / BEDROCKBedrock / 基盤モデル

  • Bedrock: マネージドな基盤モデル API（Claude、Llama、Titan、Mistral、Stable Diffusion 等）
  • Bedrock Knowledge Bases: ベクトル DB（OpenSearch Serverless、Aurora pgvector）と連携した RAG
  • Bedrock Agents: ツール／API 呼び出しを伴うエージェント
  • Bedrock Guardrails: 入出力フィルタリング、機密検出、トピック制限
  • Custom Model Import / Continued Pre-training / Fine-tuning
  • Provisioned Throughput: 大規模・低レイテンシ要件向けの専有スループット

§07 / MLOPSMLOps アーキテクチャ

  1. データ取り込み（S3 / Kinesis / Glue）→ カタログ（Glue / Lake Formation）
  2. 前処理（Glue / Processing / DataBrew / Wrangler）→ Feature Store
  3. 学習（SageMaker Training、Distributed / Spot / HyperPod）
  4. 評価（Clarify、Model Monitor ベースライン）→ Model Registry に登録・承認
  5. デプロイ（Real-time / Serverless / Async / Batch、Pipelines + Guardrails）
  6. 監視（Model Monitor + CloudWatch Alarms → EventBridge → Pipelines 再学習）
  7. ログ（CloudTrail / CloudWatch Logs）と Lineage で監査・再現性を担保

§09 / ROADMAP学習ロードマップ

Phase 1 — AWS 基礎を底上げ（40〜60時間）

未経験ならまず AWS Solutions Architect Associate（SAA）相当の知識を確認。IAM / VPC / S3 / EC2 / Lambda / CloudWatch の基本。

Phase 2 — SageMaker ハンズオン（60〜100時間）

SageMaker Studio で Training／Endpoint／Pipelines を一通り触り、JumpStart、Autopilot、Bedrock も体験。AWS Skill Builder の公式コースが効率的。

Phase 3 — 模試・本番対策（40〜60時間）

公式サンプル問題＋公式練習試験。Whizlabs/Tutorials Dojo などサードパーティ模試 2〜3 セットを 90% 安定するまで反復。
