Jetson AI Specialist は、試験問題に答えるのではなく、自作のオープンソースプロジェクトを NVIDIA に提出し、レビューを受けて取得する 認定です。Nano から Orin Nano／AGX Orin まで、限られた電力と計算資源の中で「動く AI」を作る能力が評価されます。

§02 / HWJetson ハードウェア

Jetson Nano（初代）Maxwell GPU、約 0.5 TFLOPS FP16、4GB RAM、入門向け（2024 年で EOL ロードマップ進行）
Jetson Orin NanoAmpere GPU、最大 40 TOPS（INT8）、4/8GB、現行入門
Jetson Orin NX最大 100 TOPS、8/16GB、産業組込み向け
Jetson AGX Orin最大 275 TOPS、32/64GB、ロボティクス・自動運転試作
Jetson Xavier NX / AGX XavierVolta GPU、Orin 以前の世代、現役で稼働中
Jetson Thor（次世代）Blackwell ベース、ヒューマノイドロボティクス向け

共通の特徴

  • CPU/GPU が物理的にメモリを共有する UMA（Unified Memory Architecture）
  • nvpmodel で電力モード切替（例: MAXN、15W、10W）
  • tegrastats で CPU/GPU/メモリ/温度/電力を確認
  • 専用アクセラレータ: DLA（Deep Learning Accelerator）、VIC、PVA、NVENC/NVDEC

§03 / SWJetPack / ソフトウェア

  • JetPack: Jetson のSDKメタパッケージ。Linux for Tegra（L4T）＋CUDA＋cuDNN＋TensorRT＋VPI＋Multimedia API
  • SDK Manager: ホスト Ubuntu PC から Jetson へフラッシュ
  • nvidia-container-runtime: Docker で CUDA/GPU をコンテナに渡す
  • jetson-containers: 各 JetPack 用に最適化された Docker イメージ集
  • jtop: tegrastats を読みやすく可視化する OSS ツール（要 sudo インストール）
JetPack のバージョン整合

JetPack のバージョンによって CUDA・cuDNN・TensorRT・OpenCV のバージョンが固定されます。サードパーティ ML パッケージ（PyTorch ホイール等）も JetPack バージョン依存。NVIDIA 公式の jetson-containers イメージを使うのが衝突回避の最短ルート。

§04 / TRTTensorRT 推論最適化

  • ONNX 経由でモデルを TensorRT エンジンに変換: trtexec --onnx=model.onnx --saveEngine=model.engine
  • 精度: FP32 / FP16 / INT8 / INT4（一部）。FP16 は校正不要・精度劣化小、INT8 は キャリブレーションデータ が必要
  • カーネル融合（Conv+BN+ReLU 等）でメモリ帯域削減
  • DLA を --useDLACore=0 等で活用、GPU と並列動作で電力効率向上
  • 動的形状（Dynamic Shape）、プロファイル、Workspace サイズに注意
  • TensorRT-LLM: 大規模言語モデル向け推論最適化（Jetson Orin 系で利用）

§05 / DEEPSTREAMDeepStream / メディア

  • DeepStream SDK: GStreamer ベースの動画 AI パイプライン。複数ストリーム同時処理可能
  • 主要プラグイン: nvstreammux（バッチ）、nvinfer（推論）、nvtracker（追跡）、nvdsanalytics、nvmsgbroker（クラウド送信）
  • ハードウェアアクセラレータ: NVENC（エンコード）／NVDEC（デコード）／VIC（変換）
  • カメラ取り込み: nvarguscamerasrc（CSI、Argus）、v4l2src（USB UVC）
  • Metropolis アーキテクチャ：エッジ DeepStream → クラウド分析の典型構成

§06 / ISAACIsaac ROS / Sim

  • Isaac ROS: ROS 2 互換の GPU 加速パッケージ群（知覚、SLAM、Stereo、AprilTag、DNN ベースの perception）
  • Isaac Sim: Omniverse 上のフォトリアル＋物理シミュレータ。合成データ生成、sim2real 転移
  • Replicator: 合成データ生成 SDK（Domain Randomization）
  • ROS 2 の DDS ベース通信、QoS 設定
  • JetBot / JetRacer: 教育用ロボットキット

§07 / LLMエッジ LLM

Orin 系の登場により、エッジでの中小 LLM 実行が現実的になりました。

  • 量子化: INT4/INT8 でメモリ削減（GPTQ、AWQ、SmoothQuant、llama.cpp ggml/gguf）
  • TensorRT-LLM: NVIDIA 公式の LLM 推論最適化
  • jetson-containers のLLMイメージ: text-generation-inference、vLLM 等の Jetson 向けビルド
  • KV キャッシュ管理と PagedAttention でスループット向上
  • マルチモーダルエッジ: VLM、Edge SAM、Edge YOLO + LLM 連携
  • UMA を活かした ZeroCopy 実装でメモリ帯域を節約

§08 / MLOPSエッジ MLOps

  • OTA 更新: A/B スロット、署名検証、ロールバック（Mender、AWS IoT Greengrass、Azure IoT Edge）
  • 監視: モデルパフォーマンス、温度、電力、ドリフト検出
  • Federated Learning: エッジで学習し中央には勾配のみを集約
  • 合成データ→Sim2Real: Isaac Sim + Replicator で大量ラベル付き合成データを生成
  • Triton Inference Server (Jetson 対応): 多モデル統一サービング

§09 / ROADMAP学習ロードマップ

Phase 1 — Jetson の起動と「Hello AI World」（20〜40時間）

Orin Nano Developer Kit を入手し、SDK Manager でフラッシュ。jetson-inference リポジトリの分類・検出・セグメンテーション例を一通り動作確認。

Phase 2 — TensorRT と DeepStream（40〜80時間）

PyTorch / ONNX → TensorRT エンジン化、INT8 校正の体験。DeepStream で複数ストリーム同時推論。jtop で電力プロファイル測定。

Phase 3 — プロジェクト構築（60〜120時間）

独自のテーマ（環境モニタリング、農業、ロボット制御、エッジ LLM ボイスアシスタントなど）を選び、GitHub に公開可能なリポジトリを作成。ブログ記事＋デモ動画を準備。

Phase 4 — 認定申請

NVIDIA の Jetson AI Certification Programs ページから申請。レビューに数週間。
