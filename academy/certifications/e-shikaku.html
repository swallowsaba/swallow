E資格JDLA Deep Learning for ENGINEER

ディープラーニングを実装するエンジニアの能力を認定する資格。数学的基礎・機械学習・深層学習基礎/応用・開発と運用環境の5領域を、コードと数式の両面から問う。JDLA認定プログラムの修了が受験要件であり、本検定で AI エンジニアとしての中核が形になります。

E資格は、G検定が「事業活用」を問うのに対し、「実装する側」の能力を問う検定です。数学の式を読み、ニューラルネットの構造を設計し、フレームワークでコードに落とし込み、運用環境まで考える。AI エンジニアとしての中心軸がこの一試験に集約されています。

§02 / PREREQUISITES受験要件と認定プログラム

E資格は誰でも受けられる試験ではありません。JDLA が認定する教育プログラムのいずれかを、試験日の過去2年以内に修了していることが受験要件です。プログラムは複数存在し、費用・期間・形式（オンライン／通学）・コードの主軸言語が異なります。

プログラム選定の観点

  • 主軸フレームワーク — PyTorch 中心か、TensorFlow 中心か。受験時のフレームワーク選択と整合させる
  • 期間と料金 — 短期集中型（数週間）と長期型（数ヶ月）。費用は数万円〜数十万円
  • 修了基準 — 修了試験の有無、提出物の重さ
  • サポート — メンター、質問対応、コミュニティ

多くのプログラムは「Coursera Deep Learning Specialization の和訳＋日本独自カリキュラム」のような構成です。プログラム自体が、E資格の主要範囲をなぞるカリキュラムになっています。

§03 / SYLLABUSシラバス全体像（5領域）

  1. 数学的基礎 — 線形代数、確率・統計、情報理論、最適化
  2. 機械学習 — 教師あり／なし／強化学習、評価、特徴量工学
  3. 深層学習の基礎 — 順伝播・逆伝播、最適化、正則化、活性化
  4. 深層学習の応用 — CNN、RNN、Transformer、生成モデル、強化学習、説明性
  5. 開発・運用環境 — フレームワーク、分散学習、計算資源、エッジ・量子化、MLOps

近年は領域4（応用）と領域5（運用）の比重が増加傾向。生成AI・LLM、軽量化、責任あるAI といった文脈の出題が拡充されています。

§04 / ROADMAP学習ロードマップ（標準 100〜200時間）

Phase 1 — 数学的基盤（2〜3週）

線形代数（行列演算、固有値、特異値分解）、確率（条件付き、ベイズ、KL）、最適化（凸性、勾配法）を 「手で計算できる」レベルに。E資格の数学は深くはないが、出題されるとほぼ確実に取れる項目なので、ここを落とすと致命的。

Phase 2 — 古典機械学習（2週）

SVM、決定木、アンサンブル（バギング／ブースティング）、PCA、k-means、評価指標、交差検証、正則化。scikit-learn でひと通り実装してから理論に戻るのが効率的。

Phase 3 — 深層学習の基礎（3〜4週）

多層パーセプトロン、活性化関数、損失関数、最適化アルゴリズム（SGD/Momentum/Adam）、正則化（L1/L2/Dropout/BN/LayerNorm）。逆伝播は手書きで一度導出すること。式の意味が体に入る。

Phase 4 — 深層学習の応用（3〜4週）

CNN（畳み込み・プーリング・代表アーキテクチャ）、RNN/LSTM/GRU、Attention/Transformer、生成モデル（VAE/GAN/Diffusion）、強化学習（Q学習/DQN/方策勾配/PPO）、説明性（SHAP/Grad-CAM）。論文の原典に当たると忘れにくい。

Phase 5 — 開発・運用（1〜2週）

PyTorch / TensorFlow の API、自動微分、データローダ、分散学習（DP/DDP、Horovod）、混合精度、量子化、蒸留、プルーニング、MLOps（CI/CD、モデル監視、ドリフト検知）。

Phase 6 — 模試と弱点補強（1〜2週）

市販問題集を2周以上。誤答の周辺概念ごと潰す。本サイトのクイズも反復。

§05 / DOMAIN 1数学的基礎

線形代数

  • ベクトル・行列演算、ノルム（L1/L2/∞）
  • 固有値・固有ベクトル、対称行列の対角化
  • 特異値分解（SVD）、主成分分析の基礎
  • 行列の階数、可逆性、擬似逆行列

確率・統計

  • 確率公理、独立性、条件付き確率、ベイズの定理
  • 代表的な分布（正規、ベルヌーイ、二項、ポアソン、指数、ガンマ）
  • 期待値・分散・共分散・相関、不偏推定量
  • 最尤推定（MLE）、最大事後確率推定（MAP）、ベイズ推定

情報理論

  • 情報量、エントロピー、結合・条件付きエントロピー
  • KL ダイバージェンス、交差エントロピー
  • 相互情報量

最適化

  • 凸関数・凸最適化問題
  • 勾配降下法、ニュートン法、L-BFGS
  • ラグランジュ未定乗数法

§06 / DOMAIN 2機械学習

主要アルゴリズム

  • 線形回帰／ロジスティック回帰、正則化（Ridge / Lasso / Elastic Net）
  • SVM、カーネルトリック（RBF / Polynomial）
  • 決定木、CART、情報利得、ジニ不純度
  • アンサンブル：バギング（Random Forest）、ブースティング（AdaBoost / Gradient Boosting / XGBoost / LightGBM）
  • クラスタリング：k-means、k-means++、階層クラスタリング、DBSCAN、ガウス混合モデル
  • 次元削減：PCA、カーネルPCA、t-SNE、UMAP

評価と検証

  • 過学習・未学習、バイアス・バリアンス分解
  • 交差検証（k-fold、Stratified、TimeSeriesSplit）
  • 分類指標：Precision、Recall、F1、ROC-AUC、PR-AUC、MCC
  • 回帰指標：MAE、RMSE、R²、MAPE
  • 不均衡データ対策：SMOTE、Class Weight、Focal Loss

§07 / DOMAIN 3深層学習の基礎

順伝播と逆伝播

多層パーセプトロンの順伝播の式と、連鎖律による誤差逆伝播。計算グラフの概念は PyTorch の autograd の理解に直結します。

活性化関数

  • Sigmoid、tanh — 勾配消失の主因
  • ReLU、Leaky ReLU、PReLU、ELU、SELU
  • GELU、Swish — Transformer 系で標準
  • Softmax — 多クラス分類出力

損失関数

  • 回帰：MSE、MAE、Huber
  • 分類：Cross Entropy、Binary CE、Focal Loss
  • 埋め込み：Contrastive Loss、Triplet Loss、InfoNCE

最適化アルゴリズム

  • SGD、Momentum、Nesterov
  • AdaGrad、RMSProp、Adam、AdamW、AdaBelief
  • 学習率スケジュール：Step Decay、Cosine、Warmup、One Cycle

正則化

  • L1 / L2 重み減衰
  • Dropout（学習時のみ、推論時はスケール調整）
  • Batch Normalization、Layer Normalization、Group Normalization
  • Data Augmentation、Mixup、CutMix、Label Smoothing
  • Early Stopping

§08 / DOMAIN 4深層学習の応用

畳み込みニューラルネット（CNN）

  • 畳み込み、ストライド、パディング、プーリング
  • パラメータ共有、並進不変性
  • 代表アーキテクチャ：LeNet、AlexNet、VGG、GoogLeNet（Inception）、ResNet、DenseNet、MobileNet、EfficientNet
  • 物体検出：R-CNN ファミリー、YOLO ファミリー、SSD
  • セグメンテーション：FCN、U-Net、DeepLab、Mask R-CNN、SAM

系列モデル

  • 単純 RNN、LSTM、GRU
  • Bi-RNN、Encoder-Decoder、Attention
  • Seq2Seq、Beam Search

Transformer 系

  • Self-Attention、Multi-Head Attention、位置エンコーディング
  • Encoder のみ：BERT、RoBERTa、DeBERTa
  • Decoder のみ：GPT 系、LLaMA、Mistral
  • Encoder-Decoder：T5、BART
  • 視覚 Transformer：ViT、Swin Transformer、DETR

生成モデル

  • Autoencoder、VAE、β-VAE
  • GAN（DCGAN、Conditional GAN、StyleGAN、CycleGAN）、モード崩壊
  • Diffusion Model（DDPM、Stable Diffusion、Latent Diffusion）
  • Flow-based（RealNVP、Glow）

強化学習

  • マルコフ決定過程、Bellman 方程式
  • Q学習、SARSA、DQN（Experience Replay、Target Network）
  • 方策勾配、REINFORCE、Actor-Critic、A3C
  • PPO、TRPO、SAC、DDPG
  • AlphaGo / AlphaZero / MuZero の系譜

説明可能AI

  • LIME、SHAP
  • Grad-CAM、Saliency Map、Integrated Gradients
  • Attention の可視化

§09 / DOMAIN 5開発・運用環境

フレームワークと自動微分

  • PyTorch（動的グラフ）、TensorFlow / Keras（静的＋eager）
  • JAX / Flax（関数型・XLA）
  • ONNX による相互運用

計算資源とスケーリング

  • GPU、TPU、NPU の特性
  • 混合精度学習（FP16 / BF16）、Tensor Core
  • データ並列、モデル並列、パイプライン並列、ZeRO
  • 分散通信：NCCL、All-Reduce

推論最適化と軽量化

  • 量子化（INT8、INT4、QAT、PTQ）
  • 蒸留（Knowledge Distillation）
  • プルーニング、構造化／非構造化
  • LoRA / QLoRA / Adapter（PEFT）
  • KV キャッシュ、Flash Attention、Speculative Decoding

MLOps と運用

  • データ版数管理（DVC）、モデルレジストリ
  • CI/CD パイプライン、再現性
  • ドリフト検知（データドリフト／コンセプトドリフト）
  • A/B テスト、シャドーデプロイ、カナリアリリース

§10 / FRAMEWORKPyTorch と TensorFlow の選択

E資格の試験では、試験開始時にフレームワークを選択し、試験中の変更はできません。事前に決め、過去問演習でも揃えておきます。

PyTorch を選ぶ場合

  • 研究・論文実装が多い。Hugging Face、Stable Diffusion 系もほぼ PyTorch
  • 動的グラフで「Pythonっぽく」書ける
  • 近年は受験者の主流

TensorFlow / Keras を選ぶ場合

  • 業務でデプロイ環境が GCP / TF Serving / TF Lite で固まっている
  • Keras の高レベル API でコードが短い
  • モバイル・エッジ最適化が成熟

どちらでも合格は十分可能。自分が学習教材で読んだコードのフレームワークを選ぶのが最も自然です。

§11 / CHECKLIST直前チェックリスト

クリックでチェックが入り、ブラウザに保存されます。

  • 逆伝播を手書きで1層分（線形層 + 活性化）導出できる
  • Adam の更新式を書ける（一次・二次モーメント、バイアス補正）
  • Batch Norm と Layer Norm の違いと使い分けを言える
  • 畳み込み出力サイズの式を覚えている（H、W、padding、stride）
  • LSTM の3つのゲートと役割を説明できる
  • Self-Attention の Q・K・V の役割と式を書ける
  • VAE と GAN の損失関数の本質的違いを説明できる
  • Diffusion Model の順方向・逆方向過程を説明できる
  • DQN の Experience Replay と Target Network の必要性を言える
  • PPO のクリッピングの目的を説明できる
  • PyTorch の autograd の動作を1分で説明できる
  • DataParallel と DistributedDataParallel の違いを言える
  • 量子化（PTQ vs QAT）の違いを説明できる
  • LoRA がパラメータ効率的な理由を式で説明できる
  • クロスエントロピーと KL ダイバージェンスの関係を導出できる
  • フレームワークを決定し、試験中に変更しないと決めた
  • 本サイトの E資格クイズで 75% 以上を一度取った
