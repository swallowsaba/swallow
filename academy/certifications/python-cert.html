§02 / BASIC基礎試験

Python 3 の文法・標準ライブラリ・組み込み型の基本を確認します。プログラミング初学者の到達目標として広く使われる試験。

出題範囲

  • 食欲をそそる序章、Python インタプリタの使い方
  • 形式ばらない Python の紹介（数値、文字列、リスト）
  • その他の制御フローツール、データ構造、モジュール
  • 入出力、エラーと例外、クラス、標準ライブラリめぐり、仮想環境とパッケージ

頻出ポイント

  • イテラブル／イテレータ／ジェネレータの違い、yield
  • リスト・タプル・辞書・集合の mutable/immutable と性能特性
  • 例外処理 try / except / else / finally の挙動
  • スライス記法、リスト内包表記、関数引数（*args, **kwargs, デフォルト値の罠）
  • if __name__ == '__main__': の意味

§03 / DATAデータ分析試験

NumPy・pandas・matplotlib・scikit-learn を中心に、データ前処理・可視化・モデル構築の流れを確認する試験。

出題範囲

  • データエンジニアの役割、Python と環境（pip、conda、Jupyter）
  • 数学の基礎（線形代数、微分、確率・統計の最低限）
  • ライブラリによる分析実践：NumPy、pandas、Matplotlib、scikit-learn
  • 応用：機械学習（教師あり／なし、評価指標）、前処理、特徴量

頻出ポイント

  • NumPy の ブロードキャスト・ファンシーインデキシング・ブール抽出
  • pandas の .loc / .iloc、groupby、merge、pivot_table、欠損処理
  • matplotlib の Figure/Axes 構造、subplots、ラベルと凡例
  • scikit-learn の fit / predict / transform、Pipeline、train_test_split、cross_val_score
  • 回帰・分類の評価指標（MSE/MAE/R²、Accuracy/Precision/Recall/F1/AUC）

§04 / PRACTICAL実践試験

2022年以降に登場した試験。実務寄りの Python（テスト、ロギング、型ヒント、並行処理、パッケージング、セキュリティなど）を扱う。

出題範囲

  • 環境構築、コーディング規約・lint・format（PEP 8、black、ruff）
  • 型ヒント（typing、mypy）、データクラス（dataclass、Pydantic）
  • ロギング、設定管理、CLI、Web API（FastAPI 等）
  • テスト（pytest、fixture、parametrize、coverage）、CI/CD
  • 並行・並列（threading、multiprocessing、asyncio）、性能最適化
  • パッケージング（pyproject.toml、PEP 621、Poetry / Hatch / uv）
  • セキュリティ（依存スキャン、Secrets 管理）

§05 / DATA-PRACTICALデータ分析実践試験

データ分析の 実務プロセスに踏み込んだ試験。データの取得・整形・モデリング・評価・運用までを扱う。

出題範囲

  • データの収集・前処理（API、SQL、スクレイピング基礎）
  • 特徴量エンジニアリング、リーク防止、不均衡データ
  • モデル比較（線形、決定木、ランダムフォレスト、勾配ブースティング）
  • パイプライン化、ハイパーパラメータ最適化、交差検証戦略
  • モデル解釈（SHAP、Permutation Importance）
  • 運用（モデル保存、推論API、モニタリング）

§06 / MODERNモダン Python の常識

環境管理

  • 標準: python -m venv .venv ＋ pip
  • 高機能: Poetry / Hatch / uv（高速）。pyproject.toml（PEP 621）に依存・ビルド設定を集約
  • conda: 科学計算寄り、コンパイラ依存パッケージの統一が容易

型と検証

  • 型ヒントは 3.10以降の組み込み型（list[int]、X | None）が標準
  • dataclass はデフォルト、検証や(de)シリアライズが必要なら Pydantic v2
  • 静的解析: mypy / pyright、lint+format は ruff がデファクト

非同期

  • async def ＋ await ＋ asyncio.TaskGroup（3.11+）
  • HTTP は httpx、CLI は typer、ORM は SQLAlchemy 2.0

性能と GIL

  • I/O バウンドは asyncio / threading、CPU バウンドは multiprocessing / numpy ベクトル化 / C 拡張
  • 3.12 で perf 改善、3.13 で experimental no-GIL。将来的に並列性能が一段変わる可能性

§08 / ROADMAP学習ロードマップ

Phase 1 — 基礎試験（30〜60時間）

『Pythonチュートリアル』（主教材）を写経しながら通読。リスト・辞書・例外・クラスを手を動かして書く。

Phase 2 — データ分析試験（40〜80時間）

『Pythonによるあたらしいデータ分析の教科書』（主教材）と並行で、Kaggle Titanic を NumPy / pandas / scikit-learn で完走。可視化も習慣化。

Phase 3 — 実践 / データ分析実践（各 60〜100 時間）

pytest・型ヒント・FastAPI・Docker を自作プロジェクトに組み込む。データ分析実践は中規模の Kaggle コンペや業務データで全工程を体験。
