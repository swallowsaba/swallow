Frame 02 — Advanced  ·  2026 Spring

応用Where the frontier meets practice.

「基本」で学んだ要素技術が、現代のAIシステムの中でどう組み上げられているかを俯瞰する章です。Transformer・LLM・拡散モデル・強化学習・RAG・Agent・MLOps・責任あるAI — 試験と実務の両方でいま問われる主題を、最短経路で。

§01 / TRANSFORMERTransformer の構造

2017年の「Attention Is All You Need」以降、自然言語処理のみならず画像・音声・マルチモーダルまで席巻したアーキテクチャ。鍵はSelf-Attention — 系列のすべての位置が、すべての位置を参照できる仕組み。

主要コンポーネント

  • Self-Attention — Query / Key / Value を線形変換し、Q·K のスケールド内積をソフトマックスして重み、V を加重和
  • Multi-Head Attention — 複数のヘッドで並列に異なる関係を学習
  • Position-wise FFN — 各位置に独立にかかる2層MLP
  • Positional Encoding — 系列順序を表現（Sinusoidal、学習可能、RoPE、ALiBi など）
  • Layer Norm / Residual — 学習安定化（Pre-LN が現在の主流）

Attention の計算量は系列長 L に対して O(L²)。長文を扱うときの主要なボトルネックで、Flash Attention・線形近似・Sparse Attention・状態空間モデル（Mamba 系）といった改良が活発に研究されている領域。

派生

  • Encoder のみ — BERT、RoBERTa（理解・分類）
  • Decoder のみ — GPT 系（生成）
  • Encoder-Decoder — T5、BART（翻訳・要約）
  • Vision Transformer (ViT) — 画像を patch に分解して Transformer に

§02 / LLMLLM の設計と学習

大規模言語モデル（LLM）は、Decoder-only Transformer を巨大なコーパスで自己教師あり学習させたもの。能力はスケーリング則（パラメータ数・データ量・計算量にべき乗で性能が伸びる）に従う。

学習の3段階

  1. 事前学習 (Pre-training) — 大規模テキストで次トークン予測。世界知識と言語能力を獲得
  2. 指示チューニング (SFT) — 人手で作った指示・応答ペアで微調整
  3. アライメント — RLHF / DPO / RLAIF などで人の好みに整合させる

推論時の制御

  • Temperature — 出力分布の鋭さ（低いほど決定的）
  • Top-k / Top-p (nucleus) — 候補トークンの絞り込み
  • Beam Search — 確率の積で複数候補を保持（要約・翻訳で有効）
  • Greedy — 各ステップで最大確率（決定的、貧弱になりがち）

プロンプトエンジニアリング

  • Zero-shot / Few-shot — 例示なし／少数例示
  • Chain-of-Thought (CoT) — 段階的に考えさせて推論力を引き出す
  • Self-Consistency — 複数サンプリングして多数決
  • Tree-of-Thought — 推論を木構造で探索
  • 役割設定・制約・出力形式の明示 — 実務での基本

効率化

  • 量子化 (Quantization) — INT8 / INT4 / GPTQ / AWQ で重みを縮小
  • 蒸留 (Distillation) — 大モデルを教師に小モデルを学習
  • LoRA / QLoRA — 低ランク差分のみ学習する PEFT 手法
  • KV キャッシュ — 推論時の attention 計算を再利用
  • MoE (Mixture of Experts) — 一部の専門家のみを活性化（スパース）

§03 / DIFFUSION拡散モデル

画像・音声・動画生成で主流となった生成モデル。データに少しずつノイズを加える順過程と、ノイズを除去する逆過程を学習する。

原理（直感）

  1. 本物の画像 x₀ にガウスノイズを段階的に加え、x_T はほぼ純粋ノイズになる（順過程）
  2. ニューラルネットに「いまどれだけノイズが乗っているか」を予測させる
  3. 逆向きに少しずつノイズを除去すれば、純ノイズから画像を生成できる（逆過程）

派生と発展

  • DDPM — 確率的拡散の基礎
  • DDIM — 決定論的サンプリングで高速化
  • Latent Diffusion (Stable Diffusion) — VAE の潜在空間で拡散して高解像度を可能に
  • Classifier-free Guidance — 条件あり／なしを足し合わせて条件従順性を強める
  • ControlNet・LoRA — 既存モデルへの条件付け・微調整
  • Flow Matching — 拡散の一般化として近年注目

§04 / RL強化学習

環境と相互作用しながら、累積報酬を最大化する方策を学習する枠組み。教師あり学習と違い、正解ラベルではなく報酬信号のみが手がかり。

基本概念

  • 状態 s、行動 a、報酬 r、方策 π — MDP（マルコフ決定過程）として定式化
  • 価値関数 V(s)・行動価値関数 Q(s,a) — 将来得られる累積報酬の期待値
  • ベルマン方程式 — 価値の再帰的定義
  • 探索と活用 (Exploration vs Exploitation) — ε-greedy、UCB、Thompson Sampling

代表的手法

  • Q学習 / SARSA — テーブル形式の古典
  • DQN — Q関数をニューラルネットで近似、経験再生・ターゲットネット
  • 方策勾配 / REINFORCE — 方策を直接微分
  • Actor-Critic / A2C / A3C — 価値関数と方策を同時に学習
  • PPO — 安定性が高く、LLM のアライメントにも使われる主力
  • SAC、TD3 — 連続行動空間で強い
  • AlphaGo / AlphaZero — モンテカルロ木探索 × ニューラルネット

RLHF (Reinforcement Learning from Human Feedback) — 人間の選好データから報酬モデルを学習し、PPO で LLM を微調整する手法。ChatGPT 以降のチャット型LLMを支える中核技術。近年は DPO（Direct Preference Optimization）のようにRLを使わない代替手法も普及している。

§05 / RAG検索拡張生成 (RAG)

LLM の知識の鮮度・専門性・事実性の弱点を補うための定番アーキテクチャ。外部知識ベースから関連文書を検索し、コンテキストに入れて回答させる。

標準的なパイプライン

  1. Ingest — ドキュメントをチャンクに分割（数百〜千トークン）
  2. Embed — 各チャンクを埋め込みベクトル化
  3. Index — ベクトルDBに格納（FAISS、Pinecone、pgvector、Qdrant など）
  4. Retrieve — 質問を埋め込み、類似上位 k 件を取得
  5. Rerank — 関連性スコアで再ランキング（任意）
  6. Generate — 取得文書をプロンプトに入れて LLM に回答させる

設計上の論点

  • チャンクサイズ・重なり — 小さすぎると文脈喪失、大きすぎるとノイズ
  • ハイブリッド検索 — ベクトル検索 × BM25（キーワード）
  • クエリ書き換え — HyDE、Multi-Query、Step-back prompting
  • 引用 — どの文書からの情報か明示する
  • 評価 — Recall@k、MRR、faithfulness、answer relevancy（RAGAS、TruLens）

§06 / AGENTLLMエージェント

LLM がツールを使い、外部環境に作用しながら、複数ステップで目標を達成するシステム。チャットボットの先にある実務的な活用形態。

主要なパターン

  • ReAct — Reasoning（思考）と Acting（行動）を交互に
  • Tool Use / Function Calling — 関数を呼び出して結果を観測
  • Plan-and-Execute — 先に計画を立て、サブタスクに分解して実行
  • Multi-Agent — 役割を分担した複数エージェントの協調
  • MCP (Model Context Protocol) — ツール提供を標準化するプロトコル

実務で重要なポイント

  • 失敗からの回復（リトライ、人間への引き継ぎ）
  • コスト・レイテンシ管理（無限ループや暴走の防止）
  • ガードレール（出力検証、入力サニタイズ、プロンプトインジェクション対策）
  • 観測可能性（trace、logs、評価データの蓄積）

§07 / MLOPSMLOps / LLMOps

ML モデルを本番に届け、運用し続けるための実践。コードだけでなく、データ・モデル・特徴量・実験すべてをバージョン管理する。

運用ライフサイクル

  1. データ収集・前処理 — DVC、データバージョニング
  2. 実験管理 — MLflow、Weights & Biases、Comet
  3. 特徴量ストア — Feast、SageMaker Feature Store
  4. モデルレジストリ — 学習済みモデルの管理
  5. CI/CD for ML — テスト・自動学習・デプロイ
  6. サービング — REST、gRPC、バッチ、ストリーミング
  7. モニタリング — 精度、レイテンシ、データドリフト、コンセプトドリフト
  8. 再学習 — トリガーに応じた自動／半自動の更新

LLMOps 固有の関心事

  • プロンプトのバージョン管理と A/B テスト
  • 評価データセットと回帰テスト（LLM-as-a-judge）
  • トークン課金とコスト監視
  • ハルシネーション検出、有害出力の検知
  • キャッシング、ルーティング（モデル使い分け）

§08 / RESPONSIBLE AI責任あるAI

AI を作るだけでなく、誰に・どのように影響するかを考えるのが現代のエンジニアの責任。試験でも実務でも比重が増している領域。

主要な観点

  • 公平性 (Fairness) — 性別・人種など属性ごとの性能格差を測定・是正
  • 透明性・説明可能性 (XAI) — LIME、SHAP、Grad-CAM、Integrated Gradients
  • プライバシー — 差分プライバシー、連合学習、データ最小化
  • セキュリティ — 敵対的攻撃、データポイズニング、モデル抽出、プロンプトインジェクション
  • 頑健性 — 分布外データへの耐性、不確実性の定量化
  • 環境負荷 — 学習・推論のエネルギー消費、効率化の責任

規制・ガイドライン（2026 時点）

  • EU AI Act — リスクベース規制、汎用AI（GPAI）への透明性義務
  • 日本 AI 事業者ガイドライン — 開発者・提供者・利用者向け
  • NIST AI Risk Management Framework — 米国の自主的フレームワーク
  • G7 広島AIプロセス — 国際協調の枠組み
  • OECD AI 原則 — 国際的な共通基盤

「動く」「速い」「強い」だけでなく、「説明できる」「公平である」「壊れにくい」「漏れない」を満たすことが、現代の AI システムに求められる最低限。試験対策のための知識でなく、設計判断そのものに使う。
