§02 / PYTORCHPyTorch で MNIST 分類

PyTorch画像分類所要 10 分

深層学習の Hello World。28×28 の手書き数字を分類する CNN を、数十行で書き、Colab の T4 GPU で 1 分ほど学習させます。

import torch, torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

device = 'cuda' if torch.cuda.is_available() else 'cpu'
tf = transforms.ToTensor()
train_loader = DataLoader(datasets.MNIST('.', train=True, download=True, transform=tf), batch_size=128, shuffle=True)
test_loader  = DataLoader(datasets.MNIST('.', train=False, download=True, transform=tf), batch_size=256)

class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(1,32,3,padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Conv2d(32,64,3,padding=1), nn.ReLU(), nn.MaxPool2d(2),
            nn.Flatten(), nn.Linear(64*7*7,128), nn.ReLU(), nn.Linear(128,10)
        )
    def forward(self, x): return self.net(x)

model = CNN().to(device)
opt = torch.optim.Adam(model.parameters(), lr=1e-3)
loss_fn = nn.CrossEntropyLoss()

for epoch in range(3):
    model.train()
    for x, y in train_loader:
        x, y = x.to(device), y.to(device)
        opt.zero_grad(); loss = loss_fn(model(x), y); loss.backward(); opt.step()
    model.eval(); correct = 0
    with torch.no_grad():
        for x, y in test_loader:
            x, y = x.to(device), y.to(device)
            correct += (model(x).argmax(1) == y).sum().item()
    print(f'epoch {epoch} acc={correct/10000:.4f}')

読みどころ。(1) DataLoader がバッチ・シャッフルを面倒見る (2) nn.Sequential で層を積む (3) opt.zero_grad → loss.backward → opt.step の3点セットが PyTorch 学習ループの基本形 (4) model.eval() + torch.no_grad() で評価時の挙動を切替。

§03 / HUGGING FACEHugging Face で LLM 推論

LLMTransformers所要 10 分

事前学習済みモデルを数行で動かす。日本語が得意な小型モデルを例に、テキスト生成と分類の最小サンプル。

# pip install -U transformers accelerate
from transformers import pipeline

# 1) テキスト生成（小型モデルで動作確認）
gen = pipeline('text-generation', model='rinna/japanese-gpt2-small', device_map='auto')
print(gen('今日は良い天気なので', max_new_tokens=40, do_sample=True, temperature=0.8)[0]['generated_text'])

# 2) 感情分類（事前学習済み多言語モデル）
clf = pipeline('sentiment-analysis', model='cardiffnlp/twitter-xlm-roberta-base-sentiment')
print(clf(['最高だった', 'がっかり', '普通']))

パラメータの読み方

  • do_sample=True + temperature — 確率的に生成（多様性）
  • do_sample=False — Greedy（決定的）
  • top_p — nucleus sampling（累積確率で切る）
  • repetition_penalty — 同じ語の連発を抑える
  • max_new_tokens — 生成する最大トークン数

§04 / RAGローカルで動く最小 RAG

RAGLangChain なし所要 20 分

外部APIなしで「文書を読み込んで埋め込み、質問に近い文書を検索して回答に使う」という RAG の核を、ライブラリを最小限にして書く。

# pip install sentence-transformers numpy
from sentence_transformers import SentenceTransformer
import numpy as np

# 1) コーパス（実際は PDF や Markdown から読み込む）
docs = [
    '勾配降下法は損失関数の勾配と逆方向に少しずつ進むことで最小値を探す。',
    'Transformer は Self-Attention によって系列の全位置を相互参照する。',
    '過学習は学習データに過剰適合し、未知のデータで性能が落ちる現象。',
    'L2 正則化は重みの二乗和をロスに加え、大きな重みにペナルティを課す。',
]

# 2) 埋め込み
embedder = SentenceTransformer('intfloat/multilingual-e5-small')
doc_emb = embedder.encode(['passage: ' + d for d in docs], normalize_embeddings=True)

# 3) 検索
def retrieve(query, k=2):
    q = embedder.encode('query: ' + query, normalize_embeddings=True)
    scores = doc_emb @ q
    idx = np.argsort(-scores)[:k]
    return [(docs[i], float(scores[i])) for i in idx]

# 4) 質問 → 文脈付きプロンプト（LLM への入力）
q = '過学習を防ぐ方法は？'
ctx = retrieve(q, k=2)
prompt = f'以下の文書を参考に「{q}」に答えてください。\n\n' + '\n'.join(f'- '+c[0] for c in ctx)
print(prompt)
# この prompt を LLM (OpenAI / Anthropic / ローカル) に渡せば RAG 完成

実プロダクトへの拡張

  • FAISS / Qdrant / pgvector で大量文書に対応
  • BM25 と組み合わせたハイブリッド検索
  • クエリ書き換え（HyDE、Multi-Query）
  • Reranker（cross-encoder）で上位 k を再ランキング
  • 引用箇所を回答に明示

§05 / SAGEMAKERAWS SageMaker で訓練→デプロイ

AWSMLA-C01所要 30 分（要 AWS アカウント）

SageMaker SDK の核は「Estimator で訓練 → fit → deploy → predict」の流れ。AWS MLA-C01 試験でも頻出の構文。

# pip install sagemaker
import sagemaker
from sagemaker.pytorch import PyTorch
from sagemaker import get_execution_role

session = sagemaker.Session()
role = get_execution_role()  # SageMaker Studio 内なら自動取得

estimator = PyTorch(
    entry_point='train.py',           # 学習スクリプト
    source_dir='src',                  # 周辺コード
    role=role,
    instance_type='ml.g5.xlarge',     # GPU インスタンス
    instance_count=1,
    framework_version='2.3.0',
    py_version='py311',
    hyperparameters={'epochs': 5, 'lr': 1e-3},
)

# 1) 訓練ジョブ起動（S3 上のデータを参照）
estimator.fit({'train': 's3://my-bucket/data/train'})

# 2) リアルタイム推論エンドポイントへデプロイ
predictor = estimator.deploy(
    initial_instance_count=1,
    instance_type='ml.m5.large',
)

# 3) 推論
result = predictor.predict({'inputs': [[1.0, 2.0, 3.0]]})
print(result)

# 4) 後片付け（課金を止める）
predictor.delete_endpoint()

関連サービスの早見

  • SageMaker Pipelines — MLパイプラインを DAG として定義（CI/CD for ML）
  • Model Registry — モデルバージョンと承認フローを管理
  • Model Monitor — データドリフト・モデルドリフトの監視
  • Feature Store — オンライン／オフライン特徴量の一元管理
  • Clarify — バイアス検出・説明可能性
  • JumpStart — 事前学習済みモデルのワンクリック展開
  • Inference オプション: Real-time / Serverless / Async / Batch transform

§06 / JETSONJetson でエッジ推論

NVIDIAEdge AI所要 30 分

NVIDIA Jetson（Nano / Orin Nano / AGX Orin）は手のひらサイズのエッジAIプラットフォーム。クラウドにデータを送らずに、現地で推論を完結させる。

セットアップの流れ

  1. JetPack（CUDA・cuDNN・TensorRT 同梱の SDK）を SD カードに書き込み
  2. 初回起動でユーザ設定 → jtop でリソース監視
  3. NVIDIA NGC から PyTorch / TensorRT のコンテナを docker pull
# Jetson 用の PyTorch コンテナを起動
sudo docker run --runtime nvidia -it --rm \
  -v $(pwd):/work -w /work \
  nvcr.io/nvidia/l4t-pytorch:r36.2.0-pth2.2-py3

# コンテナ内: モデルを TensorRT で最適化
import torch, torch_tensorrt
model = torch.load('model.pt').eval().cuda()
trt = torch_tensorrt.compile(model,
    inputs=[torch_tensorrt.Input((1,3,224,224), dtype=torch.float16)],
    enabled_precisions={torch.float16})
torch.jit.save(trt, 'model_trt.ts')

Jetson AI Specialist で求められる観点

  • JetPack・CUDA・cuDNN・TensorRT の役割と関係
  • FP32 → FP16 / INT8 量子化による高速化
  • DeepStream による映像パイプライン構築
  • Isaac ROS によるロボティクス統合
  • 消費電力モード（10W / 15W / MAXN）と性能のトレードオフ
