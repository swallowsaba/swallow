§01 / INTRODUCTIONはじめに：AIとは何か

1.1 「AI」という曖昧な言葉

「AI（人工知能）」という用語は、研究者の間でも明確な合意のない曖昧な言葉です。チューリングは1950年の論文 “Computing Machinery and Intelligence” で、知能を哲学的に定義する代わりに、「人間と区別できないほど自然な会話ができるかどうか」 を判定基準とすることを提案しました（後のチューリングテスト）。この発想は、知能を 関数の入出力（振る舞い） で定義しようとする「機能主義」の立場です。一方、ジョン・サールは1980年に 「中国語の部屋」 という思考実験を提示し、入出力が知的に見えても内部に 意味理解 が存在しなければ知能とは呼べない、と反論しました。

この対立は単なる哲学論ではなく、現代の LLM 評価にも直結します。LLM は流暢に応答しますが、それが「理解」なのか「統計的なパターン補完」なのか、研究者間で見解は割れています。G検定では 両方の立場を理解した上で、どちらが優れているか結論を出さない 姿勢が求められます。

1.2 強いAI／弱いAI／AGI／ASI

用語の関係を整理します：

用語定義現在の達成状況
弱いAI特定タスクに特化したAI。タスクが解ければ「意識」「理解」は不要達成（画像認識、翻訳など）
強いAI人間と同等の意識・主観・自己認識を持つAI未達成。実現可能性も論争中
ANI (特化型)弱いAIとほぼ同義。Artificial Narrow Intelligence達成
AGI (汎用)人間と同等以上に、領域横断で課題を解けるAI研究途上。LLMが近いとする意見と否定的意見が並立
ASI (超知能)あらゆる領域で人間を超えるAI仮説段階

レイ・カーツワイルは2005年の著書 『The Singularity Is Near』 で、2045年頃に技術的特異点（シンギュラリティ）が到来する と予測しました。これは「ASI に到達することで、技術進歩が人間に予測不能な速度に達する」というシナリオです。受け止め方は研究者ごとに異なります。

1.3 知能を測ることの困難さ

AI が「人間並み」かどうかを評価するには、知能を定量化する必要があります。しかし「知能」という対象自体に普遍的な定義がないため、評価には常にバイアスが伴います。現在用いられる代表的なベンチマーク：

  • MMLU（Massive Multitask Language Understanding）：57 分野の知識問題で LLM の汎用知識を測る
  • HumanEval / SWE-bench：プログラミング能力
  • ARC-AGI：抽象的パターン推論。LLM が苦手な領域として注目された
  • GAIA：実世界的なエージェント能力

ベンチマーク・スコアの上昇は 「知能」の上昇とは必ずしも等価でない 点に注意。学習データに類題が混入する contamination、過剰最適化、評価軸の偏りなどが常に問題となります。

1.4 古典的なAIの問題

G検定で出題される代表的な未解決問題：

  • フレーム問題（McCarthy, 1969）：実世界では考慮すべき要因が無数にあり、「何を考慮し何を無視するか」を有限ステップで決定できない問題。シンボリックAIの限界の象徴。
  • シンボルグラウンディング問題（Harnad, 1990）：記号と実世界の意味を結びつけられないという問題。「リンゴ」という記号が実物のリンゴと結びついていない計算機は、本当の意味を持てない。
  • トイプロブレム：理想化された限定環境（チェス盤など）で解ける問題。実世界に拡張するとフレーム問題に当たる。
  • ノーフリーランチ定理（Wolpert, 1996）：全ての問題で最強のアルゴリズムは存在しない。問題ごとに帰納バイアスを与える必要がある、という数学的命題。
→ クロスリファレンス
これらの未解決問題は、現代の生成AIにも姿を変えて現れます。ハルシネーション（事実と異なる出力）はシンボルグラウンディング問題の現代版とも言えます。詳細は 応用 §05 ハルシネーションと事実性。

§02 / HISTORYAIの歴史と冬の時代

AI の歴史は 3 つのブームと2回の「冬」 として整理されます。それぞれのブームを支えた技術と、冬を招いた限界を理解することで、現在の生成AIブーム（第4次）が将来どこに向かう可能性があるかを推測する基盤になります。

2.1 第1次AIブーム（1956–1970s前半）：探索と推論

1956年、ダートマス会議で マッカーシーが「Artificial Intelligence」という用語を提案。この会議には マービン・ミンスキー、クロード・シャノン、ナサニエル・ロチェスター ら錚々たる顔ぶれが集まりました。

当時の中心技術は 探索と推論：

  • 幅優先・深さ優先探索、後にαβ枝刈りなどで効率化
  • ヒューリスティック探索（A* など）
  • 定理証明（Logic Theorist）、初期のチャットボット（ELIZA, 1966）

この時代の AI は「ルールを書いて推論させる」ものでした。しかし 組み合わせ爆発（探索木が指数的に大きくなる）と 常識推論の困難さ（フレーム問題）から、現実問題への拡張に失敗し、最初の AIの冬 が訪れます。

2.2 第2次AIブーム（1980年代）：エキスパートシステム

1980年代、専門家の知識をルールとして書き下し、コンピュータに推論させる「エキスパートシステム」が商業的成功を収めました。代表例：

  • MYCIN：感染症診断（スタンフォード大）
  • DENDRAL：化学物質の構造同定
  • 第五世代コンピュータプロジェクト（日本、1982-1992）：論理プログラミング（Prolog）に巨額投資

しかし知識獲得のボトルネック（専門家の知識をルール化する人件費が膨大）と 例外処理の脆さ（ルールに無い状況に弱い）から、再び冬が訪れます。

2.3 第3次AIブーム（2006–現在）：機械学習と深層学習

転換点は2006年、ジェフリー・ヒントンらによる 事前学習（Deep Belief Network） の提案、そして2012年の ILSVRC（ImageNet画像認識コンテスト）で AlexNet（Krizhevsky, Sutskever, Hinton）が2位以下を10ポイント以上引き離して優勝した事件でした。

このブームの中心技術：

  • 深層学習（CNN、RNN/LSTM、Transformer）
  • 大規模データ（ImageNet、Common Crawl）
  • GPU/TPU による並列計算
  • End-to-End学習：特徴量を人手設計せず、生データから学習

主な里程標：

年出来事
2012AlexNet が ILSVRC を制覇
2014GAN（Goodfellow ら）、Seq2Seq（Sutskever ら）
2015ResNet（残差接続で152層）、AlphaGo が囲碁プロに勝利
2017Transformer 論文 “Attention Is All You Need”
2018BERT、GPT-1。事前学習＋微調整パラダイムが確立
2020GPT-3（1750億パラメータ）。Scaling Laws の発表
2022ChatGPT 公開、Stable Diffusion 公開
2023–GPT-4、Claude、Gemini、Llama。マルチモーダル化

2.4 第4次AIブームと呼ばれる現在（2022–）

ChatGPT 公開以降、基盤モデル（FM） と 生成AI を中心とした新たな段階が始まったと多くの研究者が見なしています。第3次の延長か、第4次と呼ぶべきかは議論があります。特徴：

  • 巨大スケール（パラメータ・データ・計算量）
  • 事前学習で得られた汎用能力を プロンプトで引き出す
  • マルチモーダル（テキスト・画像・音声・動画を統一）
  • エージェント化（外部ツール呼び出し、自律行動）
⚠ よくある誤解
「AIブーム」と「実用化」は別物。第2次ブームでもエキスパートシステムは商業利用されていたし、第3次ブームの初期では深層学習が一部の画像認識タスクに限定されていました。冬の時代の主因は 「期待値と実装可能性のギャップ」 です。現在の生成AIにも同じ罠が潜みます。

§03 / MATH線形代数：ベクトル・行列・固有値

機械学習は ベクトル・行列で書かれた最適化問題 です。線形代数を「公式の暗記」から「図形的直感」に切り替えることで、後段が劇的に分かりやすくなります。

3.1 ベクトル：方向と大きさを持つ点

n次元ベクトル x = (x₁, x₂, …, xₙ)ᵀ は、n 次元空間上の 原点からの矢印 です。機械学習では特徴量を並べた表現として使います。例：身長・体重を持つ人を x = (170, 65)ᵀ と書く。

内積（dot product） の定義：

x · y  =  xᵀy  =  x₁y₁ + x₂y₂ + … + xₙyₙ  =  ||x|| ||y|| cos θ
(3.1)

内積の幾何学的意味は 「y を x の方向に射影した長さ × ||x||」。すなわち 2 つのベクトルの『似ている度合い』 を表します。cos θ が 1（同方向）なら最大、0（直交）なら 0、-1（逆方向）なら最小。

3.2 ノルム：ベクトルの「長さ」

ノルム定義機械学習での用途
L¹（マンハッタン）||x||₁ = Σ|xᵢ|Lasso 回帰のスパース化正則化
L²（ユークリッド）||x||₂ = √Σxᵢ²Ridge 回帰、距離計算、勾配クリッピング
L∞（最大値）||x||_∞ = max|xᵢ|敵対的サンプル攻撃の制約

ノルムは 「ベクトル → スカラー」の写像で、(a) 非負性、(b) 同次性 ||αx|| = |α| ||x||、(c) 三角不等式 ||x+y|| ≤ ||x|| + ||y|| を満たします。

3.3 行列：線形変換

m×n 行列 A は、n 次元ベクトルを m 次元ベクトルに変換する関数 として理解できます。Ax = y は「x を A という変換で y に移す」と読みます。

幾何的にイメージしやすい2次元の変換例：

変換行列効果
恒等I = [[1,0],[0,1]]何もしない
X方向に2倍拡大[[2,0],[0,1]]横に引き伸ばす
θ回転[[cos θ,-sin θ],[sin θ,cos θ]]原点中心に回転
X軸鏡映[[1,0],[0,-1]]X 軸で折り返す

3.4 固有値と固有ベクトル：変換の「不変方向」

行列 A に対して、ベクトル v ≠ 0 とスカラー λ が Av = λv を満たすとき、v を 固有ベクトル（eigenvector）、λ を 固有値（eigenvalue） と呼びます。意味は 「行列 A をかけても、方向が変わらず大きさだけ λ 倍になる特別な方向」。

導出：固有値方程式
Av = λv より (A - λI)v = 0。v ≠ 0 で解を持つには、(A - λI) が非可逆、すなわち：
det(A - λI) = 0    (固有方程式)
これは λ に関する n 次方程式。n 個の固有値（重複・複素数を含む）が得られる。
例題：2×2 行列の固有値
A = [[3,1],[0,2]] の固有値を求める。
det(A - λI) = (3-λ)(2-λ) - 0 = 0 → λ = 3, 2
λ = 3 の固有ベクトル: (A - 3I)v = 0 → [[0,1],[0,-1]]v = 0 → v = (1,0)ᵀ
λ = 2 の固有ベクトル: (A - 2I)v = 0 → [[1,1],[0,0]]v = 0 → v = (1,-1)ᵀ
意味：x 軸方向は3倍、(1,-1) 方向は2倍に伸縮される変換。

機械学習における重要な活用：

  • PCA（主成分分析）：データの共分散行列の固有ベクトル＝最大分散方向。次元削減・可視化の標準手法
  • ヘッセ行列の固有値：最適化において、損失関数の局所形状（凸／凹／鞍点）を判定
  • 固有値分解・SVD：データ圧縮、画像圧縮、推薦システム（行列分解）

3.5 特異値分解（SVD）：万能の行列分解

任意の m×n 行列 A は次のように分解できます：

A  =  U Σ Vᵀ
(3.2)
  • U: m×m 直交行列（左特異ベクトル）
  • Σ: m×n 対角様行列（非負の特異値 σ₁ ≥ σ₂ ≥ … ≥ 0 を対角に）
  • V: n×n 直交行列（右特異ベクトル）

特異値の大きさは「その方向の重要度」。上位 k 個だけ残せば 低ランク近似 となり、データ圧縮・推薦システム（Netflix Prize）・LoRA（後述）の基礎になる。

→ シミュレーションで確認

§04 / MATH微積分と最適化

4.1 微分：変化率

関数 f(x) の微分 f'(x) は、x が微小に変化したときの f の変化率（接線の傾き）：

f'(x) = lim_{h→0} [f(x+h) - f(x)] / h
(4.1)

4.2 偏微分と勾配

多変数関数 f(x₁, …, xₙ) を、特定の変数 xᵢ について微分したものが 偏微分 ∂f/∂xᵢ。これを全ての変数について並べた列ベクトルが 勾配（gradient）：

∇f(x) = (∂f/∂x₁, ∂f/∂x₂, …, ∂f/∂xₙ)ᵀ
(4.2)

勾配の 幾何学的意味 は「その点で f が最も急に増加する方向」。逆向き -∇f は最も急に減少する方向で、これが勾配降下法の根拠 です。

4.3 連鎖律：合成関数の微分

ニューラルネットの学習を支える最重要法則。f(g(x)) の微分は：

d/dx [f(g(x))]  =  f'(g(x)) · g'(x)
(4.3)

多変数版（行列の連鎖律）が バックプロパゲーション の数学的基盤になります（§11 で詳述）。

4.4 ヘッセ行列：二階微分の集まり

多変数関数の二階偏微分を行列にまとめたものを ヘッセ行列 H と呼びます：

Hᵢⱼ = ∂²f / (∂xᵢ ∂xⱼ)
(4.4)

勾配 ∇f = 0 の点（極値候補）でのヘッセ行列の 固有値の符号 によって：

  • 全ての固有値 > 0 → 極小（凸）
  • 全ての固有値 < 0 → 極大（凹）
  • 正負混合 → 鞍点（saddle point）

高次元の損失関数では、極小よりも 鞍点 の方が遥かに多いことが知られており、これが現代の最適化研究の重要な視点です。

4.5 勾配降下法（Gradient Descent）

損失関数 L(θ) を最小化したい。パラメータ θ を、勾配の逆方向に少しずつ動かす：

θₜ₊₁  =  θₜ  -  η · ∇L(θₜ)
(4.5)

η は 学習率（learning rate）。小さいと収束が遅く、大きいと振動・発散。

変種説明
バッチ勾配降下全データの平均勾配で1回更新。安定だが遅い
SGD（確率的勾配降下）1サンプルごとに更新。ノイジーだが早い・汎化が良い場合あり
ミニバッチSGDm サンプル（典型 32〜256）で平均勾配。実務標準
Momentum過去の更新方向を慣性として加算。振動抑制
RMSProp各パラメータごとに勾配の二乗平均で学習率を適応化
AdamMomentum + RMSProp。実務でほぼ標準
例題：1次元の勾配降下を手で1ステップ
L(θ) = (θ - 3)², 初期 θ₀ = 0, η = 0.1。
∇L(0) = 2(0-3) = -6
θ₁ = 0 - 0.1 × (-6) = 0.6
θ₂ = 0.6 - 0.1 × (2(0.6-3)) = 0.6 - 0.1 × (-4.8) = 1.08
... 反復すると θ → 3（最小点）に収束する。
⚠ 学習率を大きくし過ぎると
η = 1.5 にすると θ₁ = 0 - 1.5 × (-6) = 9。次に θ₂ = 9 - 1.5 × 12 = -9。発散（振動拡大）します。学習率は最適化問題の最重要ハイパーパラメータ。
→ シミュレーションで確認
シミュレーション §01 勾配降下 で η を動かしながら、収束・振動・発散の境界を体験できます。

§05 / MATH確率と統計

5.1 確率と確率変数

確率変数 X は「値が確率的に決まる変数」。離散と連続があります。

離散の場合：確率質量関数（PMF） P(X = x)。サイコロなら P(X=1)=...=P(X=6)=1/6。

連続の場合：確率密度関数（PDF） f(x)。値そのものではなく 積分 が確率を表す：

P(a ≤ X ≤ b)  =  ∫ₐᵇ f(x) dx,     ∫₋∞^∞ f(x) dx = 1
(5.1)

5.2 期待値と分散

E[X] = Σ x · P(X=x) （離散）
E[X] = ∫ x · f(x) dx （連続）
Var[X] = E[(X - E[X])²] = E[X²] - E[X]²
(5.2)

期待値は 「無限回試行したときの平均」。分散はそのばらつき（散らばり）。標準偏差 σ = √Var。

5.3 主要な確率分布

ベルヌーイppp(1-p)二値結果（成功/失敗）
二項 Bin(n,p)n, pnpnp(1-p)n回の二値試行の成功数
ポアソン Po(λ)λλλ稀少事象（到着数など）

5.4 中心極限定理（Central Limit Theorem）

機械学習で最も重要な統計定理。独立同分布な確率変数 X₁, ..., Xₙ の和（または平均）は、n が大きいとき正規分布に従う。これは 元の分布が何であっても 成り立つ。

実用上の意味：標本平均は正規分布で近似できる → 信頼区間が計算できる → 仮説検定ができる、という連鎖が動きます。ニューラルネットの BatchNorm が機能する直観的根拠の一つでもあります。

5.5 ベイズの定理

P(A | B)  =  P(B | A) · P(A) / P(B)
(5.3)

パラメータ θ とデータ D に対して：

P(θ | D)  ∝  P(D | θ) · P(θ)
事後   ∝  尤度  ×  事前
(5.4)

ベイズ的思考は 「データを見る前の信念（事前分布）を、データを得た後の信念（事後分布）に更新する」 枠組み。L2正則化はガウス事前、L1はラプラス事前を仮定した MAP 推定と等価、など機械学習との接続も深い。

5.6 最尤推定（MLE）

観測データ x₁, ..., xₙ が与えられたとき、それを 最も「もっともらしく」する パラメータ θ を選ぶ：

θ̂_MLE = argmax_θ Π P(xᵢ | θ) = argmax_θ Σ log P(xᵢ | θ)
(5.5)
例題：ベルヌーイの最尤推定
コイン投げで n 回中 k 回表が出た。表の確率 p を MLE で推定。
尤度 L(p) = pᵏ(1-p)ⁿ⁻ᵏ
対数尤度 log L(p) = k log p + (n-k) log(1-p)
微分 d/dp log L = k/p - (n-k)/(1-p) = 0 → p̂ = k/n
結論：MLE は単に「成功比率」。直感と一致。

§06 / MATH情報理論の入口

6.1 自己情報量とエントロピー

事象 x が確率 p(x) で起こるとき、その 自己情報量（驚き） は -log p(x)。確率が低いほど大きく、確率1なら0。

エントロピー は自己情報量の期待値：

H(X) = -Σ p(x) log p(x)
(6.1)

分布の「不確実さ」 の尺度。一様分布が最大、決定論的（一点集中）なら 0。

6.2 KLダイバージェンス

2つの分布 P, Q の 違い を測る量（厳密な距離ではない）：

KL(P ‖ Q)  =  Σ p(x) log[p(x) / q(x)]
(6.2)

性質：KL(P‖Q) ≥ 0、P=Q で 0、非対称（距離公理を満たさない）。VAE の損失、強化学習の信頼領域、変分推論など至る所で出現。

6.3 クロスエントロピー（分類の損失関数）

正解分布 p（one-hot）と予測分布 q に対し：

H(p, q) = -Σ p(x) log q(x) = H(p) + KL(p ‖ q)
(6.3)

ニューラルネット分類の標準損失。最小化することは、KL(p‖q) の最小化と同等（H(p) は定数）。

§07 / ML機械学習のパラダイム

7.1 3つの大分類

パラダイム入力出力例
教師あり学習(x, y) ペアx→y を予測する関数画像分類、回帰、翻訳
教師なし学習x のみ構造（クラスタ・低次元・密度）K-means、PCA、GAN
強化学習環境からの状態と報酬方策（policy）囲碁、ロボット制御、RLHF

7.2 教師あり学習の構造

訓練データ D = {(x₁, y₁), ..., (xₙ, yₙ)} から、未知の関数 f*: X → Y を近似する関数 f̂ を学ぶ。形式化：

  • 仮説空間 H：考えるモデル族（線形回帰、決定木、NN など）
  • 損失関数 L(f(x), y)：予測誤差の定義
  • 経験リスク最小化：D 上の平均損失を最小化する f を選ぶ
f̂ = argmin_{f ∈ H} (1/n) Σ L(f(xᵢ), yᵢ)
(7.1)

7.3 半教師あり・自己教師あり学習

  • 半教師あり学習：少量のラベル付きと大量のラベルなしを併用
  • 自己教師あり学習：データ自身から擬似ラベルを作る（BERT のマスク言語モデル、画像の対比学習 SimCLR、MAE など）。近年の事前学習はほぼこれ

§08 / ML古典的機械学習手法

8.1 線形回帰：すべての出発点

入力 x ∈ ℝᵈ に対して、線形に予測：

ŷ = wᵀx + b
(8.1)

損失：二乗誤差 L = (1/2n) Σ (ŷᵢ - yᵢ)²。閉形式解（正規方程式）：

ŵ = (XᵀX)⁻¹ Xᵀy
(8.2)

X はデータ行列（n×d）。XᵀX が特異（共線性あり）だと解が不安定 → リッジ回帰へ。

8.2 リッジとラッソ（正則化）

過学習を防ぐため、重みの大きさにペナルティを課す：

Ridge (L²): L + λ ||w||² → ŵ = (XᵀX + λI)⁻¹ Xᵀy
Lasso (L¹): L + λ ||w||₁ → 解析解なし、座標降下法
(8.3)

Lasso は 多くの重みを 0 にする（スパース化） 性質があり、特徴量選択に使えます。Ridge は重みを 全体的に小さく しますが 0 にはしません。両者を併用したものが Elastic Net。

8.3 ロジスティック回帰：二値分類

線形出力 z = wᵀx + b を シグモイド関数で確率に変換：

σ(z) = 1 / (1 + e⁻ᶻ),    P(y=1 | x) = σ(wᵀx + b)
(8.4)

損失は クロスエントロピー（最尤推定と等価）。多クラス版はソフトマックス回帰。

8.4 サポートベクトルマシン（SVM）

クラス間の マージン（境界からの最近接点までの距離）を最大化 する分類器。直線で分けられない場合は カーネルトリック で高次元に写像（実際に写像せず内積だけ計算）。

代表的なカーネル：線形、多項式、RBF（ガウシアン）。RBF は最も柔軟だが、ハイパーパラメータ調整が必須。

8.5 決定木 / ランダムフォレスト / 勾配ブースティング

決定木 はデータを if-then ルールで再帰的に分割する。木が深くなりすぎると過学習。

ランダムフォレスト：複数の決定木の多数決（バギング）。各木は ブートストラップ標本 と 特徴量サブセット で学習することで多様化。

勾配ブースティング（GBDT）：木を逐次的に追加し、前の木の誤差（負勾配）を埋める ように学習。実装としては XGBoost, LightGBM, CatBoost。表データの王者として現在も多くのコンペで上位を占める。

8.6 K-means クラスタリング

  1. K 個のセントロイドをランダム初期化
  2. 各データを最も近いセントロイドに割当（割当ステップ）
  3. 各クラスタのセントロイドを平均で更新（更新ステップ）
  4. 収束まで 2-3 を反復

Lloyd アルゴリズムと呼ばれます。初期化が悪いと局所解に落ちるので K-means++ 初期化が推奨。K の選択には エルボー法・シルエット係数。

→ シミュレーションで確認
シミュレーション §06 K-means で1ステップずつ収束を観察できます。

8.7 主成分分析（PCA）

高次元データを 分散最大方向 に射影して次元削減。共分散行列の固有ベクトル（第一主成分、第二主成分、...）に射影します。

数学的には、データ中心化 → 共分散行列計算 → 固有値分解（または SVD）→ 上位 k 個の固有ベクトルに射影、という手順。

§09 / ML評価と過学習

9.1 訓練・検証・テストの分割

モデルの汎化性能を正しく測るには、データを 三分割：

  • 訓練（train）：パラメータを学習
  • 検証（validation）：ハイパーパラメータ・モデル選択
  • テスト（test）：最終的な汎化性能の評価（複数回見てはいけない）

データが少ない場合は K-fold 交差検証（CV）：訓練データを K 分割し、各 fold を検証用にして K 回学習し平均。

9.2 過学習とバイアス・バリアンス分解

汎化誤差 = バイアス² + バリアンス + 既約誤差。

用語意味原因対策
バイアスモデルが真の関数から系統的にずれるモデルが単純すぎる（underfit）モデル容量を上げる、特徴量を増やす
バリアンス訓練データのノイズに過敏モデルが複雑すぎる（overfit）正則化、データ拡大、早期終了、Dropout

9.3 分類の評価指標

混同行列：

予測：陽性予測：陰性
真：陽性TP（真陽性）FN（偽陰性）
真：陰性FP（偽陽性）TN（真陰性）
  • Accuracy = (TP+TN)/全件。クラス不均衡だと役立たず
  • Precision（適合率） = TP/(TP+FP)。陽性と判定したうち本当に陽性
  • Recall（再現率） = TP/(TP+FN)。本当の陽性のうち捕まえられた
  • F1 = Precision と Recall の調和平均 = 2PR/(P+R)
  • ROC-AUC：閾値非依存の総合指標。完璧 1.0、ランダム 0.5
  • PR-AUC：不均衡データでは ROC より適切
例題：医療診断の評価
がん検診で TP=80, FN=20, FP=200, TN=9700。
Accuracy = (80+9700)/10000 = 97.8%（一見良い）
Precision = 80/(80+200) = 28.6%
Recall = 80/(80+20) = 80%
医療では Recall（見落としを減らす）が重要。Accuracy は誤解を招く。

9.4 正則化の手段一覧

  • L1 / L2 / Elastic Net：重みのペナルティ
  • Early Stopping：検証誤差が悪化し始めたら学習停止
  • Dropout（DL）：学習時にユニットをランダムに無効化
  • Batch Normalization：各層の入力を正規化（暗黙の正則化）
  • データ拡張：画像なら回転・反転、テキストなら言い換え
  • アンサンブル：複数モデルの予測を平均

§10 / NNニューラルネットの最小構成

10.1 パーセプトロン

Rosenblatt (1958) が提案した最初期のニューラルネットモデル。入力 x の線形結合に活性化関数（ステップ関数）を適用：

y = step(wᵀx + b),    step(z) = 1 if z ≥ 0 else 0
(10.1)

1969年、Minsky & Papert が 『Perceptrons』 で XOR が表現できない（線形分離不可能）ことを証明。これが第1次 AI 冬の一因に。

10.2 多層パーセプトロン（MLP）と万能近似定理

パーセプトロンを多層に重ねると 非線形関数 を表現できるようになります（XOR も解ける）。

万能近似定理（Cybenko, 1989; Hornik, 1991）：1層の隠れ層を持つ MLP は、ユニット数を十分大きくすれば 任意の連続関数を任意の精度で近似できる。

⚠ 万能近似定理の限界
「表現できる」と「学習できる」は別。1層で表現可能でも、現実には深層化することで遥かに少ないユニット数で表現できることが多い（指数的圧縮）。これが「Deep Learning」の本質的価値の一つ。

10.3 活性化関数

関数式特徴用途
Sigmoid1/(1+e⁻ᶻ)出力 (0,1)、勾配消失二値分類出力
Tanh(eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ)出力 (-1,1)、ゼロ中心RNN の中で歴史的に多用
ReLUmax(0, z)計算軽、勾配消失緩和、Dying ReLU 問題CNN/MLP 標準

§11 / NNバックプロパゲーション

MLP の学習を可能にした最重要アルゴリズム。連鎖律を計算グラフ上で系統的に適用することで、多層に対する勾配を効率的に計算します。

11.1 計算グラフ

ニューラルネットを「ノード（演算）」と「エッジ（テンソル）」のグラフとして表現します。例：2層 MLP は：

x → [線形 W₁] → z₁ → [ReLU] → h → [線形 W₂] → z₂ → [Sigmoid] → ŷ → [損失 L]

11.2 フォワードパス

入力 x から損失 L まで、順に値を計算します。途中の中間値（活性化）は全て保存しておく（バックワードで使うため）。

11.3 バックワードパス（連鎖律の系統的適用）

出力側から入力側に向かって、各パラメータについて勾配を計算：

∂L/∂W₂ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂W₂
∂L/∂W₁ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂h · ∂h/∂z₁ · ∂z₁/∂W₁
(11.1)

共通部分を 再利用 するのが鍵。これにより 勾配計算は順伝播の定数倍程度のコスト で済みます。これがなければ深層学習は計算量的に不可能でした。

11.4 自動微分（Automatic Differentiation）

バックプロパゲーションを 「計算グラフから勾配を機械的に計算する」 一般的なアルゴリズムとして実装したもの。PyTorch・TensorFlow・JAX の中核。

  • 順モード自動微分：入力1つに対する出力多数の場合に効率的
  • 逆モード自動微分（バックプロパゲーションと同じ）：出力1つ（損失）に対する入力多数（パラメータ）の場合に効率的。深層学習はこちら

11.5 勾配消失と勾配爆発

深い NN では、勾配を順番にかけ算するため：

  • 小さい数字を多数かける → 勾配消失。下層が学習しない
  • 大きい数字を多数かける → 勾配爆発。発散

対策：

  • ReLU 系活性化：正領域で勾配 1
  • 適切な重み初期化：He（ReLU 用）、Xavier/Glorot（tanh 用）
  • 残差接続（ResNet, 2015）：y = F(x) + x で勾配を直接伝える
  • BatchNorm / LayerNorm：各層の入力分布を安定化
  • 勾配クリッピング：勾配の大きさを閾値で抑制
→ シミュレーションで確認

§12 / NN学習を成功させる工夫

12.1 ミニバッチ学習

バッチサイズ m（典型 32〜256）で勾配を平均することで、SGD のノイズを抑えつつ並列計算の恩恵を受ける。バッチサイズが大きすぎると汎化性能が落ちる傾向（シャープミニマ問題）。

12.2 学習率スケジューリング

  • Step Decay：一定エポックごとに学習率を 1/10 に
  • Cosine Annealing：コサイン関数で滑らかに減衰
  • Warmup：最初の数ステップは小さい学習率から徐々に上げる（Transformer で標準）

12.3 重み初期化

不適切な初期化は学習が一切進まない原因になります。

  • Xavier/Glorot 初期化：Var(W) = 2/(n_in + n_out)。sigmoid/tanh 向け
  • He 初期化：Var(W) = 2/n_in。ReLU 系向け
  • LeCun 初期化：Var(W) = 1/n_in。SELU 向け

12.4 正規化レイヤ

正規化正規化軸主な用途
BatchNormバッチ方向CNN（バッチサイズが大きいとき）
LayerNorm特徴量方向（サンプルごと）RNN、Transformer（バッチ非依存）
GroupNormチャネルグループ方向バッチサイズが小さい CNN
InstanceNorm各サンプル、各チャネル画像スタイル変換

12.5 Dropout の本質

学習時に 各ユニットを確率 p（典型 0.5）で 0 にする。これにより：

  • 暗黙のアンサンブル効果（毎ステップ違うサブネットを学習）
  • ユニット間の共依存抑制

推論時は無効化（あるいは出力を 1-p 倍）するのが「inverted dropout」。

12.6 最適化アルゴリズム（再訪）

Adam が事実上の標準。ただし：

  • 大規模言語モデルでは AdamW（重み減衰を分離）が主流
  • 古典 ML や小規模 NN では SGD + Momentum の方が汎化が良いこともある
  • 巨大モデルでは LAMB, LION, Sophia など新世代も実用化
