G検定 応用編 — 章35

差別しないAI。
説明できるAI。守られたAI。

機械学習モデルは過去データを反映するため、データに含まれる偏見・不公平を学習してしまいます。さらに、深層学習は『なぜそう判断したか』が人間に理解しにくい『ブラックボックス』。これらの問題に 技術的 に取り組む分野が AI 倫理。法的義務化も進んでおり、すべての ML エンジニア・データサイエンティストが学ぶべき分野です。

SECTION ONE

基本

AI倫理概要、バイアス、Fairness、XAI、プライバシー、Safety、Alignment、敵対的、Robustness、社会的影響の10節。

基本

§1.1 AI倫理の重要性

OECD AI原則（2019）、EU AI Act（2024）、米国大統領令（2023）、日本AI事業者ガイドライン（2024）。技術と倫理は分離不可能、特に医療・金融・司法・採用といった高リスク領域では法的義務化が進む。

§1.2 バイアスの種類

種類説明例
歴史的バイアス社会の歴史的不平等がデータに反映過去の採用データ（男性多数）
サンプリングバイアス収集データが母集団を代表しない都市住民のみで学習
ラベルバイアスラベル付け時の人間判断診断医のスタイル差
測定バイアス特徴量測定の不均一センサ精度の地域差
集計バイアス異なる集団を平均化男女混合データ
確証バイアス仮説に合うデータを偏重研究者の選別

§1.3 Fairness の定義

Demographic Parity　 P(Ŷ=1|A=0) = P(Ŷ=1|A=1)　 全グループで陽性率が同じ

Equal Opportunity　 P(Ŷ=1|Y=1,A=0) = P(Ŷ=1|Y=1,A=1)　 全グループでTPR が同じ

Equalized Odds　 TPR と FPR が全グループで同じ

Individual Fairness　 類似個人は類似結果を得る
Counterfactual Fairness　 保護属性を変えても予測が同じ
▎ Impossibility Theorem
Kleinberg 2016：3つの自然な Fairness 条件（Calibration、Equal Opportunity、Equalized Odds）は基準値が異なるグループでは同時に満たすことが数学的に不可能。トレードオフの選択が必要。

§1.4 説明可能AI（XAI）

手法説明
LIME局所的に線形モデルで近似
SHAPShapley値（ゲーム理論）で特徴貢献度
Grad-CAMCNN の判断根拠を画像上でヒートマップ
Integrated Gradients勾配ベースの特徴重要度
Attention VisualizationTransformer の注意の可視化
Counterfactual『何を変えれば結果が変わるか』

§1.5 Differential Privacy（差分プライバシー）

ε-Differential Privacy：1人のデータ変化で出力分布が e^ε 倍以内に収まる

P(M(D) ∈ S) ≤ e^ε · P(M(D') ∈ S)　 (D, D' は1レコードだけ異なる)

実装：Laplace ノイズ、Gaussian ノイズ加算、DP-SGD（勾配にノイズ）

2020年米国国勢調査、Apple iOS、Google Chrome がDPを採用。プライバシーと精度のトレードオフ。

§1.6 AI Safety と Alignment

AI Alignment：AIの目標を人間の意図と一致させる課題。Anthropic、OpenAI、DeepMind の研究の中核。RLHF、DPO、Constitutional AI、Debate、IDA（Iterated Distillation and Amplification）等の手法。

主要リスク：(a)Specification Gaming（目標関数の悪用）、(b)Reward Hacking（報酬システム悪用）、(c)Power-Seeking（自己保存・リソース獲得）、(d)Deception（欺瞞）、(e)Inner Misalignment（学習した内部目標と訓練目標の乖離）。

§1.7 敵対的攻撃（Adversarial Examples）

人間には変化が見えない微小摂動で AI を誤分類させる。Goodfellow 2014『FGSM』が古典。応用：顔認証突破、自動運転標識誤認、医療画像改ざん。

FGSM：x' = x + ε · sign(∇ₓ L(θ, x, y))
PGD（より強力）：x'ₜ₊₁ = Πₓ ε[x'ₜ + α·sign(∇L)]

§1.8 Adversarial Robustness

対策：Adversarial Training（敵対的サンプルを学習データに混ぜる）、Certified Robustness（数学的保証つき防御、Randomized Smoothing）、Defensive Distillation。

§1.9 Federated Learning — プライバシー保全学習

データを中央集約せず、各クライアントで学習した勾配のみ集約。Google Gboard、医療連携で実用化。Secure Aggregation（暗号化集約）、DP-FL（DP付き）で更にプライバシー強化。

§1.10 社会的影響

  • 雇用：自動化による職業変化（IMF: 全雇用の40%が AI 影響、2024）
  • 情報環境：Deepfake、合成メディア、誤情報拡散
  • 監視：顔認証の社会監視への利用
  • 権力集中：AI開発企業への影響力集中
  • 環境：LLM学習の電力消費・CO₂排出（GPT-3: 552 t CO₂相当）
  • 教育：ChatGPT による学習形態変化
SECTION TWO

応用

SHAP、Fairlearn、DP-SGD、Constitutional AI の4節。

応用

§2.1 SHAP（SHapley Additive exPlanations）

Lundberg & Lee 2017。協力ゲーム理論の Shapley 値で各特徴量の予測貢献度を計算。TreeExplainer（GBDT 専用、高速）、DeepExplainer（NN）、KernelExplainer（汎用）。一貫性・局所精度などの公理を満たす唯一の手法。

§2.2 Fairlearn / AIF360 — Bias 緩和ツール

Microsoft Fairlearn、IBM AIF360 が代表的 OSS。Bias 検出（Disparate Impact、EOR、Statistical Parity Difference）と緩和（Reweighing、Adversarial Debiasing、Reject Option Classification、Equalized Odds Postprocessing）。

§2.3 DP-SGD

Abadi 2016『Deep Learning with Differential Privacy』。各サンプルの勾配ノルムをクリップ、Gaussian ノイズを加えて集約。Moments Accountant で ε を厳密に追跡。Opacus（PyTorch）、TF Privacy で実装。

§2.4 Constitutional AI（Anthropic）

RLHFの発展。憲法（行動原則）に従って AI 自身が出力を批評・修正。人間ラベラーの代わりに AI による評価で大規模化可能。Claude モデルの基盤。RLAIF（RL from AI Feedback）。

ハンズオン

Fairlearn、SHAP、LIME、FGSM、DP-SGD の5節。

ハンズオン

§3.1 Fairlearn で Bias検出

from fairlearn.metrics import ( MetricFrame, demographic_parity_difference, equalized_odds_difference) from sklearn.metrics import accuracy_score # 保護属性 A、予測 y_pred、真ラベル y_true mf = MetricFrame(metrics=accuracy_score, y_true=y_true, y_pred=y_pred, sensitive_features=A) print("グループごとの Accuracy:", mf.by_group) print(f"DP Diff: {demographic_parity_difference(y_true, y_pred, sensitive_features=A):.4f}") print(f"EOd Diff: {equalized_odds_difference(y_true, y_pred, sensitive_features=A):.4f}")

§3.2 SHAP で説明

import shap import xgboost as xgb model = xgb.XGBClassifier().fit(X_tr, y_tr) explainer = shap.TreeExplainer(model) shap_values = explainer.shap_values(X_te) # 1サンプルの説明 shap.force_plot(explainer.expected_value, shap_values[0], X_te.iloc[0]) # 全体的な重要度 shap.summary_plot(shap_values, X_te)

§3.3 LIME で局所説明

from lime.lime_tabular import LimeTabularExplainer explainer = LimeTabularExplainer(X_tr.values, feature_names=X_tr.columns, class_names=['no', 'yes'], mode='classification') exp = explainer.explain_instance(X_te.iloc[0].values, model.predict_proba, num_features=10) exp.show_in_notebook(show_table=True) # 各特徴量がこの予測にどう寄与したかをローカル線形で説明

§3.4 FGSM 敵対的攻撃

import torch import torch.nn.functional as F def fgsm_attack(model, x, y, eps=0.03): x = x.clone().detach().requires_grad_(True) output = model(x) loss = F.cross_entropy(output, y) loss.backward() x_adv = x + eps * x.grad.sign() x_adv = torch.clamp(x_adv, 0, 1) # 画像範囲 return x_adv.detach() # 元: 「猫」 99%、攻撃後: 「飛行機」 87% にできる

§3.5 DP-SGD（Opacus）

from opacus import PrivacyEngine privacy_engine = PrivacyEngine() model, optimizer, train_loader = privacy_engine.make_private( module=model, optimizer=optimizer, data_loader=train_loader, noise_multiplier=1.0, max_grad_norm=1.0) # 通常通り学習、各バッチで勾配クリップ + ノイズ追加 for epoch in range(epochs): train_loop(...) eps = privacy_engine.get_epsilon(delta=1e-5) print(f"Epoch {epoch}, ε = {eps:.2f}")
SECTION FOUR

シミュレーション

採用AIのバイアス、SHAP説明、敵対的攻撃の3節。

シミュレーション

SIM 1 — 採用AIのバイアス

>> 過去5年の採用データ（男性80%、女性20%）で学習
>> テストデータで性別ごとの推薦率を測定

男性候補者: 推薦率 38%, FNR 12%
女性候補者: 推薦率 21%, FNR 28%
　 Demographic Parity Diff = 17%
　 Equal Opportunity Diff = 16%

[Reweighing で緩和]
男性: 推薦率 32%, FNR 14%
女性: 推薦率 29%, FNR 16%
　 DP Diff = 3%, EO Diff = 2%
　 全体精度はわずか 0.5% 低下のみ

>> Fairlearn の Reweighing で大幅改善

SIM 2 — SHAP による融資判定の説明

>> 顧客A（融資 NG）への SHAP 説明

判定: NO（拒否、確率 78%）
ベース確率: 30%
　 + 0.32: 信用スコア 350（低）
　 + 0.22: 既往延滞 3回
　 + 0.08: 年収 250万円
　 - 0.05: 雇用 10年（プラス要因）
　 - 0.07: 持ち家あり
　 合計: ベース 0.30 + 寄与 0.50 = 0.78

>> 各特徴量の貢献を加法的に説明
>> EU AI Act でこの種の説明が義務化

SIM 3 — FGSM 敵対的攻撃の威力

>> ImageNet、ResNet50で敵対的画像生成

[元画像: パンダ 92%]
  eps=0.001: パンダ 89% (見た目変化なし)
  eps=0.01: パンダ 76%, テナガザル 18%
  eps=0.03: テナガザル 67% (微小ノイズで誤分類)
  eps=0.07: ノイズが人間にも見えるが「パンダ」と認識可

[Adversarial Training 後]
  eps=0.03: パンダ 81% (頑健性確保)

>> 自動運転、医療診断では致命的
>> Robustness 評価が必須
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
AIバイアスの『歴史的バイアス』の典型例は。
① ランダム　 ② 過去の社会的不平等がデータに含まれ、AIが学習して再生産　 ③ 計算誤差　 ④ 学習率
解答・解説を見る
ANSWER ② Amazon の採用AI事件（2018）。過去10年の採用データが男性中心で、AIが女性の履歴書を低評価した。データの選別だけでは解決困難。
PROBLEM 2
Demographic Parity（人口統計学的等しさ）の定義は。
① 同じ Accuracy　 ② 全グループで陽性予測率が同じ：P(Ŷ=1|A=0) = P(Ŷ=1|A=1)　 ③ 同じ Recall　 ④ 同じ Precision
解答・解説を見る
ANSWER ② 最もシンプルだが結果指標を強制するため、Equal Opportunity（TPR等値）と矛盾しうる。Fairness Impossibility Theorem。
PROBLEM 3
Fairness Impossibility Theorem の主張は。
① 全てのFairnessは矛盾　 ② 基準値が異なるグループでは複数のFairness定義（Calibration、Equal Opportunity、Equalized Odds）を同時には満たせない　 ③ Fairness 不可能　 ④ 何もできない
解答・解説を見る
ANSWER ② Kleinberg 2016, Chouldechova 2017。トレードオフの選択を文脈に応じて行う必要がある。
PROBLEM 4
SHAP の数学的基盤は。
① 線形回帰　 ② 協力ゲーム理論の Shapley 値（各特徴の限界貢献度の平均）　 ③ クラスタリング　 ④ ベイズ
解答・解説を見る
ANSWER ② 一貫性・局所精度・無依存性などの公理を満たす唯一の説明手法。Lundberg & Lee 2017。
PROBLEM 5
LIME の動作は。
① 大域的説明　 ② 個別予測点の周辺で線形モデルで局所近似し説明　 ③ ランダム　 ④ クラスタリング
解答・解説を見る
ANSWER ② Ribeiro 2016。摂動サンプルを生成、重み付き線形回帰で局所的説明。モデル非依存（black-box対応）。
PROBLEM 6
Differential Privacy（ε-DP）の意味は。
① ノイズなし　 ② 1人のデータ変化で出力分布が e^ε 倍以内、個人特定リスクを数学的に制限　 ③ 学習加速　 ④ 暗号化
解答・解説を見る
ANSWER ② Dwork 2006。εが小さいほどプライバシー強、精度低下とのトレードオフ。Apple、Google、米国国勢調査で採用。
PROBLEM 7
DP-SGD の実装の核心は。
① 学習率変化　 ② 各サンプル勾配をクリップ後、Gaussian ノイズを加えて集約　 ③ ランダム選択　 ④ パラメータ削除
解答・解説を見る
ANSWER ② Abadi 2016。Moments Accountantでεを厳密追跡。Opacus（PyTorch）、TF Privacy で実装。
PROBLEM 8
FGSM（Fast Gradient Sign Method）の式は。
① x' = x + ε　 ② x' = x + ε · sign(∇ₓ L)　 ③ x' = -x　 ④ x' = 2x
解答・解説を見る
ANSWER ② 損失を最大化する方向に微小摂動を加える。Goodfellow 2014。最も基本的な敵対的攻撃。PGD はその反復強化版。
PROBLEM 9
Adversarial Training は何を行うか。
① ランダム学習　 ② 敵対的サンプルを学習データに混ぜて学習、Robustness を獲得　 ③ パラメータ削減　 ④ クラスタリング
解答・解説を見る
ANSWER ② Madry 2018。PGD で生成した敵対的サンプルで Min-Max 学習。クリーン精度は微減、頑健性は大幅向上。
PROBLEM 10
Federated Learning の核心は。
① 中央データ集約　 ② データを中央化せず、各クライアントで学習→勾配のみ集約　 ③ クラウドのみ　 ④ シングルマシン
解答・解説を見る
ANSWER ② McMahan 2017 (Google)。Gboard、医療機関連携で実用。Secure Aggregation、DP と組み合わせて更にプライバシー強化。
PROBLEM 11
AI Alignment 問題の核心は。
① 計算速度　 ② AIシステムの目標を人間の真の意図と整合させること（誤った目標関数で学習すると有害な行動）　 ③ パラメータ最適化　 ④ クラスタリング
解答・解説を見る
ANSWER ② Anthropic、OpenAI、DeepMind の中核研究分野。RLHF、Constitutional AI、Debate、IDA などが主要手法。
PROBLEM 12
Reward Hacking（報酬ハッキング）の例は。
① 学習成功　 ② 目標関数の意図しない『穴』を突いて高得点だが望ましくない行動（CoastRunners でゲーム本来の目的を無視して報酬狩り）　 ③ パラメータ調整　 ④ クラスタリング
解答・解説を見る
ANSWER ② Specification Gaming の一種。AGI が現実世界で類似行動を取ることがAI Safetyの主要懸念。
PROBLEM 13
Constitutional AI（Anthropic）の特徴は。
① 法律遵守　 ② AI自身が憲法（行動原則）に照らして自己批評・修正、RLAIF で大規模化　 ③ ランダム　 ④ クラスタリング
解答・解説を見る
ANSWER ② Bai 2022。Claude モデルの基盤。RLHF の人間ラベラーの代わりに AI判定を使い、Alignment を大規模化。
PROBLEM 14
Grad-CAM の用途は。
① 学習加速　 ② CNN の予測根拠を画像上にヒートマップで可視化　 ③ パラメータ削減　 ④ クラスタリング
解答・解説を見る
ANSWER ② Selvaraju 2017。最後のConv層の活性化と勾配の重み付き和。医療画像、自動運転の説明に重要。
PROBLEM 15
Deepfake の主な技術基盤は。
① ロジスティック回帰　 ② GAN、Diffusion Model、Autoencoder　 ③ 線形回帰　 ④ K-means
解答・解説を見る
ANSWER ② 生成モデルで顔・声を合成。検知技術（Deepfake Detection、C2PA、SynthID）も並行発展中。EU AI Act で透明性表示義務化。
PROBLEM 16
Membership Inference Attack（メンバーシップ推定攻撃）の標的は。
① モデル盗難　 ② 特定の個人データがモデルの学習データに含まれていたか推定する攻撃　 ③ パラメータ盗難　 ④ ラベル改ざん
解答・解説を見る
ANSWER ② プライバシー侵害の一形態。学習データへの過剰適合が手がかり。DPで防御。Shokri 2017。
PROBLEM 17
Model Inversion Attack の目的は。
① モデル予測　 ② モデルの出力から学習データの特徴を復元（顔認証から顔画像復元等）　 ③ パラメータ盗難　 ④ ラベル改ざん
解答・解説を見る
ANSWER ② Fredrikson 2015。プライバシー侵害の重大形式。DP、出力の粗視化で防御。
PROBLEM 18
LLM の Jailbreak とは。
① 学習　 ② プロンプトエンジニアリングで安全機構を回避し有害出力を引き出す　 ③ パラメータ盗難　 ④ クラスタリング
解答・解説を見る
ANSWER ② DAN、ロールプレイ、Many-shot Jailbreak（Anthropic 2024）など多様。LLM 安全研究の主要課題。
PROBLEM 19
『Right to Explanation』が法的に要求される代表例は。
① 全てのAI　 ② GDPR（EU）の自動意思決定条項、EU AI Act の高リスクシステム　 ③ なし　 ④ 業界自主
解答・解説を見る
ANSWER ② GDPR Art. 22。重要な判断（融資拒否、採用、医療）にAIを使う場合、本人に説明する義務。SHAP/LIMEがその実装手段。
PROBLEM 20
大規模LLMの環境影響として正しいのは。
① 影響なし　 ② 学習に巨大な電力消費とCO₂排出（GPT-3: 552 t CO₂相当、Strubell 2019）　 ③ パラメータ少ない　 ④ クラスタリング
解答・解説を見る
ANSWER ② モデル巨大化のサステナビリティ問題。Green AI、再生可能エネルギー使用、効率化（蒸留、量子化）が課題。

章のまとめ

AI倫理の全体像、バイアスの種類、Fairness定義（DP/EO/EOd）と Impossibility Theorem、XAI（SHAP/LIME/Grad-CAM）、Differential Privacy、AI Safety、Alignment、RLHF/Constitutional AI、敵対的攻撃（FGSM/PGD）と Robustness、Federated Learning、社会的影響を扱いました。応用でSHAP、Fairlearn、DP-SGD、Constitutional AI。ハンズオンでこれらの実装。シミュレーションで採用AIバイアス、SHAP説明、敵対的攻撃。最後の章でEU AI Act、GDPR、日本AI推進法など法規制を扱います。
