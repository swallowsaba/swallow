§1 / BIASバイアスの種類

選択バイアスデータ収集方法に偏り。例：富裕層のみのアンケート
測定バイアス測定方法に偏り。例：肌の色で精度が違う顔認識
表現バイアスマイノリティのデータが不足。例：欧米中心の医療データ
確証バイアス仮説を支持する情報のみ集める
歴史的バイアス過去の社会的不平等がデータに反映。例：採用履歴の性別偏り
集約バイアス異なる集団を一括処理してしまう
評価バイアステストデータが現実と乖離
有名事例：Amazon の採用 AI（2018年廃止）が女性候補者を不当に低評価。原因：過去10年の採用データ（男性中心）で学習 → 「女子大」「女性チェスクラブ」を負の特徴として学習。これが 歴史的バイアス + フィードバックループ。

§2 / FAIRNESS公平性指標

「公平」の定義は1つではない。複数の指標が同時に成立しない（不可能性定理）。

Demographic Parity（人口統計学的均等）各グループで陽性率が同じ：P(ŷ=1|A=0) = P(ŷ=1|A=1)
Equal Opportunity（機会均等）真陽性率（Recall）が等しい：P(ŷ=1|y=1, A=0) = P(ŷ=1|y=1, A=1)
Equalized OddsTPRとFPRの両方が等しい
Calibration（較正）予測確率が実際の頻度と一致：P(y=1|ŷ=p, A=a) = p（全グループで）
個人公平性類似する個人は類似する予測を受ける
反事実公平性保護属性を変えても予測が変わらない

有名な事例：COMPAS（再犯予測AI）。ProPublica が「黒人に対し False Positive 率が高い」と批判。一方で開発元は「Calibration（較正）は等しい」と反論。両者とも数学的に正しいが、公平性の定義が異なるため対立。

§3 / XAI説明可能性（Explainable AI）

「なぜそう予測したか」を人間に説明できるようにする技術。EU AI Act の高リスク用途で義務化される。

本質的解釈可能モデル線形回帰、決定木、Generalized Additive Models（モデル自体が透明）
事後説明：Permutation Importance特徴量をシャッフルして性能低下を見る
事後説明：LIME個別予測の周辺を線形モデルで局所近似
事後説明：SHAPゲーム理論（Shapley値）で各特徴の貢献度を算出。理論的根拠が強い
事後説明：Integrated Gradientsニューラルネットの勾配を統合
事後説明：Attention Visualization注意の重みをヒートマップ表示
事後説明：Grad-CAMCNN で「画像のどこを見たか」を可視化

§4 / PRIVACYプライバシー保護

差分プライバシー（Differential Privacy）個人データの影響を数学的に限定。ノイズを加える。Apple、Google が採用
連合学習（Federated Learning）データをサーバに送らず、各クライアントで学習し勾配のみ集約
準同型暗号暗号化したまま計算可能。重い
セキュアマルチパーティ計算（SMPC）複数当事者の秘密データを共有せず計算
k-匿名化、l-多様性、t-近接個人特定リスクを下げるデータ加工
機械学習で抑制すべき攻撃メンバーシップ推論攻撃、モデル反転攻撃、属性推論攻撃

§5 / TOXICToxicity と有害性

LLM が生成・あるいは応答できる有害コンテンツのカテゴリ：

  • 差別・ヘイトスピーチ
  • 暴力・自傷・自殺の助長
  • 違法行為の指示（武器、薬物、ハッキング）
  • 性的・搾取的コンテンツ（特に未成年）
  • 誤情報・選挙妨害
  • 個人情報の漏洩

対策：

  • 事前学習データのフィルタリング
  • RLHF / DPO（章7）で安全な応答を学習
  • システムプロンプトでの挙動制約
  • 出力フィルタ（Perspective API など）
  • Constitutional AI（§ 8）

§6 / HALLUハルシネーション

LLM が 事実でないことを自信を持って生成する現象。原因：

  • 言語モデルは「もっともらしさ」を最適化、「正しさ」ではない
  • 知識のカットオフ、訓練データの誤り
  • RLHF が「自信ある回答」を強化する副作用

対策：

  • RAG（章8）で外部知識を参照
  • 引用付き生成（出典明示）
  • Self-Critique（モデル自身に確認させる）
  • Chain-of-Verification
  • 不確実性の伝達（「わかりません」を許容）

評価指標：TruthfulQA、HaluEval、FActScore。

§7 / REDTEAMRed Teaming

セキュリティ用語からの輸入：意図的にAIを攻撃し、脆弱性を見つける。OpenAI、Anthropic、Google など全社で実施。

  • Jailbreak の発見
  • プロンプトインジェクション
  • 有害コンテンツ生成の誘導
  • 偏見・差別の引き出し
  • 個人情報の抽出

§8 / CONSTConstitutional AI（Anthropic）

2022年、Anthropic が提案。AI に「憲法」（自然言語の原則）を与え、AI 自身がそれに従ってフィードバックを生成する。

2ステップ：

  1. SL-CAI：教師あり段階。モデルの応答を憲法に基づきモデル自身が批判・修正
  2. RL-CAI：強化学習段階。AI が好み順をつけ、それで RLHF（RLAIF とも呼ぶ）

利点：人間アノテーションの大幅削減、原則の明文化、スケーラビリティ。Claude のアライメントの中核。

§9 / SUMMARY章のまとめ

1バイアスは多種類：選択・測定・表現・歴史・確証...
2公平性指標は複数あり、同時に成立しない（不可能性定理）
3COMPAS 事件：定義により公平／不公平が分かれる典型例
4XAI：LIME, SHAP, Grad-CAM などで「なぜ」を説明

8RAG・引用・Self-Critique がハルシネーション対策
