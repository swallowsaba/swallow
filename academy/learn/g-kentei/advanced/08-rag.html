応用 章8 — RAGRetrieval-Augmented Generation

「LLM は最新情報を知らない」「社内文書を知らない」「ハルシネーションする」── これらの問題に対する 実務上の決定打 が RAG。検索 → 拡張 → 生成という3段階で、LLM を 外部知識ベース に接続します。

§1 / WHYなぜ RAG が必要か

LLM の根本的な制約：

  • 知識のカットオフ：訓練データの時点までしか知らない
  • 非公開情報：社内文書・個人ファイル等は知らない
  • ハルシネーション：知らないことも自信満々に答える
  • 再学習コスト：知識を更新するには莫大な計算資源

RAG の解決策：「必要な情報をその場で検索して、プロンプトに添えて LLM に渡す」。実装容易・効果絶大で、2023-2025年の LLM 応用の中心。

§2 / EMBED埋め込みベクトル

テキストを 意味を保った高次元ベクトル に変換。意味が近い文は近いベクトルになる。

embed("猫が魚を食べる") ≈ embed("ネコがサカナを食べる")

代表的なモデル：

  • text-embedding-3（OpenAI）
  • BGE（中国 BAAI、オープン強力）
  • E5（Microsoft）
  • multilingual-e5（多言語）
  • Cohere Embed
  • Sentence-BERT（古典）

類似度は コサイン類似度（基本編 章3）：

cos_sim(q, d) = (q · d) / (||q|| · ||d||)

§3 / VECTORDBベクトルデータベース

埋め込みベクトルを格納し、近似最近傍探索（ANN） で高速検索するDB。

Pineconeマネージド SaaS。実務で人気
Weaviateオープンソース、グラフ機能あり
Chroma軽量・開発者向け
Qdrant高性能 Rust 実装、オープン
Milvus大規模分散
FAISSMeta、ライブラリ（DBではない）。HNSW, IVF 等のアルゴリズム提供
pgvectorPostgreSQL の拡張。既存 DB に統合

主な ANN アルゴリズム：HNSW（階層的近傍グラフ）、IVF（転置インデックス）、PQ（直積量子化）。

§4 / FLOWRAG の3段階

FIG 8.1 RAG の3段階。質問 → 検索 → プロンプトに添付 → LLM が文書に基づいて回答。LLM の重みを変えずに、新しい知識・最新情報・社内データを使えるようになる。

§5 / CHUNKチャンク化と前処理

長文書はそのままだとベクトル化できない（モデルの入力長制限）。チャンク（文書の塊） に分割する必要がある。

チャンク戦略：

  • 固定長：512〜1024 文字。シンプルだが文脈分断のリスク
  • セマンティック分割：見出し・段落で区切り、意味の塊で
  • オーバーラップ：隣接チャンクで重複を持たせて文脈保持
  • 親子チャンク：小さい単位で検索、大きい単位で文脈提供

§6 / HYBRIDハイブリッド検索

埋め込み検索（ベクトル）だけでは 固有名詞や数値の精密マッチ に弱い。BM25（キーワード検索） と組み合わせる：

score = α · vector_sim + (1-α) · BM25_score

BM25 は古典的な TF-IDF の改良版。完全一致が必要な状況（製品コード、人名など）に強い。2024-2025年の本番システムではほぼ必須。

§7 / RERANK再ランキング（Re-ranking）

ベクトル検索で top-100 を取り、より高精度な Cross-Encoder で再順位付けして top-10 を選ぶ：

  • Bi-Encoder（埋め込み）：高速だが精度に限界
  • Cross-Encoder（再ランカー）：クエリと文書をペアで入力 → 高精度だが遅い

代表的な再ランカー：Cohere Rerank、BGE-Reranker、ColBERT。

§8 / ADV高度な RAG パターン

HyDEクエリから仮想回答を生成し、それで検索（クエリと文書の語彙ギャップを埋める）
Query Decomposition複雑な質問を複数のサブクエリに分解
Multi-hop RAG1回の検索で足りないとき、複数回検索
Self-RAGLLM が「検索すべきか」「結果が良いか」を判断
GraphRAG知識グラフを併用（Microsoft 2024）
Long-context vs RAG200K+ トークン文脈 vs RAG。後者の方が依然優位な場面が多い

RAG の評価

  • Retrieval 評価：Recall@k、MRR、NDCG
  • Generation 評価：Faithfulness（文書への忠実性）、Answer Relevance
  • RAGAS：RAG 専用の自動評価フレームワーク

§9 / SUMMARY章のまとめ

1RAG = Retrieval（検索）+ Augment（拡張）+ Generate（生成）
2LLM の知識制約・ハルシネーション対策の決定打
3埋め込みベクトルで意味的類似度を計算
4ベクトル DB：Pinecone, Weaviate, Chroma, Qdrant, Milvus, FAISS, pgvector
5ANN アルゴリズム：HNSW、IVF、PQ
6チャンク化が品質を左右
7ハイブリッド検索（ベクトル + BM25）が実務標準
8Re-ranking で精度向上
9HyDE, Self-RAG, GraphRAG などの拡張
10評価は Retrieval と Generation の両軸、RAGAS が代表
