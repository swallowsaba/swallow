応用 章9 — エージェントとツール使用AI Agents

2024-2025年、AI の中心は 「答える」LLM から 「行動する」エージェント へ。ReAct パターン、Function Calling、Computer Use、Multi-Agent System まで、現代エージェントの設計原理を網羅します。

§1 / WHATエージェントとは

狭義のエージェント：環境を観察し、目標を達成するために行動を選択する自律的システム。古典AI でも長く扱われてきた概念。

現代の LLM エージェント：

  • LLM が 「次に何をするか」を考える脳
  • 外部ツール（API、検索、コード実行、ファイル操作）が 手足
  • これらを ループ で回し、複雑なタスクを達成

§2 / REACTReAct パターン（Reason + Act）

Yao et al. (2022) が提案。思考 → 行動 → 観察 → 思考 → ... のループ。

FIG 9.1 ReAct パターン。LLM は次に何を考えるか・何を行動するかを言語で出力し、ツールがそれを実行、結果を LLM に戻す。最も基本的かつ強力なエージェント設計。
例：「東京の今日の最高気温は何度？」
Thought: 気温を知るには検索が必要。weather API を使おう。
Action: weather_api(location="Tokyo", date="today")
Observation: {"max_temp": 28, "unit": "C"}
Thought: 結果が得られた。回答できる。
Final Answer: 東京の今日の最高気温は 28℃ です。

§3 / FCFunction Calling（ツール使用）

2023年6月、OpenAI が GPT-4 で Function Calling を正式機能化。モデルに利用可能な関数のスキーマを渡すと、モデルが JSON 形式で関数呼び出しを出力 する。

関数定義（JSON Schema）：
{
  "name": "get_weather",
  "parameters": { "location": "string", "date": "string" }
}
モデル出力：
{ "function": "get_weather", "args": {"location": "Tokyo", "date": "2026-05-31"} }

主要 API：

  • OpenAI Function Calling / Tools
  • Anthropic Tool Use（Claude）
  • Google Function Calling（Gemini）
  • MCP（Model Context Protocol）：Anthropic が2024年に提唱、ツールサーバの標準プロトコル

§4 / COMPUTERComputer Use（コンピュータ操作）

2024年10月、Anthropic が Computer Use を発表。LLM が マウス・キーボード操作で PC を直接操作 する。スクリーンショットを観察 → 次の操作（クリック、入力、スクロール）を出力 → 実行 → ループ。

競合：

  • OpenAI Operator（2025年）：ブラウザ操作に特化
  • Google Project Mariner：Chrome 拡張
  • Microsoft Copilot Vision：画面理解と操作

§5 / MEMORY長期記憶

LLM のコンテキスト長は有限。会話やタスクが長くなると過去の情報が消える。外部記憶 を持たせる必要がある。

短期記憶現在のコンテキスト内（数万〜数百万トークン）
エピソード記憶過去の会話・タスクの要約をベクトルDBに
知識記憶事実、ユーザー設定、ドキュメント → RAG で参照
手続き記憶使えるツール、過去に成功した手順

実装例：MemGPT、LangChain Memory、Letta（旧 MemGPT）。

§6 / MULTIMulti-Agent System

複数のエージェントが 役割を分担して協力するシステム。

CrewAI役割ベースのエージェントオーケストレーション
AutoGenMicrosoft、エージェント間の会話で問題解決
LangGraph状態機械として記述。LangChain系
MetaGPTソフトウェア開発を「会社」に見立てる（PM, Dev, QA）
OpenAI Swarm2024年公開の軽量フレームワーク

§7 / FW主要フレームワーク

  • LangChain：最も普及、ツール・記憶・チェーンの抽象化
  • LlamaIndex：RAG に強み
  • DSPy：宣言的・自動最適化
  • Haystack：エンタープライズ RAG・QA
  • OpenAI Assistants API：マネージドエージェント基盤
  • Claude Code / Cursor / Devin：コーディング特化エージェント

§8 / RISKリスクと対策

  • プロンプトインジェクション：ツール結果に悪意ある指示が紛れ込み、エージェントを乗っ取る
  • 意図しない実行：「ファイルを削除」のような破壊的操作の誤発火
  • 暴走・無限ループ：終了条件不備
  • 権限過剰：エージェントに与える権限の最小化原則
  • 監査困難：行動ログの記録と人間レビュー

対策：サンドボックス実行、Human-in-the-Loop の確認、Constitutional Classifiers、ツール権限の細分化。
