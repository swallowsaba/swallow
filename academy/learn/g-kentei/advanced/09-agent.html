『質問に答える』LLM から、『目標を達成するために自律的に行動する』LLM へ ── これが AI Agent 革命です。ReAct で 『考えて行動する』、ツール呼び出しで 外部世界と対話、メモリで 長期コンテキストを保持、Self-Reflection で 自己修正。2024 年の Computer Use（Claude）、OpenAI o1、Devin AI ── Agent は AI の次の Frontier です。本章で ReAct、Tool Use、Planning、Memory、Multi-Agent までを5部構成で扱います。

G-KENTEI · ADVANCED · CHAPTER 21

考え、ツールを使い、
世界に働きかける AI。

LLM 単独では『1ターンの応答』を生成するだけ。Agent は LLM を脳として使い、『観察 → 推論 → 行動 → 観察…』のループを回します。検索する、コードを実行する、API を叩く、ファイルを書く、画面を操作する ── ツールを通じて世界と対話する AI へ。

SECTION ONE

基本

Agent定義、ReAct、Tool Use、Planning、Memory、Self-Reflection、Multi-step、LLM Backbone、Function Calling、Agent Loopの10節。

基本

§1.1 AI Agent の定義

『目標が与えられ、その達成のために自律的に推論・計画・行動できるシステム』。LLMを脳として、ツールを手足として、メモリを記憶として、ループで動作。Russell & Norvig の古典的 Agent 定義（PEAS: Performance, Environment, Actuators, Sensors）の LLM 時代版。

§1.2 ReAct パラダイム（2022）

Yao et al. ReAct = Reasoning + Acting。LLM が Thought → Action → Observation のサイクルを繰り返す：

Thought: 何をすべきか考える
Action: ツール呼び出し（例：search("...")）
Observation: ツールの結果を受け取る
Thought: 次の行動を考える
...
最終 Action: 回答を返す
FIG 21.1 ／ ReActループ。LLMが Thought（推論）→ Action（ツール実行）→ Observation（結果）を繰り返し、最終回答に到達。

§1.3 Tool Use — 外部世界へのインタフェース

Agent が呼び出せる関数・API。代表的なツール：

  • 検索：Web search、社内DB検索、RAG
  • 計算：Code Interpreter、数式評価
  • 外部API：天気、株価、SaaS（Slack、GitHub、Notion）
  • ファイル操作：読み書き、要約、変換
  • 制御：ブラウザ自動化、デスクトップ操作（Computer Use）

§1.4 Function Calling — 構造化されたツール呼出

OpenAI 2023 が標準化。ツール仕様（名前、説明、JSON Schema）をモデルに渡し、モデルが構造化された JSON で呼出パラメータを生成。現在 Claude、Gemini、Llama 等すべてサポート。

§1.5 Planning と Decomposition

複雑な目標を サブタスクに分解。手法：

  • Chain-of-Thought：1ターン内での思考連鎖
  • Plan-and-Execute：最初に計画策定 → 実行
  • Tree of Thoughts：分岐探索で最良パス選択
  • Graph of Thoughts：DAGで複雑な依存関係表現

§1.6 Memory — 短期と長期

種類実装用途
短期（Conversation Buffer）直近Nターンをプロンプトに保持会話の流れ
要約メモリ過去をLLMで要約し圧縮長時間セッション
Entity Memory登場人物・キー情報を抽出保管事実トラッキング
長期（Vector Memory）過去会話をベクトル化、関連時取得個人化、学習
Episodic Memory特定のタスク経験を保管・引用少数派手段

§1.7 Self-Reflection と Self-Critique

Reflexion (Shinn 2023)、Self-Refine (Madaan 2023)。Agent が自分の出力を 批評・修正 するメタループ。Code生成タスクで人間レベル超え。OpenAI o1, Claude 3.5 Sonnetの『thinking』 もこの思想。

§1.8 Multi-step Reasoning

数式問題、論理パズル、コーディングなど多段階思考が必要なタスク。CoT、ToT、Program-Aided Language Models（PAL）、Math型 LLM などの手法。

§1.9 LLM Backbone の選択

LLMAgent向けの強み
GPT-4 / GPT-4oFunction Calling 成熟、Code Interpreter
Claude 3.5/3.7 SonnetComputer Use、ツール推論強い
Gemini 1.5/2.0長コンテキスト 2M tokens
OpenAI o1/o3推論専門、複雑タスクで高精度
Llama 3.xオープン、Self-host可
DeepSeek V3/R1オープン、推論強

§1.10 Agent Loop の終了条件

  • 最終回答に到達（Final Answer 出力）
  • 最大反復数到達（典型 10〜50）
  • 同一ツール呼出ループ検出
  • エラー閾値超過
  • 人間承認待ち
SECTION TWO

応用

AutoGPT、LangChain Agents、Claude Computer Use、Multi-Agentの4節。

応用

§2.1 AutoGPT、BabyAGI（2023）

2023年初、GPT-4 を用いた自律 Agent の先駆け。タスクを自動分解・実行・評価のループ。技術的限界（コスト・ループ・幻覚）で運用には課題があったが、Agent ブームの火付け役。

§2.2 LangChain Agents / LangGraph

LangChain：宣言的なAgent構築。LangGraph：状態機械として Agent をモデル化、Cycle・分岐・人間介入を扱える。Production 用途で広く使用。

§2.3 Claude Computer Use（Anthropic 2024）

2024 年 10 月発表。Claude 3.5 Sonnet が 画面のスクリーンショットを見て、マウス・キーボードを操作。表計算記入、ブラウザ作業、フォーム入力など、人間と同じ方法でPC操作。

§2.4 Multi-Agent（AutoGen、CrewAI）

複数の専門 Agent が協調：AutoGen（Microsoft）はカンファレンス形式、CrewAI は役割（Researcher、Writer、Critic）を明示。複雑タスク（コード生成、報告書作成）で単一Agentより高品質。

SECTION THREE

ハンズオン

OpenAI Function Calling、LangChain ReAct、Tools、LangGraph、CrewAIの5節。

ハンズオン

§3.1 OpenAI Function Calling

from openai import OpenAI client = OpenAI() tools = [{ "type": "function", "function": { "name": "get_weather", "description": "指定都市の現在の天気を取得", "parameters": { "type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"] } } }] response = client.chat.completions.create( model="gpt-4o", messages=[{"role": "user", "content": "東京の天気は？"}], tools=tools) tool_call = response.choices[0].message.tool_calls[0] print(tool_call.function.name) # get_weather print(tool_call.function.arguments) # {"city": "東京"}

§3.2 LangChain ReAct Agent

from langchain.agents import create_react_agent, AgentExecutor from langchain_core.tools import tool from langchain_openai import ChatOpenAI @tool def search(query: str) -> str: """Web を検索する""" return f"検索結果: {query}に関する情報..." @tool def calculator(expression: str) -> float: """数式を計算する""" return eval(expression) llm = ChatOpenAI(model="gpt-4o", temperature=0) agent = create_react_agent(llm, [search, calculator], prompt) executor = AgentExecutor(agent=agent, tools=[search, calculator], max_iterations=10) result = executor.invoke({"input": "GPT-4のパラメータ数は？それは1750億の何倍？"})

§3.3 自前 Agent ループ

def simple_agent(query, tools, max_steps=10): messages = [{"role": "user", "content": query}] for step in range(max_steps): response = llm.chat(messages, tools=tools) if response.tool_calls: for tc in response.tool_calls: result = tools[tc.name](**tc.arguments) messages.append({"role": "tool", "name": tc.name, "content": str(result)}) else: return response.content # 最終回答 return "最大反復に到達"

§3.4 LangGraph で状態機械

from langgraph.graph import StateGraph, END from typing import TypedDict class State(TypedDict): query: str plan: list results: list answer: str def plan_node(state): # LLM でタスクを分解 state["plan"] = llm.invoke(f"分解: {state['query']}") return state def execute_node(state): # 各サブタスクを実行 for task in state["plan"]: state["results"].append(execute_task(task)) return state graph = StateGraph(State) graph.add_node("plan", plan_node) graph.add_node("execute", execute_node) graph.add_edge("plan", "execute") graph.add_edge("execute", END) graph.set_entry_point("plan") app = graph.compile()

§3.5 CrewAI で Multi-Agent

from crewai import Agent, Task, Crew researcher = Agent( role="Researcher", goal="最新情報を集める", backstory="AIの最新動向に精通したリサーチャー", tools=[search_tool]) writer = Agent( role="Writer", goal="洗練された記事を書く", backstory="技術ライター") task1 = Task(description="2025年のLLM動向調査", agent=researcher) task2 = Task(description="調査結果から1500字記事執筆", agent=writer, context=[task1]) crew = Crew(agents=[researcher, writer], tasks=[task1, task2]) result = crew.kickoff()

SIM 1 — ReAct vs 単純LLM

>> ツール（検索）アクセスで大幅向上
>> 知識カットオフ問題も解決

SIM 2 — Self-Reflection

>> HumanEval（コード生成ベンチマーク）

[GPT-4 1回生成]　　 pass@1 = 67%
[+ Self-Critique]　 pass@1 = 79%

[Claude 3.5 + 思考] pass@1 = 92%

>> 反復的自己批評で大幅改善

SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
AI Agent の定義として最も適切なのは。
① 単発の質問応答　 ② 目標達成のため自律的に推論・計画・行動するシステム　 ③ Vector DBのみ　 ④ ファインチューニング
ANSWER ② LLMを脳、ツールを手足、メモリを記憶とする自律システム。1ターン応答との根本的違い。
PROBLEM 2
ReAct パラダイムの構成は。
① Search→Generate　 ② Thought→Action→Observation のループ　 ③ Train→Test　 ④ Encode→Decode
ANSWER ② Yao 2022 ReAct論文。LLM が推論→ツール実行→結果観察を繰り返す。Agent の基本パターン。
PROBLEM 3
Function Calling の役割は。
① 学習　 ② LLMが構造化された JSON でツール呼出パラメータを生成　 ③ 正則化　 ④ クラスタリング
ANSWER ② OpenAI 2023標準化、GPT/Claude/Gemini/Llama すべて対応。ツール仕様（JSON Schema）に従いLLMがパラメータ抽出。
PROBLEM 4
Claude Computer Use（Anthropic 2024）の特徴は。
① テキスト生成のみ　 ② スクリーンショットを見てマウス・キーボードを操作　 ③ Vector検索のみ　 ④ 音声処理
ANSWER ② 2024年10月発表。Claude 3.5 Sonnet が画面を理解してPC操作。人間と同じ方法での自動化を可能に。
PROBLEM 5
Self-Reflection / Self-Critique の効果は。
① 計算高速化　 ② Agent が自分の出力を批評・修正することで品質向上　 ③ パラメータ削減　 ④ ランダム化
ANSWER ② Reflexion (2023), Self-Refine (2023)。HumanEvalでpass@1が67%→91%に。o1 の思考時間長期化もこの思想。
PROBLEM 6
Multi-Agent フレームワークの代表は。
① TensorFlow　 ② AutoGen (Microsoft)、CrewAI　 ③ scikit-learn　 ④ NumPy
ANSWER ② AutoGen（カンファレンス形式）、CrewAI（役割明示）、ChatDev（ソフトウェア会社シミュレーション）など。協調で複雑タスクの品質向上。
PROBLEM 7
Agent の短期メモリの典型実装は。
① Vector DB のみ　 ② 直近Nターンを会話履歴としてプロンプトに保持　 ③ ファイル　 ④ なし
ANSWER ② Conversation Buffer Memory。長くなると要約・圧縮、または Vector Memory に長期化。
PROBLEM 8
Plan-and-Execute Agent の特徴は。
① 1ステップで全実行　 ② 最初に計画策定 → サブタスクを順次実行　 ③ ランダム　 ④ 学習
ANSWER ② Chain-of-Thought の発展形。複雑タスクで失敗を減らせる。BabyAGI、AutoGPT の基礎パターン。
PROBLEM 9
LangGraph が LangChain Agent と異なる点は。
① 同じ　 ② 状態機械として明示的にモデル化、Cycle・分岐・人間介入を扱える　 ③ Vector DB専用　 ④ 学習特化
ANSWER ② 暗黙的ループではなく明示的グラフで Agent を制御。Production-grade Agent の標準。
PROBLEM 10
Agent の典型的な失敗パターンは。
① 無限ループ、過剰なツール呼出、コスト爆発　 ② 学習速度低下　 ③ メモリリーク　 ④ なし
ANSWER ① max_iterations 設定、ループ検出、コスト上限、人間承認などのガードレールが本番必須。
PROBLEM 11
AutoGPT (2023) の意義は。
① 学習アルゴリズム　 ② 自律 LLM Agent の先駆け、技術的限界はあったがAgent ブームの火付け役　 ③ Vector DB　 ④ 単なる検索
ANSWER ② 2023年初、GPT-4 ベース。タスク自動分解実行のループ。実用性は限定的だったが、Agent概念を一般に広めた。
PROBLEM 12
『Tool Use』の代表的ツールカテゴリは。
① 学習　 ② 検索、計算、API、ファイル操作、ブラウザ・PC制御　 ③ 単一機能のみ　 ④ Vector のみ
ANSWER ② 多様なツールが LLM の能力を拡張。Web search、Code Interpreter、SaaS API、Computer Use など。
PROBLEM 13
Tree of Thoughts（ToT）の発展点は。
① 単線思考　 ② 思考を木構造で分岐探索し最良経路選択　 ③ Vector検索　 ④ 教師あり
ANSWER ② Yao 2023。CoTを枝分かれにし、BFS/DFS的に探索。数独、24パズル等で高性能。
PROBLEM 14
Reflexion（2023）の動作は。
① 教師あり　 ② Agent が失敗から自己批評＆メモリ蓄積→次回反映　 ③ Vector DB のみ　 ④ クラスタ
ANSWER ② Shinn 2023。失敗の言語的反省を Memory に蓄積、次の試行で参照。コード生成でpass@1 を大幅向上。
PROBLEM 15
OpenAI o1 / o3 が従来 LLM と異なる点は。
① パラメータが100倍　 ② 推論時に長い『思考』を行い、Self-Reflection内蔵　 ③ Vector検索専用　 ④ 多言語のみ
ANSWER ② Test-time compute scaling。数十秒〜数分の思考連鎖で複雑タスク（数学、コード）で人間レベル超え。
PROBLEM 16
Agent のメモリで Vector Memory が解決する問題は。
① 計算速度　 ② 長期記憶（過去会話・経験から関連情報を検索可能に）　 ③ 学習　 ④ パラメータ
ANSWER ② 会話履歴をベクトル化し、関連時に取得。個人化、長期セッション、経験学習に必須。
PROBLEM 17
Program-Aided Language Models (PAL) の特徴は。
① 計算　 ② LLM が Python コードを生成→実行→結果を使う（数値計算誤りを防ぐ）　 ③ Vector検索　 ④ 教師あり
ANSWER ② Gao 2022。LLM の苦手な厳密計算を Python に委任。数学・データ処理タスクで高精度。
PROBLEM 18
2024年に登場した Devin AI の特徴は。
① 単純チャット　 ② 自律的にコーディングタスクを完遂、ブラウザ・エディタ操作可能　 ③ Vector検索のみ　 ④ 画像生成
ANSWER ② Cognition Labs。コード生成、デバッグ、デプロイまで一貫実行のAI Software Engineer。SWE-Bench で当時 SOTA。
PROBLEM 19
Agent の安全性で重要なのは。
① 全自動が常に良い　 ② Human-in-the-Loop、ツール権限制限、コスト上限、監査ログ　 ③ なし　 ④ 速度
ANSWER ② 自律実行のリスク：誤ったAPI呼出、機密漏洩、コスト爆発。人間承認、サンドボックス、レート制限が必須。
PROBLEM 20
Agent と LLM 単独 RAG の本質的違いは。
① 同じ　 ② Agentは多段階推論＋ツール選択＋ループ、RAGは1ターン取得+生成　 ③ Vector不使用　 ④ 学習
ANSWER ② RAG は静的なパイプライン（検索→生成）、Agent は動的な意思決定（どのツールを何回呼ぶか LLM が判断）。Agent内でRAGをツールとして使える。
