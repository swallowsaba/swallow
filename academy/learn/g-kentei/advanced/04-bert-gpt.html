G検定 応用編 — 章28

事前学習＋微調整。
NLPの新パラダイム。

2018年6月、OpenAI が GPT-1（Improving Language Understanding by Generative Pre-Training）を発表。同年10月、Google が BERT（Bidirectional Encoder Representations from Transformers）を発表。『教師なし事前学習で言語を学び、教師ありで個別タスクに適応』というシンプルなアイデアが、NLPベンチマークを次々塗り替えました。本章でその核心を学びます。

SECTION ONE

基本

事前学習パラダイム、BERT、GPT、Tokenization、Position Encoding、T5、RoBERTa/DeBERTa/ELECTRA、微調整 vs プロンプティングの10節。

基本

§1.1 事前学習＋微調整パラダイム

(1)大規模ラベルなしコーパスで 自己教師あり学習（MLM、NTP）、(2)タスク特化の少量データで 微調整。これにより：

  • 各タスクで一から学習する必要がない
  • 言語の汎用知識を活用
  • 少量データでも高精度（Few-shot 含む）
  • 計算リソースの集中投資が可能（事前学習1回 → 多くのタスクに）

§1.2 BERT — Masked Language Model

BERT　 Bidirectional Encoder Representations from Transformers
事前学習タスク
　① MLM（Masked LM）　 入力の15%を [MASK] に置換し、元のトークンを予測
　② NSP（Next Sentence Prediction）　 2文が連続かを予測（後の RoBERTa で削除）

アーキテクチャ　 Transformer エンコーダのみ（12層 base / 24層 large）
双方向性　 各トークンは左右両方の文脈を見る → 文章理解タスクに強い

§1.3 GPT — Causal Language Model

GPT　 Generative Pre-trained Transformer
事前学習タスク　 Next Token Prediction（次のトークンを予測）

アーキテクチャ　 Transformer デコーダのみ
因果Attention（Causal Mask）　 各トークンは『過去』のみ参照
強み　 自然な文章生成、In-Context Learning
FIG 16.1 ／ BERT vs GPT。BERTはエンコーダ、MLMで双方向文脈を学ぶ。GPTはデコーダ、NTPで左から右の生成を学ぶ。タスク特性が異なる。

§1.4 Tokenization — テキストを単位に分割

手法採用特徴
WordPieceBERT頻度ベース、サブワード
BPE（Byte-Pair Encoding）GPT-2, RoBERTa頻度ペアを反復マージ
SentencePieceT5, ALBERT, mBART言語非依存、生のテキストから直接
TiktokenGPT-3.5/4/4oBPE 改良、約 5万〜10万語彙

§1.5 Position Encoding

Transformer 自体は順序を扱わない（permutation invariant）。位置情報を追加する手法：

  • 正弦波（Sinusoidal）：原論文 Transformer、固定パターン
  • 学習可能位置埋め込み：BERT、GPT-2、最大長制限
  • RoPE（Rotary Position Embedding）：LLaMA、相対位置を回転で表現、長さ外挿可能
  • ALiBi（Attention with Linear Biases）：BLOOM、超長文脈対応

§1.6 BERT のバリエーション

モデル変更点性能
BERT-base12層・768次元・110Mパラメータ標準
BERT-large24層・1024次元・340Mパラメータ+2〜3%
RoBERTaNSP削除・データ10倍・長く学習大幅改善
ALBERTパラメータ共有でメモリ削減速い
DeBERTa分離型Attention（disentangled）SuperGLUE 1位
ELECTRAReplaced Token Detection（識別タスク）学習効率高

§1.7 GPT のスケーリング

モデル年パラメータ特徴
GPT-12018117M事前学習＋微調整の概念実証
GPT-220191.5BZero-shot で文章生成、危険性懸念で段階公開
GPT-32020175BIn-Context Learning、Few-shot

GPT-42023非公開（推定1T+）マルチモーダル、推論力大幅向上
GPT-4o2024非公開音声・画像・動画ネイティブ統合
o1 / o32024-25非公開テスト時推論で数学・コーディング SOTA

§1.8 T5 — Text-to-Text 統一

Google 2020。すべてのNLPタスクを Text-to-Text として統一：分類は『positive』『negative』を文字列で出力、QAは答えを文字列で。プレフィックスでタスク指定（『translate English to French:』）。マルチタスク事前学習。

§1.9 微調整 vs プロンプティング

微調整プロンプティング
方法追加学習入力テキストで指示
必要データ数百〜数万件0〜数十件（few-shot）
コストGPU時間、専門知識API呼び出しのみ
精度タスク特化で高汎用、In-Context で学習
適性固定タスク・高精度要多様なタスク・素早く試す

§1.10 LoRA — 微調整の効率化

Hu et al. 2021。事前学習済みモデルのパラメータを凍結し、低ランクの差分行列だけ学習。パラメータ数を1万分の1に削減しつつ性能維持。LLM の実用的微調整で必須技術。QLoRA は4-bit量子化＋LoRAでさらに効率化。

SECTION TWO

応用

文章分類、QA、NER、文章生成の4つの代表的応用。

応用

§2.1 文章分類 — BERT の最頻用ケース

[CLS] トークンの出力埋め込みに分類層を追加。感情分析（IMDB、Twitter）、意図分類、トピック分類、毒性検出など、エンタープライズの NLP の主役。

§2.2 質問応答（QA） — SQuAD ベンチマーク

パラグラフ＋質問 → 答えの開始・終了位置を予測。BERT が SQuAD で人間超え（2018）。現代では RAG（章19）で外部知識と組み合わせる。

§2.3 固有表現認識（NER）

各トークンに 'PERSON', 'LOC', 'ORG' などのタグを付与。BERTトークン分類で標準。医療文書から薬剤名抽出、契約書から日付・金額抽出など実用例多数。

§2.4 文章生成 — GPT の特長

記事執筆、メール、コード、対話、要約、翻訳。サンプリング戦略（Temperature、Top-k、Top-p/Nucleus、Beam Search）が品質を左右。Hallucination 抑制が現代の課題。

SECTION THREE

ハンズオン

Hugging Face、Tokenization、感情分類微調整、GPT生成、Embeddingの5節。

ハンズオン

§3.1 Hugging Face Transformers の基本

from transformers import pipeline # 感情分析 classifier = pipeline("sentiment-analysis") result = classifier("I love this product!") print(result) # [{'label': 'POSITIVE', 'score': 0.9998}] # 質問応答 qa = pipeline("question-answering") ctx = "Claude is an AI assistant created by Anthropic." print(qa(question="Who created Claude?", context=ctx)) # {'answer': 'Anthropic', 'score': 0.98} # NER ner = pipeline("ner", aggregation_strategy="simple") print(ner("Tokyo is in Japan.")) # [{'entity_group': 'LOC', 'word': 'Tokyo'}, ...]

§3.2 Tokenization 観察

from transformers import AutoTokenizer # BERT WordPiece tok = AutoTokenizer.from_pretrained("bert-base-uncased") text = "unhappiness is generally undesirable" print(tok.tokenize(text)) # ['un', '##happiness', 'is', 'generally', 'un', '##desirable'] # GPT-2 BPE tok2 = AutoTokenizer.from_pretrained("gpt2") print(tok2.tokenize(text)) # ['un', 'happiness', ' is', ' generally', ' und', 'esirable'] print("BERT 語彙数:", tok.vocab_size) # 30522 print("GPT-2 語彙数:", tok2.vocab_size) # 50257

§3.3 感情分析の微調整

from transformers import AutoModelForSequenceClassification, Trainer, TrainingArguments from datasets import load_dataset model = AutoModelForSequenceClassification.from_pretrained( "bert-base-uncased", num_labels=2) tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased") dataset = load_dataset("imdb") def tokenize(b): return tokenizer(b["text"], truncation=True, padding="max_length") tokenized = dataset.map(tokenize, batched=True) args = TrainingArguments(output_dir="./out", num_train_epochs=3, per_device_train_batch_size=16, learning_rate=2e-5) trainer = Trainer(model=model, args=args, train_dataset=tokenized["train"], eval_dataset=tokenized["test"]) trainer.train() # 3エポックで test accuracy 94%+

§3.4 GPT による文章生成

from transformers import AutoTokenizer, AutoModelForCausalLM model = AutoModelForCausalLM.from_pretrained("gpt2") tok = AutoTokenizer.from_pretrained("gpt2") input_ids = tok("Once upon a time", return_tensors="pt").input_ids outputs = model.generate( input_ids, max_length=100, temperature=0.8, # 創造性 top_p=0.9, # Nucleus サンプリング do_sample=True, no_repeat_ngram_size=2 # 繰り返し抑制 ) print(tok.decode(outputs[0], skip_special_tokens=True))

§3.5 BERT で文埋め込み抽出

import torch from transformers import AutoModel model = AutoModel.from_pretrained("bert-base-uncased") tok = AutoTokenizer.from_pretrained("bert-base-uncased") sents = ["I love cats.", "Dogs are great.", "AI is fascinating."] encoded = tok(sents, padding=True, truncation=True, return_tensors="pt") with torch.no_grad(): out = model(**encoded) # [CLS] トークン埋め込み（典型） cls_emb = out.last_hidden_state[:, 0, :] # または mean pooling（より良いことが多い） mean_emb = out.last_hidden_state.mean(dim=1) print("形状:", cls_emb.shape) # [3, 768] # コサイン類似度 sim = torch.cosine_similarity(mean_emb[0], mean_emb[1], dim=0) print("cat-dog 類似度:", sim.item())
SECTION FOUR

シミュレーション

BERT vs GPT、Tokenization の効果、スケーリングの3節。

シミュレーション

SIM 1 — BERT vs GPT、得意分野

>> 同じタスクで両方の事前学習モデルを微調整

タスク：感情分析（文章理解）
　 BERT-base: 92.4%　 ← 双方向理解で強い
　 GPT-2:　　 88.7%

タスク：要約（生成）
　 BERT2BERT: ROUGE 38.5
　 GPT-2:　　 ROUGE 42.1　 ← 生成タスクで強い

タスク：QA（SQuAD）
　 BERT-large: F1 93.2
　 GPT-3 zero-shot: F1 71.2　 ← 微調整なしでも動く

>> BERT=理解（分類、QA、NER）
>> GPT=生成（翻訳、要約、対話）
>> 現代はLLMが両方こなす

SIM 2 — Tokenization の効果

>> 多言語データでの効率

[英語のみ訓練の語彙]
　 "Hello world"　　 → [Hello, world]　 2トークン
　 "こんにちは世界" → [bytes×16]　　　 16トークン

[mBERT 多言語語彙]
　 "Hello world"　　 → [Hello, world]　 2トークン
　 "こんにちは世界" → [こん, ##に, ##ち, ##は, 世, ##界]　 6トークン

>> 適切なトークナイザは多言語性能を大幅改善
>> GPT-4 の Tiktoken は多言語に強い改良版
>> 漢字1文字を1トークンで処理可能

SIM 3 — スケーリング則の威力

>> GLUE平均スコア（言語理解ベンチマーク）

BERT-base (110M):　 79.6
BERT-large (340M):　 82.1　 → +2.5
RoBERTa-base (110M):　85.8　 → 学習方法改善
RoBERTa-large(355M): 88.5
DeBERTa-v3-large (700M): 91.5
GPT-3 few-shot (175B): 90+
GPT-4 (~1T+):　　　　 ~95+

>> パラメータ×データ×計算でスムーズに改善
>> これが『スケーリング則』として体系化（Kaplan 2020）
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
BERT の事前学習タスクで主要なものは。
① 次トークン予測　 ② Masked Language Model（MLM）と Next Sentence Prediction　 ③ クラスタリング　 ④ 強化学習
解答・解説を見る
ANSWER ② 入力の15%を [MASK] に置換し復元するMLMが核。NSPは後の改良（RoBERTa）で削除。
PROBLEM 2
GPT の事前学習タスクは。
① MLM　 ② Next Token Prediction（因果的言語モデル）　 ③ 分類　 ④ 強化学習
解答・解説を見る
ANSWER ② 左から右に次のトークンを予測する自己回帰モデル。生成タスクに自然に適合。
PROBLEM 3
BERT と GPT のアーキテクチャの違いは。
① BERTはエンコーダ（双方向Attention）、GPTはデコーダ（因果Attention）　 ② 全く同じ　 ③ BERTはRNN　 ④ GPTはCNN
解答・解説を見る
ANSWER ① どちらもTransformerベースだが、Attentionの方向と用途が異なる。理解はBERT、生成はGPTが得意。
PROBLEM 4
『事前学習＋微調整』パラダイムの利点は。
① タスクごとに学習し直す必要なし、汎用言語知識を活用、少量データで高精度　 ② 計算量が増える　 ③ 精度が下がる　 ④ 言語別必要
解答・解説を見る
ANSWER ① 大規模事前学習1回 → 多タスクに転移。NLPを変革したパラダイム。
PROBLEM 5
BERT で使われる Tokenization は。
① 単語分割のみ　 ② WordPiece（サブワード分割）　 ③ 文字単位　 ④ ランダム
解答・解説を見る
ANSWER ② WordPiece は未知語に強く、語彙サイズを抑える。'unhappiness' → 'un', '##happiness' のような分割。
PROBLEM 6
『RoPE（Rotary Position Embedding）』を採用しているLLMは。
① BERT　 ② GPT-2　 ③ LLaMA, GPT-NeoX 等の現代LLM　 ④ T5
解答・解説を見る
ANSWER ③ Su 2021。相対位置を回転で表現、長さ外挿に有利。LLaMA 系の標準。
PROBLEM 7
T5 が提唱した paradigm は。
① RNN復活　 ② すべてのタスクを Text-to-Text として統一　 ③ 線形化　 ④ ハードコーディング
解答・解説を見る
ANSWER ② 分類も生成も翻訳もすべて『テキスト入力→テキスト出力』。プレフィックスでタスク指定。
PROBLEM 8
RoBERTa が BERT を改良した主要点は。
① NSP削除、データ量と学習時間を大幅増、動的マスキング　 ② アーキテクチャ変更　 ③ パラメータ削減　 ④ Sigmoid化
解答・解説を見る
ANSWER ① Liu et al. 2019。アーキテクチャは同じだが学習レシピで GLUE 大幅改善。『正しく訓練』の重要性を示した。
PROBLEM 9
ELECTRA の事前学習タスクは。
① MLM　 ② NTP　 ③ Replaced Token Detection（置換されたトークンを識別）　 ④ Clustering
解答・解説を見る
ANSWER ③ 生成器が置換したトークンを判別器が識別。MLMより学習効率良く、同パラメータで高性能。
PROBLEM 10
GPT-3 で注目された『In-Context Learning』とは。
① パラメータ更新　 ② プロンプト内の数件の例から、新タスクを推論時に学習　 ③ 微調整　 ④ クラスタリング
解答・解説を見る
ANSWER ② Few-shot Learning の革命。重みは変更せず、コンテキスト内の例だけでタスク適応。Brown 2020。
PROBLEM 11
『微調整』と『プロンプティング』の違いは。
① 同じ　 ② 微調整=モデル重み更新、プロンプティング=入力テキストで指示（重み不変）　 ③ どちらも教師なし　 ④ 必ず両方必要
解答・解説を見る
ANSWER ② 微調整はGPU・データ必要、プロンプティングはAPI呼び出しのみ。実用ではプロンプトエンジニアリングが先、必要に応じて微調整。
PROBLEM 12
LoRA（Low-Rank Adaptation）の役割は。
① 学習率調整　 ② 元モデルを凍結し、低ランク差分行列だけ学習し効率的微調整　 ③ 量子化　 ④ クラスタ
解答・解説を見る
ANSWER ② Hu 2021。学習可能パラメータを1/10000に削減しつつ性能維持。LLM実用微調整の標準。QLoRAで4-bit量子化と組合せ。
PROBLEM 13
BERT の文埋め込み取得で[CLS]以外に良いとされる方法は。
① ランダム　 ② 全トークンのmean pooling　 ③ 1番目のトークンのみ　 ④ 不可能
解答・解説を見る
ANSWER ② [CLS] よりも全トークンの平均が下流タスクで強いことが多い。Sentence-BERT で標準化。
PROBLEM 14
GPT の生成サンプリングで『創造性と一貫性のバランス』を制御する典型パラメータは。
① 学習率　 ② Temperature、Top-p（Nucleus）、Top-k　 ③ バッチサイズ　 ④ Dropout
解答・解説を見る
ANSWER ② Temperature 低→決定的、高→多様。Top-p=0.9 で上位累積90%から選択、Hallucination抑制と多様性のバランス。
PROBLEM 15
GPT-3 の InstructGPT/ChatGPT で導入された手法は。
① CNN　 ② RLHF（Reinforcement Learning from Human Feedback）で指示追従性を学習　 ③ K-means　 ④ Random Forest
解答・解説を見る
ANSWER ② 人間のフィードバックで報酬モデルを学習し、PPO で方策を最適化。指示追従性と無害化の核心技術。
PROBLEM 16
BERT が最も得意なタスクは。
① 長文生成　 ② 文章理解（分類、QA、NER）　 ③ 音楽生成　 ④ 画像認識
解答・解説を見る
ANSWER ② 双方向 Attention で文脈理解が強い。SQuAD で人間超え、GLUEベンチマークを大幅刷新（2018）。
PROBLEM 17
『スケーリング則』（Kaplan 2020）の主張は。
① パラメータ数だけが効く　 ② パラメータ・データ・計算の3軸でモデル性能が予測可能に改善　 ③ ランダム　 ④ 関係なし
解答・解説を見る
ANSWER ② 損失がパワーロー則で改善。これがGPT-3, GPT-4のスケール拡大を駆動。Chinchilla（DeepMind 2022）が最適配分を改良。
PROBLEM 18
SentencePiece が WordPiece より優れる点は。
① 必ず速い　 ② 言語非依存、生のテキストから直接学習、空白を特殊文字として扱う　 ③ 英語専用　 ④ なし
解答・解説を見る
ANSWER ② 日本語・中国語のような空白区切りでない言語に強い。T5, mBART, ALBERTで採用。
PROBLEM 19
『MLM（Masked LM）』が双方向文脈学習を可能にする仕組みは。
① ランダム　 ② マスクされたトークンを予測するため、両側の文脈を見る必要がある（因果制約なし）　 ③ 線形　 ④ 距離計算
解答・解説を見る
ANSWER ② 『私は[MASK]を見た』の[MASK]を予測するには前後両方が必要。これがGPTの因果Attentionとの本質的違い。
PROBLEM 20
日本語特化の事前学習モデルとして代表的なものは。
① 全く存在しない　 ② 東北大BERT、cl-tohoku/bert-base-japanese、rinna/japanese-gpt等　 ③ GPT-1のみ　 ④ ELMo
解答・解説を見る
ANSWER ② 多言語BERTより日本語特化モデルが精度高いことが多い。最近はLLM（Swallow、Stockmark、ELYZA等）も日本語特化版が登場。
