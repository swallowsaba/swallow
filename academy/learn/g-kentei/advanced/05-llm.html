応用 章5 — LLM とスケーリング則LLMs and Scaling Laws

「大きくすれば賢くなる」が現代AIの一つの帰結。本章ではスケーリング則の数式・Chinchilla の最適配分・創発能力（Emergence）・In-context Learning・Chain-of-Thought の主要トピックを押さえます。

§1 / SCALINGスケーリング則（Kaplan et al., 2020）

Kaplan らが OpenAI で発見：LLM の損失は、パラメータ数 N、データ量 D、計算量 C それぞれに対し、冪乗則で減少する。

L(N) ≈ (N_c / N)^α_N
L(D) ≈ (D_c / D)^α_D
L(C) ≈ (C_c / C)^α_C
FIG 5.1 スケーリング則：両対数プロットで直線。これが「大きくすれば賢くなる」の数学的根拠。10倍にすれば一定割合だけ性能が上がる。

§2 / CHINCHILLAChinchilla の発見（2022）

2022年、DeepMind の Chinchilla 論文が 「パラメータ数とデータ量を同じ速度で増やすべき」 という発見を提示。それまでの GPT-3 (175B) は パラメータ過剰・データ不足 だった。

新しい指針：

  • 計算予算が固定 → N ≈ D（パラメータ数とトークン数を比例）
  • Chinchilla（70B params, 1.4T トークン）が GPT-3 を上回った
  • これ以降、データ量を意識した学習設計が標準に

§3 / EMERGENCE創発能力（Emergent Abilities）

Wei et al. (2022) が報告：ある種の能力（多段推論、コード生成、ジョークの理解など）は、モデルサイズが一定閾値を超えると突然出現する。

FIG 5.2 創発能力。一定サイズまで「ランダム水準」だった性能が、閾値を超えると急上昇する。これが「LLM は AGI に近づいた」議論の背景。

創発される能力の例：

  • 多段算術（複数桁の掛け算）
  • 多言語推論
  • Chain-of-Thought 推論（次節）
  • コード生成
  • ジョーク・皮肉の理解

※ 創発の存在自体に疑義を呈する研究もある（Schaeffer et al. 2023, 「測り方の問題」）。

§4 / ICLIn-context Learning（再訪）

章4 で言及したとおり、GPT-3 はプロンプトに例を含めるだけでタスクを学習する。これは weight 更新なし の「学習」。研究上の謎の一つ。

近年の理解（仮説）：

  • Self-Attention が「メタ学習」のように動作している
  • Transformer 内部で勾配降下に似た計算が実行されている
  • 事前学習データに同様のパターンが存在し、それを「想起」している

§5 / COTChain-of-Thought（思考の連鎖）

Wei et al. (2022) が提案。「ステップバイステップで考えて」 とプロンプトに添えるだけで、複雑な推論問題の正答率が劇的に上がる。

CoT の効果（数学問題）
普通のプロンプト：「リンゴが12個、4人で分けると1人何個？ 答え: ?」
→ モデル：「3個」（正解だが、もっと難しい問題では誤答が増える）

CoT プロンプト：「ステップバイステップで考えて。リンゴが12個、4人で分けると1人何個？」
→ モデル：「12個を4人で割る。12 ÷ 4 = 3。答え: 3個」
→ より複雑な問題でも正答率が大幅上昇

派生：

  • Zero-shot CoT：「Let's think step by step」だけ追加
  • Self-Consistency：複数の推論経路を生成し多数決
  • Tree of Thoughts：推論を木構造で探索
  • OpenAI o1, o3 / DeepSeek R1：CoT を学習で内在化した「推論モデル」

§6 / TUNING微調整の手法

Full Fine-tuning全パラメータを更新。最高性能だが計算コスト高
LoRA低ランク行列 ΔW = AB だけ学習。元の重みは凍結。主流
QLoRALoRA + 4bit 量子化。GPU メモリ大幅削減
Prefix Tuning各層に学習可能なプロンプトを追加
Prompt Tuning入力にだけ学習可能なトークンを追加
Adapter各層に小さな MLP を挿入
Instruction Tuning「指示 → 応答」形式の教師あり学習
RLHF人間フィードバックで強化学習（章7で詳述）
