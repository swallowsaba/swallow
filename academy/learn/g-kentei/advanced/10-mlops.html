応用 章10 — MLOps と社会実装MLOps

「モデルを Jupyter で動かす」と「24時間稼働する本番システム」は別世界。MLOps はその間を埋める 運用工学。データ・モデルバージョニング、モニタリング、ドリフト検出、CI/CD、デプロイ戦略を体系的に押さえます。

§1 / WHYMLOps とは

「機械学習モデルを本番環境で継続的に動かす」ための工学的アプローチ。DevOps の概念を機械学習に拡張。

古典的ソフトウェアとの違い：

  • コード+データ+モデルの3つを管理（DevOpsはコードのみ）
  • 挙動がデータに依存 → データの変化で性能が劣化（ドリフト）
  • 再現性が難しい（乱数・GPU・データ更新）
  • 本番デプロイ後も継続的に 再学習 が必要

Gartner の指摘：「機械学習プロジェクトの 80% は本番化に失敗」。MLOps はこの失敗を減らす実践。

§2 / CYCLEML ライフサイクル

FIG 10.1 MLライフサイクル。データ→前処理→学習→評価→デプロイ→監視→再学習という継続的なループ。各段階で MLOps の異なる工夫が必要。

§3 / VERSIONデータ・モデルのバージョン管理

コードGit。当然の標準
データDVC（Data Version Control）、LakeFS、Delta Lake。Git と連携し、大容量データのバージョン管理
モデルMLflow Model Registry、Weights & Biases Artifacts、SageMaker Model Registry
実験MLflow Tracking、W&B、Neptune、Comet。ハイパラ・メトリクス・成果物を記録
特徴量Feature Store：Feast、Tecton、SageMaker Feature Store。訓練と推論で同じ特徴量計算を保証

§4 / CICDCI/CD for ML

従来の CI/CD（コードのテスト・自動デプロイ）に加えて：

  • CT（Continuous Training）：データ更新時に自動再学習
  • CV（Continuous Validation）：データ品質・モデル性能の自動テスト
  • CM（Continuous Monitoring）：本番モデルの継続監視

ツール：GitHub Actions、GitLab CI、Jenkins、Argo Workflows、Kubeflow Pipelines、Vertex AI Pipelines、SageMaker Pipelines。

§5 / DEPLOYデプロイ戦略

Blue-Green新旧2環境を並行稼働、スイッチで切替。ロールバック容易
Canary新モデルに少数のトラフィックを徐々に流す
Shadow本番トラフィックを新モデルにも流すが、応答は使わない（ログ比較のみ）
A/B テスト2モデルを統計的に比較。ビジネスKPI で判定
Multi-armed Bandit探索と活用のバランスで最適モデルに自動切替

推論基盤：TorchServe、TensorFlow Serving、Triton Inference Server（NVIDIA）、KServe（Kubernetes）、Ray Serve。LLM 推論は vLLM、TGI、SGLang。

§6 / MONITORモニタリング

機械学習特有の監視項目：

  • 性能メトリクス：accuracy、F1、AUC（正解ラベルがある場合）
  • プロキシメトリクス：CTR、コンバージョン率、ユーザ反応
  • レイテンシ・スループット：応答時間、QPS
  • データ品質：欠損、外れ値、スキーマ違反
  • 予測分布：予測確率のヒストグラム

ツール：Prometheus + Grafana（汎用）、Evidently AI（ML特化）、WhyLabs、Arize、Fiddler。

§7 / DRIFTドリフト検出

機械学習特有の劣化要因。3種類：

Data Drift（共変量シフト）入力 X の分布が変わる。例：ユーザ層の変化
Concept DriftX と y の関係が変わる。例：COVID 前後の消費行動
Label Drifty の分布が変わる。例：流行病の発生率上昇

検出方法：

  • 統計的検定：KS テスト、χ²、PSI（Population Stability Index）
  • 分布距離：KL ダイバージェンス、JS、Wasserstein
  • 埋め込みベース：再学習データと運用データの埋め込み分布比較

章末問題

  1. MLOps が DevOps と異なる3つの点
  2. データ・モデル・実験のバージョン管理にそれぞれ使われるツール
  3. Feature Store が解決する問題
  4. Blue-Green と Canary の違い
  5. Shadow Deployment の用途
  6. A/B テストで気をつけるべき統計的観点
  7. Data Drift と Concept Drift の違い
  8. PSI（Population Stability Index）が監視する対象
  9. LLM の推論サービングで使われる代表ツール（vLLM, TGI）
  10. 「ML プロジェクトの 80% が本番化失敗」と言われる主な原因
