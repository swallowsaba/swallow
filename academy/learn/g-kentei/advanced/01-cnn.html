応用 章1 — CNNConvolutional Neural Networks

画像認識の長年の主役。「局所性」「平行移動不変性」「重み共有」 という3つの帰納バイアスを利用し、MLP では破綻する高解像度画像を扱えるように設計された深層NN。

§1 / WHYなぜ MLP では駄目か

224×224×3 の画像を MLP に入れると、入力次元は 150,528。1層目で 1000 ユニットなら重みが 1.5億。これを学習するには：

  • 巨大なパラメータ数 → 過学習
  • 位置依存：左上のリンゴと右下のリンゴを別物として学ぶ
  • 局所性無視：隣接ピクセルの関係を捨てる

CNN はこれらを 3つの設計 で解決：

  1. 局所結合：各ニューロンが入力の一部のみ見る
  2. 重み共有：同じフィルタを全位置で再利用
  3. プーリング：階層的に解像度を下げる

§2 / CONV畳み込み演算

小さなフィルタ（カーネル）を画像上にスライドし、内積を取って特徴マップを作る。

FIG 1.1 畳み込み演算。フィルタ（カーネル）が入力上をスライドし、各位置で要素ごとの積の総和を取る。これにより「縦エッジがある場所」「丸い形がある場所」など、特定パターンへの応答が画像として得られる。

パラメータ：

  • カーネルサイズ：3×3 が標準（VGG, ResNet）。1×1 は次元圧縮用
  • ストライド：スライド幅。2 にすると出力解像度が半分
  • パディング：境界の処理。"same" は出力サイズ維持
  • チャネル：入力 RGB（3）、出力は任意（例：64, 128, 256...）

§3 / POOLプーリング

近接領域の代表値を取り、解像度を下げる。学習パラメータなし。

  • Max Pooling：領域内の最大値。最も一般的
  • Average Pooling：平均値
  • Global Average Pooling：特徴マップ全体を1つの値に集約。出力層直前で使用（FC層の代替）

§4 / STRIDEストライド・パディング・出力サイズ

出力サイズ = (入力サイズ + 2×P - K) / S + 1

K=カーネルサイズ、P=パディング、S=ストライド。

§5 / ARCH典型的 CNN 構造

古典的な構造：

[Conv + ReLU + Pool] × N → Flatten → FC → Softmax
FIG 1.2 典型的な CNN（VGG 風）の構造。各畳み込み層で空間解像度が半分になりつつ、チャネル数が増える。低層は局所パターン（エッジ）、高層は意味的特徴（顔・物体）を学習。

§6 / MODELS代表モデル史

1998LeNet（LeCun）— 手書き数字認識
2012AlexNet — ImageNet 革命。GPU、ReLU、Dropout 導入
2014VGG — 3×3 畳み込みを深く積む単純設計
2014GoogLeNet/Inception — 並列の異なるカーネルサイズ
2015ResNet — 残差接続で152層、ImageNet 優勝
2016DenseNet — すべての層を密に接続
2017MobileNet — Depthwise Separable Conv で軽量化
2019EfficientNet — Compound Scaling（幅・深さ・解像度の協調）
2020Vision Transformer (ViT) — CNN ではなく Transformer で画像
2022ConvNeXt — ViT に対抗する近代化 CNN

§7 / RESNETResNet の意義（再訪）

基本編 章11 で扱った残差接続 y = F(x) + x は、ResNet で導入。これにより：

  • 152層、最終的に 1000層超の NN が学習可能に
  • Bottleneck ブロック（1×1 → 3×3 → 1×1）でパラメータ削減
  • 後続の Transformer など あらゆる深層モデルの基本構造 となる

§8 / MODERNEfficientNet・ConvNeXt

EfficientNet（2019）：モデルサイズ拡大の3軸（幅・深さ・解像度）を 協調的にスケール。同精度でパラメータ数 1/10 を実現。

ConvNeXt（2022）：ViT に押された CNN を、Transformer の良い設計（LayerNorm、GELU、大カーネル）を逆輸入することで現代化。シンプルさを保ちつつ ViT に匹敵。

2畳み込み = フィルタを画像上にスライドして内積
