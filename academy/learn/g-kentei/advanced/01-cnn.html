G検定 応用編 — 章25

画像を、空間構造のまま
機械に理解させる。

普通のMLPで32x32の画像を扱うと、入力1024次元。フル結合で過剰なパラメータ。CNNはこの問題を、画像の 空間的局所性（近い画素は関係する）と 平行移動不変性（猫はどこにいても猫）の2つの仮定で解決します。畳み込みフィルタを画像全体にスライドさせてパラメータを共有、プーリングで階層的に圧縮 ── これが現代視覚AIの基本骨格です。

SECTION ONE

基本

畳み込み・フィルタ・ストライド/パディング・プーリング・受容野・チャネル・AlexNet・VGG・ResNet・EfficientNet/ViTの10節。

基本

§1.1 畳み込み演算 — CNNの心臓

(I * K)(i, j) = Σ_m Σ_n I(i+m, j+n) · K(m, n)

I：入力画像、K：カーネル（フィルタ、典型 3x3 や 5x5）
カーネルを画像上でスライドし、要素積の和を計算

『局所領域の重み付き和』を空間全体で繰り返す。パラメータ共有（同じカーネルを全位置で使用）が CNN の効率の根源。MLP（全結合）と比較して劇的にパラメータ削減＋平行移動不変性獲得。

FIG 13.1 ／ 畳み込み演算の例。3x3カーネルが5x5入力上をスライドし、各位置で要素積の和を計算して3x3出力を得る。学習で多様なカーネルが特徴検出器として獲得される。

§1.2 フィルタ・カーネル

典型サイズ：3x3、5x5、7x7。VGG以降は3x3を多層積み重ねるアプローチが主流（同じ受容野を少ないパラメータで実現）。フィルタは学習可能、初期はエッジ・色、深層は複雑なパーツへ。

§1.3 ストライドとパディング

出力サイズ = ⌊(入力 + 2P - K) / S⌋ + 1
　 K: カーネルサイズ、S: ストライド、P: パディング

パディング　 'same'：出力サイズ=入力、'valid'：パディング無し（縮小）
ストライド　 1=密、2=半分にダウンサンプリング

§1.4 プーリング（Pooling）

  • Max Pooling：領域内の最大値。最頻使用
  • Average Pooling：領域内の平均
  • Global Average Pooling：全体平均で空間次元を1に。最終層の代替
  • 典型 2x2、stride 2：空間次元を半分に
  • パラメータなし、計算高速、平行移動不変性強化

§1.5 受容野（Receptive Field）

出力の1ピクセルが見ている入力上の領域。深い層ほど受容野が広い。3x3 conv を 2層重ねると 5x5 受容野、3層で 7x7、しかしパラメータは少ない（VGG の発見）。

§1.6 マルチチャネル

カラー画像：RGB 3チャネル入力。中間層でフィルタ数 = 出力チャネル数（典型 64, 128, 256, 512）。チャネルとフィルタの全組合せで畳み込み、合計を出力。

§1.7 AlexNet — 深層学習革命の起点

Krizhevsky 2012、ImageNet 優勝（誤り率 15.3%、2位 26.2%）。8層、ReLU、GPU、Dropout、Data Augmentation を組み合わせて圧勝。深層学習時代の幕開け。

§1.8 VGG — シンプルな深層化

Simonyan 2014、Oxford。3x3 conv のみ16〜19層。シンプルで強力、転移学習で長く使われた。パラメータ多すぎ（約 140M）。

§1.9 ResNet — 残差接続の発明

He 2015、Microsoft。残差接続 y = F(x) + x で 152層 を学習可能に。ImageNet 誤り率 3.57%、人間（5.1%）を上回る。現代の事実上の標準構造。

§1.10 EfficientNet / Vision Transformer

モデル年特徴
EfficientNet2019深さ・幅・解像度を複合スケーリング、効率最良
ViT (Vision Transformer)2020画像をパッチに分けてTransformer適用、JFT-300Mで CNN を上回る
Swin Transformer2021階層的ViT、局所窓注意
ConvNeXt2022ViT のテクニックを CNN に逆輸入
DINOv22023自己教師あり学習でSOTA
SECTION TWO

応用

物体検出、セマンティックセグメンテーション、転移学習、ViTの4節。

応用

§2.1 物体検出

モデル方式特徴
R-CNN 系（Fast/Faster R-CNN）2-stageRoI 提案 + 分類、精度高
YOLO 系（YOLOv8 等）1-stageリアルタイム、グリッド予測
SSD1-stageマルチスケール特徴
DETR / DINOTransformerEnd-to-End、NMS不要
SAM (Segment Anything)Foundation2023 Meta、プロンプト型

§2.2 セマンティックセグメンテーション

ピクセル単位のクラス分類。U-Net（医療画像で標準）、DeepLab、Mask R-CNN（インスタンスセグメンテーション）、SAM（Segment Anything Model）。

§2.3 画像分類の転移学習

ImageNet で事前学習したモデル（ResNet50、EfficientNet、ViT）の最終層だけ置き換えて少量データで学習。データが少ないタスクで標準。Hugging Face Hub に多数公開。

§2.4 Vision Transformer の革命

2020 Dosovitskiy ら：画像を 16x16 パッチに分けて Transformer に入力。CNN の局所性の帰納バイアスなしで、大量データなら CNN を上回る。CLIP、DINOv2、Stable Diffusion で標準。

SECTION THREE

ハンズオン

PyTorch CNN、転移学習、特徴マップ、Grad-CAM、データ拡張の5節。

ハンズオン

§3.1 PyTorch で CNN

import torch.nn as nn class SimpleCNN(nn.Module): def __init__(self, num_classes=10): super().__init__() self.features = nn.Sequential( nn.Conv2d(3, 32, 3, padding=1), nn.BatchNorm2d(32), nn.ReLU(), nn.Conv2d(32, 64, 3, padding=1), nn.BatchNorm2d(64), nn.ReLU(), nn.MaxPool2d(2), # 32→16 nn.Conv2d(64, 128, 3, padding=1), nn.BatchNorm2d(128), nn.ReLU(), nn.MaxPool2d(2), # 16→8 ) self.classifier = nn.Sequential( nn.AdaptiveAvgPool2d(1), # GAP nn.Flatten(), nn.Linear(128, num_classes)) def forward(self, x): return self.classifier(self.features(x))

§3.2 ImageNet 転移学習

from torchvision import models import torch.nn as nn # 事前学習済みResNet50をロード model = models.resnet50(weights='IMAGENET1K_V2') # 最終層を新しいタスク用に置換 num_classes = 10 model.fc = nn.Linear(model.fc.in_features, num_classes) # 特徴抽出は固定、最終層のみ学習 for param in model.parameters(): param.requires_grad = False for param in model.fc.parameters(): param.requires_grad = True

§3.3 特徴マップの可視化

import matplotlib.pyplot as plt # 中間層出力を取得（hook） features = {} def hook(name): def _hook(module, inp, out): features[name] = out.detach() return _hook model.layer1.register_forward_hook(hook('layer1')) model.layer2.register_forward_hook(hook('layer2')) model.layer3.register_forward_hook(hook('layer3')) # 順伝播後、features['layer1']が16x56x56等の特徴マップ output = model(image) # 浅層 → エッジ、中層 → テクスチャ、深層 → セマンティック

SECTION FOUR

シミュレーション

畳み込みフィルタの効果、プーリング比較、受容野の3節。

シミュレーション

SIM 1 — 畳み込みフィルタが何を学ぶか

>> ImageNet学習済みResNet50の各層フィルタを可視化

Layer 1 (conv1, 7x7):
　 → 縦線、横線、斜線、色のグラデーション
　 → ガボールフィルタ風、視覚野V1に類似

Layer 5-10 (中層):
　 → 円、十字、テクスチャパターン
　 → 部品検出器（目、車輪、葉脈など）

Layer 15-50 (深層):
　 → 犬の顔、車の前部、文字の組合せ
　 → 高レベル概念検出器

>> 階層的に低レベル→高レベル特徴を学習
>> これが転移学習が機能する根拠

SIM 2 — Max Pooling vs Average Pooling

>> CIFAR-10、同じCNN、Pooling種類だけ変更

Max Pooling:　　test=92.3%　 ← 最頻使用、エッジ・特徴を強調
Avg Pooling:　　test=89.8%
Stride=2 Conv:　test=92.1%　 ← Pooling 代替、学習可能
Global Avg Pool: 最終層、パラメータ削減効果

>> Max Pooling が経験的に最良
>> 強い活性化を保持、ノイズ抑制

SIM 3 — 受容野の拡大

>> 各層の理論的受容野（VGG-16）

Conv1_1: 3x3
Conv1_2: 5x5
Conv2_1 (after pool): 10x10
Conv3_1: 24x24
Conv4_1: 60x60
Conv5_3 (深層): 196x196　 ← 入力224x224の大半をカバー

>> 深い層は画像全体を見る能力
>> だから高レベル概念を捉えられる
>> Dilated Conv で受容野を更に拡大可能
SECTION FIVE

問題

G検定形式の演習問題 20問。

問題

PROBLEM 1
CNN が MLP より画像処理に適している理由は。
① 計算が遅い　 ② 空間的局所性と平行移動不変性の帰納バイアス、パラメータ共有でパラメータ削減　 ③ 学習が不安定　 ④ メモリを大量に必要とする
ANSWER ② CNN は同じフィルタを全位置で適用（パラメータ共有）し、近い画素関係を仮定。MLPの全結合に比べパラメータが2〜3桁少なく、性能も高い。
PROBLEM 2
2012年の AlexNet が ImageNet コンペで起こした革命の核心は。
① 浅い NN を使った　 ② 深層CNN + ReLU + Dropout + GPU + データ拡張で従来手法を圧倒　 ③ 教師なし学習　 ④ 強化学習
ANSWER ② Krizhevsky 2012。誤り率 15.3% vs 2位 26.2%。深層学習革命の起点となった。
PROBLEM 3
入力 32x32、3x3 Conv、stride=1、padding=1 のとき出力サイズは。
① 30x30　 ② 32x32（same パディング）　 ③ 16x16　 ④ 34x34
ANSWER ② (32 + 2·1 - 3) / 1 + 1 = 32。padding='same' で入力と同じサイズを維持。VGG以降で標準。
PROBLEM 4
Max Pooling の役割として正しいのは。
① 学習可能パラメータ追加　 ② 領域内最大値を取り、空間ダウンサンプリング・平行移動不変性強化　 ③ 正則化のみ　 ④ 出力層
ANSWER ② パラメータなし、空間次元削減、強い活性化を保持。Average Pooling より一般的。
PROBLEM 5
ResNet（2015）が解決した深層化の壁は。
① 計算量　 ② 勾配消失と退化問題：残差接続 y=F(x)+x で 152層を学習可能に　 ③ メモリのみ　 ④ ラベル不足
ANSWER ② He 2015。スキップ接続で勾配が直接伝播。ImageNet 誤り率 3.57%、人間（5.1%）超え。現代CNN/Transformerの標準構造。
PROBLEM 6
VGG の特徴的な設計は。
① 1x1 Conv のみ　 ② 3x3 Conv のみを多層積み重ね（16〜19層）　 ③ ランダム　 ④ 全結合のみ
ANSWER ② Simonyan 2014。シンプルで強力。3x3 を 2層で 5x5 相当の受容野をパラメータ少なく実現する発見。
PROBLEM 7
『受容野（Receptive Field）』とは。
① 出力ピクセルが影響を受ける入力上の領域　 ② パラメータ数　 ③ バッチサイズ　 ④ 学習率
ANSWER ① 深い層ほど受容野が広い。3x3 conv を 2層重ねると 5x5 受容野。Dilated Conv で更に拡大可能。
PROBLEM 8
EfficientNet (2019) の貢献は。
① ランダム探索　 ② 深さ・幅・解像度を複合スケーリングで最適化、同精度で大幅パラメータ削減　 ③ 教師なし　 ④ ViT 提案
ANSWER ② Tan & Le 2019。compound scaling で計算予算に対する精度効率を最大化。モバイル向けにも適用。
PROBLEM 9
Vision Transformer (ViT) の入力処理は。
① 画素単位　 ② 画像を 16x16 などのパッチに分けてトークン化し Transformer に入力　 ③ Conv のみ　 ④ MLP のみ
ANSWER ② Dosovitskiy 2020。CNN の帰納バイアスなしで、大量データ（JFT-300M）で CNN を超えた。CLIP、Stable Diffusion で標準。
PROBLEM 10
YOLO シリーズの特徴は。
① 2-stage　 ② 1-stage：画像をグリッド分割し各セルで物体予測、リアルタイム　 ③ 教師なし　 ④ セグメンテーション特化
ANSWER ② Redmon 2016 で導入、YOLOv8/v10 まで進化。FPS が高く、産業応用で広く使われる。
PROBLEM 11
U-Net (2015) が広く使われる分野は。
① 物体検出　 ② 医療画像セグメンテーション、エンコーダ-デコーダ＋スキップ接続　 ③ 言語モデル　 ④ 強化学習
ANSWER ② Ronneberger 2015。U字型構造で詳細を保持。Diffusion モデル（DDPM、Stable Diffusion）の denoiser にも採用。
PROBLEM 12
転移学習（ImageNet → 別タスク）が機能する理由は。
① 偶然　 ② 浅層が汎用的特徴（エッジ、テクスチャ）を学習、それは多くのタスクで再利用可能　 ③ ランダム　 ④ パラメータ多いから
ANSWER ② 階層的特徴学習。少量データのタスクで事前学習モデルをファインチューニングすると、ゼロから学習より大幅に高精度。
PROBLEM 13
Global Average Pooling (GAP) の役割は。
① 学習する　 ② 各チャネルの空間平均でベクトル化、最終FC層代替、パラメータ削減　 ③ ノイズ追加　 ④ 正則化のみ
ANSWER ② NIN/ResNet以降で標準。Flatten + 巨大 FC をGAPで置換、過学習防止＋パラメータ削減。Class Activation Map の基礎にも。
PROBLEM 14
Grad-CAM の目的は。
① 学習加速　 ② 予測根拠の可視化（どの画像領域がクラス予測に貢献したか）　 ③ 圧縮　 ④ ノイズ除去
ANSWER ② Selvaraju 2017。最終 Conv 層の勾配を使ってヒートマップを生成。XAI（説明可能AI）の標準ツール、医療画像で重要。
PROBLEM 15
1x1 Convolution（pointwise）の役割は。
① 役立たず　 ② チャネル間の線形結合、次元削減・拡張、Inception/ResNet で頻用　 ③ プーリング代替　 ④ 正則化
ANSWER ② 空間方向は1x1で変えず、チャネル方向のみ変換。Bottleneck構造で計算量削減。MobileNet（Depthwise Separable Conv）でも核心。
PROBLEM 16
Segment Anything Model (SAM, 2023) の特徴は。
① 単一タスク　 ② プロンプト型セグメンテーション、未知物体も可、Foundation Modelとして広く転用可　 ③ 教師なしのみ　 ④ 軽量モデル
ANSWER ② Meta AI 2023。点・ボックス・テキストプロンプトで任意の物体をセグメント。10億+のマスクで事前学習。
PROBLEM 17
DenseNet (2017) の特徴は。
① 全層をスキップ接続でなく完全に連結（dense connection）、特徴再利用と勾配伝播改善　 ② パラメータ激増　 ③ 浅い NN　 ④ 物体検出専用
ANSWER ① Huang 2017。各層が前のすべての層の出力を入力に。パラメータ効率と精度が高い。
PROBLEM 18
物体検出の評価指標として標準的なのは。
① Accuracy　 ② mAP（mean Average Precision）@IoU 0.5、0.5:0.95　 ③ MSE　 ④ R²
ANSWER ② COCO標準。クラスごとの AP の平均、IoU 閾値で性能評価。YOLO, Faster R-CNN等の比較で使用。
PROBLEM 19
ConvNeXt (2022) の主張は。
① ViTが万能　 ② CNNにViTの工夫（GELU、LN、大きなカーネル等）を取り入れれば、CNNがViTと同等以上の精度に　 ③ Conv 不要　 ④ 教師なし
ANSWER ② Liu 2022。ViT の優位性はアーキテクチャ自体ではなく、学習レシピと工夫にあると示唆。
PROBLEM 20
Dilated（Atrous）Convolution の効果は。
① パラメータ増加　 ② カーネル内のサンプル間隔を空けて受容野を拡大、パラメータ・計算量は同じ　 ③ ノイズ追加　 ④ 教師なし
ANSWER ② DeepLab系で活用。セグメンテーションで詳細を保ちつつ広い文脈を捉える。
