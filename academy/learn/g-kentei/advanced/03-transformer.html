応用 章3 — TransformerAttention is All You Need

2017年 Vaswani らによる「Attention Is All You Need」が現代AIの分水嶺。RNN/CNN を捨て、Self-Attention だけで系列モデルを作るという提案でした。GPT、BERT、ChatGPT、Claude、Stable Diffusion、ViT すべての基礎。本章はその構造を完全に理解することを目標にします。

§1 / WHYなぜ Transformer か

RNN の問題：

  • 逐次計算：時刻 t の計算には t-1 が必要 → 並列化不可
  • 長期依存：LSTM でも数百トークン以上は厳しい

Transformer の利点：

  • 完全並列：全位置を同時に処理 → GPU を最大活用
  • 長距離依存：任意の2位置間が直接接続される
  • スケーリングが効く：パラメータ・データ・計算量を増やせば性能が伸びる

§2 / ATTNAttention の基本

Attention の本質：「クエリ q と複数のキー k₁, ..., kₙ の類似度に基づき、対応する値 v₁, ..., vₙ を加重和する」。

Attention(q, K, V) = Σᵢ softmax(q · kᵢ / √d) · vᵢ

つまり「辞書検索の柔らかいバージョン」：

  • 普通の辞書：q とキーが完全一致したものを返す
  • Attention：q と各キーの類似度に応じて値を加重平均

§3 / QKVQuery, Key, Value

各単語の埋め込みベクトル x から、3つの異なる表現を線形変換で作る：

Q = X W_Q, K = X W_K, V = X W_V

意味：

  • Query (Q)：「何を探しているか」の表現
  • Key (K)：「自分はどんな内容か」の見出し
  • Value (V)：実際に渡したい内容

§4 / SELFSelf-Attention

同じ系列の単語同士で attention を取る。文章「The cat sat on the mat」では、各単語が他のすべての単語と関係を計算する。

FIG 3.1 Self-Attention は各単語が他のすべての単語に「注目度」を計算する。"cat" は "sat" や "mat" との関係性が強く、文脈に応じた表現が得られる。

4.1 計算の流れ

Attention(Q, K, V) = softmax(QKᵀ / √d_k) V

√d_k で割るのは「スケーリング」。次元 d_k が大きいと QKᵀ も大きくなり、softmax が極端な分布に偏るのを防ぐ。

§5 / MULTIMulti-Head Attention

1つの Attention だけでなく、h 個の Attention を並列に 行い、結果を結合する。

MultiHead(Q,K,V) = Concat(head₁, ..., head_h) W_O
where headᵢ = Attention(Q W_Qᵢ, K W_Kᵢ, V W_Vᵢ)

各ヘッドが 異なる種類の関係性 を学習する：

  • 1つのヘッド：主語-動詞関係
  • 別のヘッド：名詞-修飾語関係
  • さらに別のヘッド：照応関係（he ↔ John）

§6 / POSPosition Encoding

Attention は順序を持たない（全位置が並列）。"cat sat on mat" と "mat on sat cat" を区別できない。
→ 位置情報を埋め込みベクトルに加える。

古典的な Sinusoidal Encoding：

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

近年は RoPE（Rotary Position Embedding）、ALiBi など改良版が主流。Llama、Mistral などが RoPE を採用。

§7 / ARCH全体構造

FIG 3.2 Transformer の Encoder-Decoder 構造。各ブロックは Multi-Head Attention + FFN + Add&Norm の組合せ。Decoder は Masked Self-Attention（未来を見ない）と Cross-Attention（Encoder の出力に注目）を持つ。

§8 / VARIANTS派生モデル

Encoder のみBERT, RoBERTa, DeBERTa — 双方向の言語理解
Decoder のみGPT 系, Llama, Claude, Gemini, Mistral — 自己回帰生成
Encoder-DecoderT5, BART — 入出力が独立の系列対系列
Vision TransformerViT（2020）— 画像をパッチに分割して Transformer に
AudioWhisper, AudioLM — 音声処理
マルチモーダルCLIP, GPT-4V, Gemini — テキスト＋画像＋動画

§9 / SUMMARY章のまとめ

1Transformer：RNN/CNN を捨て Self-Attention だけで構成（2017）
2並列計算可能、長距離依存に強い
3Attention = softmax(QKᵀ/√d) V：類似度に応じた加重平均
4Q, K, V は同じ入力から異なる線形変換で生成
5Self-Attention：同系列内で attention
6Multi-Head：複数の attention を並列、異なる関係性を学習
7Position Encoding で順序情報を追加（Sinusoidal、RoPE、ALiBi）
8Encoder：Multi-Head + FFN + Add&Norm
9Decoder：Masked Self-Attention + Cross-Attention + FFN
10派生：BERT（Enc）、GPT（Dec）、T5（Enc-Dec）、ViT（画像）
