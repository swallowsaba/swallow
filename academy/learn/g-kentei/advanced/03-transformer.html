G検定 応用編 — 章27

『Attention is all you need.』
2017年6月、AIが変わった日。

Transformer の本質は、RNN の逐次性を捨てたこと。系列全体を一度に処理し、各位置が他のすべての位置を Attention で参照する。並列計算が可能になり、長距離依存も学習できる。これが GPU の力と相まって、現代の基盤モデル時代を実現したのです。

SECTION ONE

基本

Attention is All You Need、Self-Attention、Multi-Head、Position Encoding、LayerNorm+残差、FFN、Encoder/Decoder、Masking、計算量、影響の10節。

基本

§1.1 Attention is All You Need（2017）

Vaswani et al. 2017、Google Brain。WMT 英→独翻訳で SOTA、しかも RNN より速い。論文タイトル『Attention is all you need』は的を射ていた ── Recurrence も Convolution も不要、Self-Attention だけで十分。

§1.2 Scaled Dot-Product Attention

Attention(Q, K, V) = softmax(QKT / √d_k) V

Q（Query）　 　 検索したい情報
K（Key）　 　 　キーワード
V（Value）　 　 値
√d_k で割るのは、softmax の数値安定性のため
FIG 15.1 ／ Scaled Dot-Product Attention の計算フロー。Q と K の内積で類似度、√d_k でスケール、Softmaxで重み化、V との積で出力。すべて行列演算でGPU並列化容易。

§1.3 Multi-Head Attention

MultiHead(Q,K,V) = Concat(head₁, ..., head_h) W^O
where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)

典型 h=8〜16 ヘッド、各ヘッドが異なる『観点』で注目できる

§1.4 Position Encoding

Transformerは位置情報を持たない（permutation invariant）。位置埋め込みを足し込む：

PE(pos, 2i) = sin(pos / 10000^{2i/d})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d})

学習可能位置埋め込み（BERT, GPT-2 等）、相対位置（T5、RoPE）も

§1.5 Layer Norm + Residual

各サブ層は y = LayerNorm(x + Sublayer(x)) で構成（Post-Norm）。最新のLLMは Pre-Norm（y = x + Sublayer(LayerNorm(x))）が一般的、学習が安定。

§1.6 Position-wise Feed-Forward Network

FFN(x) = max(0, xW₁ + b₁) W₂ + b₂　 (ReLU)
典型 d_model=768, d_ff=3072（4倍）
LLaMA 等は SwiGLU を使用、パラメータ多いが高性能

§1.7 Encoder と Decoder

EncoderDecoder
Self-Attention双方向Causal Masked
Cross-Attentionなしあり（EncOutへ）
用途理解（BERT）生成（GPT）

§1.8 Causal Masking

生成タスク（GPT）では未来情報を見せない。Attention 行列の上三角を -∞ で埋め、softmax後 0 に。

§1.9 計算量とメモリ

Self-Attention の計算量・メモリ：O(N² · d)
N = 系列長、d = 埋め込み次元

長文（N=10000）で爆発。Flash Attention（メモリO(N)実装）、Sparse Attention、Linear Attention で対処

§1.10 Transformer の影響

2018 BERT/GPT-1 → 2020 GPT-3 → 2022 ChatGPT → 2024 GPT-4o/Claude 3.5/Llama 3 → 2025 GPT-5/Claude 4 ── すべて Transformer 系。Vision Transformer (2020) で画像も統一、AlphaFold 2 でタンパク質構造、AlphaGo は碁、すべて Transformer 派生。Web 全体に大きな影響を与え続けるアーキテクチャ。

SECTION TWO

応用

Encoder-only、Decoder-only、Encoder-Decoder、マルチモーダルの4節。

応用

§2.1 Encoder-only（BERT 系）

BERT、RoBERTa、DeBERTa、ALBERT。MLM（Masked Language Model）で事前学習、文分類・NER・QAなどの理解タスクで強力。

§2.2 Decoder-only（GPT 系）

GPT-1/2/3/4、Llama、Claude、Mistral、Gemini。Next Token Prediction で学習、生成タスクの王者。スケーリング則で性能が予測可能に拡大。

§2.3 Encoder-Decoder（T5、BART、NMT）

T5、BART、mT5。テキスト→テキスト変換、要約・翻訳・QAに強い。最近は decoder-only が主流だが、機械翻訳など特定タスクで依然有効。

§2.4 Vision/Multi-Modal Transformer

ViT（2020）、CLIP（2021）、Flamingo、GPT-4V、Claude Opus 4.6、Gemini 2.5。画像をパッチ化してテキストと統一処理。マルチモーダル基盤モデルへ。

SECTION THREE

ハンズオン

Attention 実装、Multi-Head、Transformer ブロック、HuggingFace、可視化の5節。

ハンズオン

§3.1 Scaled Dot-Product Attention

import torch import torch.nn.functional as F def attention(Q, K, V, mask=None): d_k = Q.size(-1) scores = Q @ K.transpose(-2, -1) / d_k**0.5 if mask is not None: scores = scores.masked_fill(mask == 0, -1e9) weights = F.softmax(scores, dim=-1) return weights @ V, weights

§3.2 Multi-Head Attention

class MultiHeadAttention(nn.Module): def __init__(self, d_model=512, h=8): super().__init__() self.d_k = d_model // h self.h = h self.W_q = nn.Linear(d_model, d_model) self.W_k = nn.Linear(d_model, d_model) self.W_v = nn.Linear(d_model, d_model) self.W_o = nn.Linear(d_model, d_model) def forward(self, q, k, v, mask=None): B = q.size(0) Q = self.W_q(q).view(B, -1, self.h, self.d_k).transpose(1, 2) K = self.W_k(k).view(B, -1, self.h, self.d_k).transpose(1, 2) V = self.W_v(v).view(B, -1, self.h, self.d_k).transpose(1, 2) out, _ = attention(Q, K, V, mask) out = out.transpose(1, 2).contiguous().view(B, -1, self.h * self.d_k) return self.W_o(out)

§3.3 Transformer ブロック

class TransformerBlock(nn.Module): def __init__(self, d_model, h, d_ff): super().__init__() self.attn = MultiHeadAttention(d_model, h) self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model)) self.norm1 = nn.LayerNorm(d_model) self.norm2 = nn.LayerNorm(d_model) def forward(self, x, mask=None): x = self.norm1(x + self.attn(x, x, x, mask)) x = self.norm2(x + self.ffn(x)) return x

§3.4 HuggingFace で簡単利用

from transformers import AutoModel, AutoTokenizer tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased') model = AutoModel.from_pretrained('bert-base-uncased') inputs = tokenizer("Hello, Transformers!", return_tensors='pt') outputs = model(**inputs) print(outputs.last_hidden_state.shape) # [1, 6, 768]

§3.5 Attention 重みの可視化

import matplotlib.pyplot as plt import seaborn as sns model = AutoModel.from_pretrained('bert-base-uncased', output_attentions=True) outputs = model(**inputs) # outputs.attentions: tuple of (batch, heads, seq, seq) for each layer attn = outputs.attentions[5][0, 0].detach() # layer6, head1 tokens = tokenizer.convert_ids_to_tokens(inputs['input_ids'][0]) sns.heatmap(attn, xticklabels=tokens, yticklabels=tokens, cmap='Blues') plt.savefig('attn.png')
SECTION FOUR

シミュレーション

Attention重みの観察、長距離依存、計算量比較の3節。

シミュレーション

SIM 1 — Attention重みは何を学ぶか

>> 文 "The cat sat on the mat" の self-attention

Head 1（構文）: "sat" → "cat"（主語）の重み 0.62
　　　　　　　 → "mat"（場所）の重み 0.18

Head 4（共参照）: "the" → 該当する名詞へ強く

Head 7（文末）: 文末トークン → 全単語へ均等に集約

>> マルチヘッドが異なる言語学的役割を学習する
>> 解釈可能性の研究分野（BERTology）の発見

SIM 2 — 長距離依存の効果

>> 200トークンの文書で『代名詞の指示先』を予測

[LSTM]
　 距離10:　 acc 87%
　 距離50:　 acc 64%
　 距離150:　acc 38%　 ← 遠いほど落ちる

[Transformer]
　 距離10:　 acc 92%
　 距離50:　 acc 90%
　 距離150:　acc 87%　 ← ほぼ不変

>> Attention は距離に依存しない
>> これが長文理解の革命的要因

SIM 3 — 計算量 N² の意味

>> 系列長 N とメモリ使用量（A100 GPU）

N = 512: メモリ 4 GB（標準）
N = 2048: メモリ 64 GB（限界近い）
N = 4096: メモリ 256 GB（不可能）
N = 16384: メモリ 4 TB（笑）

>> だから長文脈 LLM には工夫が必要：
>> Flash Attention（メモリO(N)）
>> Sliding Window Attention
>> Mamba/SSM（O(N)）
>> これらで Claude 3 は 200K、Gemini 1.5 は 1M トークン
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
『Attention is All You Need』論文（2017）の革命性は。
① CNN を提案　 ② RNN/Convolutionなしで Self-Attention のみで Seq2Seq を実現、並列化可能　 ③ K-means 改良　 ④ 強化学習
ANSWER ② Vaswani 2017、Google Brain。WMT 英→独 で SOTA、しかも RNN より高速。現代AIの基盤。
PROBLEM 2
Self-Attention の計算式は。
① QV+K　 ② softmax(QKᵀ/√dₖ) V　 ③ Q·K　 ④ Σ Q·V
ANSWER ② Scaled Dot-Product Attention。√dₖで割るのは数値安定性のため。
PROBLEM 3
Multi-Head Attention の利点は。
① 計算量削減　 ② 異なる『観点』で並列に注目、表現力増加　 ③ パラメータ削減　 ④ 教師なし
ANSWER ② 典型 8〜16 ヘッド。各ヘッドが構文、共参照、意味類似など異なる側面を学習。
PROBLEM 4
Transformer が Position Encoding を必要とする理由は。
① 高速化　 ② Self-Attentionは置換不変、位置情報を別途与える必要　 ③ 正則化　 ④ 教師信号
ANSWER ② sin/cos の固定位置埋め込み（オリジナル）、学習可能埋め込み（BERT）、RoPE（LLaMA、相対位置）など。
PROBLEM 5
RoPE（Rotary Position Embedding）が現代LLMで採用される理由は。
① 学習不要　 ② 相対位置を回転で表現、外挿性能と長文脈で優位　 ③ 計算最遅　 ④ メモリ大
ANSWER ② Su 2021。LLaMA, GPT-NeoX, PaLM 等で採用。長文脈学習・補間で優れる。
PROBLEM 6
Self-Attention の計算量は。
① O(N)　 ② O(N · d)　 ③ O(N² · d)　 ④ O(N³)
ANSWER ③ N = 系列長、d = 次元。長文で爆発するため Flash Attention、Sparse Attention、Mamba 等で対処。
PROBLEM 7
FlashAttention の貢献は。
① 精度向上　 ② メモリO(N²) → O(N) の実装、GPU メモリ階層を活用　 ③ 教師なし　 ④ ランダム
ANSWER ② Dao 2022。SRAM/HBM のデータ移動を最小化。計算結果は数値的に同一、高速・低メモリ。長文脈 LLM 必須。
PROBLEM 8
Decoder の Causal Mask の役割は。
① 高速化　 ② 生成時に未来のトークンを見せない（カンニング防止）　 ③ 正則化　 ④ 教師なし
ANSWER ② Attention 行列の上三角を -∞ で埋める。GPT 系の自己回帰生成で必須。
PROBLEM 9
Encoder-only Transformer の代表例は。
① GPT　 ② BERT、RoBERTa、DeBERTa　 ③ T5　 ④ ResNet
ANSWER ② 理解タスク（分類、NER、QA）に強い。MLM で事前学習。
PROBLEM 10
Decoder-only Transformer の代表例は。
① BERT　 ② GPT、LLaMA、Claude、Mistral　 ③ T5　 ④ ViT
ANSWER ② Next Token Prediction で事前学習。生成タスクの王者。現代 LLM のデファクト。
PROBLEM 11
Encoder-Decoder Transformer の代表例は。
① GPT　 ② T5、BART、mT5、オリジナル Transformer　 ③ BERT　 ④ K-means
ANSWER ② テキスト→テキスト変換に強い。要約、翻訳、QAを統一フレームで扱える（T5）。
PROBLEM 12
Layer Normalization が Transformer で標準採用される理由は。
① 高速　 ② バッチサイズ非依存、系列長変化OK、学習安定化　 ③ パラメータ削減　 ④ 教師なし
ANSWER ② Batch Norm はバッチ依存、可変長系列・小バッチで不安定。LayerNorm が Transformer の標準。最近は RMSNorm も（LLaMA）。
PROBLEM 13
Vision Transformer (ViT, 2020) の入力処理は。
① 画素単位　 ② 画像を 16x16 などのパッチに分けてトークン化　 ③ Conv のみ　 ④ MLP のみ
ANSWER ② Dosovitskiy 2020。CNN の帰納バイアスなしで、大量データなら CNN 超え。CLIP、Stable Diffusion 等で採用。
PROBLEM 14
Cross-Attention の用途は。
① Encoder内　 ② Decoderが Encoder出力を参照、機械翻訳・要約で使用　 ③ 教師なし　 ④ 計算不要
ANSWER ② Q を Decoder から、K/V を Encoder から。Encoder-Decoder アーキの架け橋。
PROBLEM 15
Pre-Norm と Post-Norm の違いは。
① 同じ　 ② Pre: y=x+Sub(LN(x)), Post: y=LN(x+Sub(x))、Pre が学習安定で現代LLM標準　 ③ パラメータ違う　 ④ 学習なし
ANSWER ② オリジナル Transformer は Post-Norm だが Warmup が必要。Pre-Norm（GPT-2 以降）は学習がより安定。
PROBLEM 16
KV Cache の役割は。
① 訓練加速　 ② 自己回帰生成で過去トークンの K, V を保存して再計算回避、推論を大幅高速化　 ③ 正則化　 ④ ノイズ
ANSWER ② 各トークン生成時、過去全トークンの K, V は変わらないため再利用。LLM 推論の効率化に必須。メモリは N に線形に増加。
PROBLEM 17
SwiGLU 活性化が LLaMA 等の現代 LLM で採用される理由は。
① 古典的　 ② Gated Linear Unit + Swish、ReLU/GELU より高性能　 ③ 計算最速　 ④ 教師なし
ANSWER ② Shazeer 2020。SwiGLU(x) = (xW)⊙Swish(xV) で2つの線形変換のゲート積。LLaMA, PaLM 等で標準。
PROBLEM 18
『Sparse Attention』の戦略は。
① 全位置に注目　 ② 一部位置だけに注目し計算量を O(N√N) や O(N log N) に削減　 ③ 教師なし　 ④ Conv 化
ANSWER ② Sparse Transformer、Longformer、BigBird、Reformer 等。長文脈モデルで重要、ただし Flash Attention + 通常 Attention で多くの場合十分。
PROBLEM 19
Mixture of Experts (MoE) の Transformer での利用は。
① 不要　 ② FFN を複数の Expert に分け、Router が選択。総パラメータ大、アクティブパラメータ小　 ③ Attention のみ　 ④ 教師なし
ANSWER ② GShard、Switch Transformer、Mixtral 8x7B、GPT-4 等で採用。スケーリングと計算効率の両立。
PROBLEM 20
Transformer が CNN/RNN に勝った根本的理由は。
① 帰納バイアスが弱く、大量データで純粋に学習できる（スケーリング有利）+ 並列計算可能　 ② 計算量が少ない　 ③ パラメータ少ない　 ④ 教師なし
ANSWER ① CNN は局所性、RNN は逐次性の帰納バイアス。Transformer はそれらが弱い分、データから自由に学習可能でスケーリング則に最適。並列学習で計算リソース活用も容易。

章のまとめ

Attention is All You Need、Scaled Dot-Product Attention、Multi-Head、Position Encoding（sin/cos、学習可能、RoPE）、LayerNorm+残差（Pre/Post-Norm）、FFN（ReLU/GELU/SwiGLU）、Encoder/Decoder、Causal Mask、計算量 O(N²)、Flash Attention、KV Cache、Sparse Attention、MoE を扱いました。応用で Encoder-only/Decoder-only/Encoder-Decoder/Vision/MultiModal。ハンズオンで Attention/Multi-Head/Transformer ブロック/HuggingFace/可視化。次章は BERT と GPT — Transformer の2大派生で、現代 NLP を作った2つの哲学へ。
