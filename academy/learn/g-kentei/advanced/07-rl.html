G検定 応用編 — 章31

報酬から、行動を学ぶ。
AlphaGoから、RLHFまで。

教師あり学習が『正解』を必要とするのに対し、強化学習は『報酬』だけで学習します。チェス、囲碁、StarCraft、Atariゲーム、ロボット制御、自動運転、レコメンド、そして LLM の人間嗜好調整（RLHF）── 報酬を最大化する行動の学習は、AI の表現力を爆発的に拡張しました。

SECTION ONE

基本

MDP、価値関数、Bellman、Q-Learning、SARSA、DQN、方策勾配、Actor-Critic、PPO、探索・活用の10節。

基本

§1.1 マルコフ決定過程（MDP）

MDP = (S, A, P, R, γ)
　 S：状態空間（盤面、ロボットの位置など）
　 A：行動空間
　 P(s'|s, a)：遷移確率
　 R(s, a, s')：報酬関数
　 γ ∈ [0,1)：割引率（将来の報酬の現在価値）

マルコフ性　 P(s_{t+1}|s_t, a_t) = P(s_{t+1}|s_t, a_t, s_{t-1}, ...)　 (過去の歴史は s_t に集約)

§1.2 方策と価値関数

方策　 π(a|s)　 状態sで行動aを取る確率

状態価値関数　 Vπ(s) = Eπ[Σ γt rt | s0=s]
行動価値関数（Q関数）　 Qπ(s, a) = Eπ[Σ γt rt | s0=s, a0=a]

最適価値　 V*(s) = maxπ Vπ(s)　 　Q*(s, a) = maxπ Qπ(s, a)

§1.3 Bellman 方程式 — 強化学習の核

Bellman期待方程式
　 Vπ(s) = Σa π(a|s) Σs' P(s'|s,a)[R(s,a,s') + γ Vπ(s')]

Bellman最適方程式
　 V*(s) = maxa Σs' P(s'|s,a)[R + γ V*(s')]
　 Q*(s, a) = Σs' P(s'|s,a)[R + γ maxa' Q*(s', a')]

動的計画法　 Bellman を反復解：Value Iteration、Policy Iteration

§1.4 Q-Learning — モデルなしオフポリシー

Q-Learning 更新
　 Q(s, a) ← Q(s, a) + α [r + γ maxa' Q(s', a') - Q(s, a)]

性質
　・モデルなし（P, R を知らなくて良い）
　・オフポリシー（行動方策と異なる方策を学習可能）
　・収束保証（適切な学習率、無限訪問）
　・テーブル形式は状態数爆発で大規模問題に弱い

§1.5 SARSA — オンポリシー版

SARSA 更新　 Q(s, a) ← Q(s, a) + α[r + γ Q(s', a') - Q(s, a)]

Q-Learning との違い：max ではなく、実際に取った行動 a' の Q を使う。
オンポリシー：探索を考慮した方策を学習。崖歩きで安全な経路を学ぶ。

§1.6 DQN — ディープQネットワーク

2013 DeepMind。Q 関数を NN で近似：

Q(s, a; θ) ≈ Q*(s, a)　 (NN で表現)

3つの工夫
　 1. Experience Replay：過去経験をバッファに蓄積、ランダムサンプル
　 2. Target Network：ターゲットQ用に別NN、定期的に同期
　 3. ε-greedy 探索

成果　 Atari 49ゲームで人間超え（2015 Nature）、強化学習革命の始まり
FIG 19.1 ／ 強化学習の基本ループ。エージェントが行動を選び、環境が状態遷移と報酬を返す。試行錯誤で最適方策を学習。

§1.7 方策勾配（Policy Gradient）

方策を直接パラメータ化　 πθ(a|s)

REINFORCE　 ∇J(θ) = Eπ[∇log πθ(a|s) · Gt]
　 Gt = Σ γk rt+k　 (累積報酬)

価値ベースは離散行動向き、方策ベースは連続行動も扱える。

§1.8 Actor-Critic — 価値と方策の組合せ

Actor（方策 π）と Critic（価値関数 V or Q）を同時学習。分散低減効果。A2C、A3C、DDPG、TD3、SAC、PPO の基礎。

§1.9 PPO — Proximal Policy Optimization

OpenAI 2017。方策更新を クリップ で制限し急変動を防ぐ。安定で簡易、現代RLの主力。ChatGPTのRLHFでも使用された（最近はDPOへ移行）。

LCLIP(θ) = E[min(r(θ)A, clip(r(θ), 1-ε, 1+ε) A)]
　 r(θ) = πθ(a|s) / πold(a|s)　 (重要度比)
　 A：アドバンテージ A(s,a) = Q(s,a) - V(s)

§1.10 探索 vs 活用

手法説明
ε-greedyεの確率でランダム、1-εで最良Q行動
Boltzmann（Softmax）Q値に応じた確率的選択
UCB不確実性ボーナス、バンディットで標準
Thompson Samplingベイズ的、posterior からサンプル
Curiosity / RND内発的動機、スパース報酬向け
SECTION TWO

応用

AlphaGo、AlphaZero、MuZero、RLHF/DPOの4節。

応用

§2.1 AlphaGo（2016）

Silver et al.、DeepMind。人間プロ棋譜での教師あり学習＋自己対戦RL＋MCTS（モンテカルロ木探索）。2016年に世界トップ棋士イ・セドル氏を4-1で破った歴史的事件。

§2.2 AlphaZero（2017）— 人間データ不要

囲碁・将棋・チェスをゲームのルールだけから自己対戦のみでトップ性能に。汎用RL+MCTSアーキテクチャの威力。

§2.3 MuZero（2020）— モデルベース

環境モデル自体も学習。プランニング+RL の統合。Atari、囲碁、将棋、チェスすべて単一フレームワーク。

§2.4 RLHF と DPO — LLM 人間嗜好調整

RLHF（Reinforcement Learning from Human Feedback）：人間ラベラーがモデル出力をランク付け → 報酬モデル学習 → PPO で LLM をファインチューン。InstructGPT、ChatGPT、Claude、Geminiで使用。DPO（Direct Preference Optimization、Rafailov 2023）：報酬モデルなしで直接嗜好データから最適化、より安定。Llama、Mistral などで採用。

SECTION THREE

ハンズオン

Q-Learning、DQN、PPO、Gymnasium、RLHF概念の5節。

ハンズオン

§3.1 Q-Learning スクラッチ

import numpy as np import gymnasium as gym env = gym.make('FrozenLake-v1', is_slippery=False) n_states, n_actions = env.observation_space.n, env.action_space.n Q = np.zeros((n_states, n_actions)) alpha, gamma, epsilon = 0.1, 0.99, 0.1 for ep in range(5000): s, _ = env.reset() done = False while not done: # ε-greedy a = env.action_space.sample() if np.random.rand() < epsilon else Q[s].argmax() s2, r, done, _, _ = env.step(a) Q[s, a] += alpha * (r + gamma * Q[s2].max() - Q[s, a]) s = s2 print("学習済Q-table:", Q)

§3.2 DQN（簡略版）

import torch import torch.nn as nn from collections import deque import random class QNet(nn.Module): def __init__(self, obs_dim, n_act): super().__init__() self.net = nn.Sequential( nn.Linear(obs_dim, 128), nn.ReLU(), nn.Linear(128, n_act)) def forward(self, x): return self.net(x) qnet = QNet(4, 2) # CartPole target_net = QNet(4, 2); target_net.load_state_dict(qnet.state_dict()) opt = torch.optim.Adam(qnet.parameters(), lr=1e-3) buffer = deque(maxlen=10000) env = gym.make('CartPole-v1') for ep in range(500): s, _ = env.reset() done = False while not done: eps = max(0.01, 1.0 - ep/300) a = env.action_space.sample() if random.random() < eps else qnet(torch.FloatTensor(s)).argmax().item() s2, r, term, trunc, _ = env.step(a); done = term or trunc buffer.append((s, a, r, s2, done)) if len(buffer) >= 64: batch = random.sample(buffer, 64) # Bellman 更新（Target Network使用） # ... 詳細省略 s = s2 if ep % 10 == 0: target_net.load_state_dict(qnet.state_dict())

§3.5 RLHF の概念フロー

# Step 1: SFT (Supervised Fine-Tuning) # 人間のデモデータで LLM を初期ファインチューン sft_model = finetune(base_llm, demos) # Step 2: Reward Model 学習 # 人間ラベラーが2つの応答を比較ランク付け # 報酬モデル：r(prompt, response) を学習 rm = train_reward_model(pairs_with_preference) # Step 3: PPO で LLM を強化学習 for _ in range(num_iters): prompts = sample(batch) responses = sft_model.generate(prompts) rewards = rm.score(prompts, responses) # KL正則化：オリジナルから離れすぎ防止 kl = kl_div(sft_model, base_llm) loss = -mean(rewards) + beta * kl update_ppo(sft_model, loss) # これが ChatGPT、Claude、Gemini の人間嗜好調整の核 # 最近は DPO で簡略化（報酬モデル不要）
SECTION FOUR

シミュレーション

探索vs活用、AlphaGo成功、RLHFの3節。

シミュレーション

SIM 1 — ε-greedy のεと性能

>> CartPole で 500 エピソード学習、εを変化

ε=0.0 (探索なし): 最終報酬=45（局所最適）
ε=0.05　　　　　 : 最終報酬=187
ε=0.1　　　　　　: 最終報酬=475（ほぼ最大）
ε=0.3　　　　　　: 最終報酬=320（探索しすぎ）
ε=1.0 (ランダム): 最終報酬=21

>> 適切な探索率が重要。実用では学習進行に従い ε を減衰
>> ε-greedy schedule: 1.0 → 0.01 を 1000エピソードで線形減衰

SIM 2 — AlphaGo の進化

>> 各バージョンの強さ（Elo）

AlphaGo Fan (2015)　 : Elo 3144（プロ初挑戦）
AlphaGo Lee (2016)　 : Elo 3739（イ・セドル氏 4-1）
AlphaGo Master(2017): Elo 4858（柯潔氏 3-0）
AlphaGo Zero (2017) : Elo 5185（人間データなし）
AlphaZero (2017)　　 : Elo 5300+（囲碁・将棋・チェス）

>> 自己対戦RLだけで人間を超え、さらに進化
>> 教師あり学習の限界を強化学習が突破した象徴

SIM 3 — RLHF が LLM を変えた

>> InstructGPT (2022): GPT-3 + RLHF

[GPT-3 (ベース、175B)]
　 ヘルプフル評価: 30%
　 有害応答率: 18%

[InstructGPT (1.3B + RLHF)]
　 ヘルプフル評価: 70%（GPT-3の100倍小さいが上回る）
　 有害応答率: 8%

>> RLHF が LLM の品質を劇的向上
>> ChatGPT (2022) で大衆化、AI ブーム引火
>> 最近は DPO で簡略化（報酬モデル不要）
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
MDP の5要素（S, A, P, R, γ）の γ は何か。
① 学習率　 ② 割引率（将来報酬の現在価値）　 ③ 探索率　 ④ ノイズ
ANSWER ② γ ∈ [0,1)。0.99 が典型。1なら無限地平の累積発散、0なら短期報酬のみ重視。
PROBLEM 2
Q-Learning がオフポリシーである理由は。
① 学習しない　 ② 行動方策と異なる方策（最適）を学習可能、更新で max を使う　 ③ オンライン　 ④ モデルベース
ANSWER ② Q(s,a) ← Q + α[r + γ max Q(s', a') - Q]。max を使うので実際の次行動と独立。SARSAはオンポリシー。
PROBLEM 3
DQN（DeepMind 2015）の核心的貢献は。
① 教師あり　 ② Experience Replay + Target Network で深層NN+RL を安定化、Atari 49ゲーム人間超え　 ③ 教師なし　 ④ クラスタリング
ANSWER ② Mnih et al. 2015 Nature。これが Deep RL 革命の始まり。
PROBLEM 4
AlphaGo の構成要素として正しいのは。
① 教師あり学習のみ　 ② Policy Net + Value Net + MCTS + 自己対戦RL　 ③ Q-tableのみ　 ④ ランダム
ANSWER ② 人間棋譜での教師あり初期化＋自己対戦RL強化＋モンテカルロ木探索による推論時計画。2016年世界トップ棋士を破る。
PROBLEM 5
AlphaZero が AlphaGo と異なる点は。
① 教師データ不要、純粋自己対戦のみ　 ② より小さなNN　 ③ MCTS不使用　 ④ 教師あり強化
ANSWER ① ルールだけからゼロベース自己対戦、人間データ完全不要。囲碁・将棋・チェスすべて同一アーキテクチャでトップ性能。
PROBLEM 6
PPO（Proximal Policy Optimization）の特徴は。
① クリップ損失で方策更新を安定化、実装簡単で汎用性高い　 ② オンポリシーのみ　 ③ Q-tableのみ　 ④ ランダム
ANSWER ① OpenAI 2017。L^CLIP = E[min(rA, clip(r, 1-ε, 1+ε)A)]。RLHFの主力アルゴリズム、現代RL標準。
PROBLEM 7
RLHF（ChatGPT/Claudeで使用）の3ステップは。
① SFT → 報酬モデル → PPO　 ② 教師あり → 教師なし → 強化　 ③ 事前 → 中間 → 微調整　 ④ Pretrain → Fine-tune → Distill
ANSWER ① Supervised Fine-Tuning → Reward Model（人間ランクで学習）→ PPO で報酬最大化。InstructGPT (2022) で確立。
PROBLEM 8
DPO（Direct Preference Optimization、2023）の利点は。
① 計算重い　 ② 報酬モデル不要で嗜好データから直接最適化、PPOより安定で簡単　 ③ 教師あり不要　 ④ 必ず劣る
ANSWER ② Rafailov 2023。報酬モデル+PPOの2段階を1段階に。Llama-2-Chat、Mistral、Zephyr などで採用。最近のRLHF主流。
PROBLEM 9
価値関数 V(s) と Q(s,a) の違いは。
① 同じ　 ② V は状態の価値、Q は(状態,行動)の価値　 ③ V は数値、Q は関数　 ④ V は離散
ANSWER ② V(s) = E[累積報酬|s]、Q(s,a) = E[累積報酬|s,a]。関係：Vπ(s) = Σ π(a|s) Qπ(s,a)。
PROBLEM 10
Bellman 最適方程式 V*(s) = max_a Σ P(s'|s,a)[R + γV*(s')] が示すのは。
① 最適価値の再帰関係　 ② ランダム　 ③ 教師ラベル　 ④ 必ず線形
ANSWER ① 最適価値は『最善の次行動の期待値』として再帰的に定義。Value Iteration、DQN学習の基礎。
PROBLEM 11
ε-greedy の役割は。
① 学習率　 ② 探索と活用のバランス（εでランダム、1-εで最良Q行動）　 ③ 正則化　 ④ クラスタ
ANSWER ② 探索（未知行動を試す）と活用（既知の良い行動を取る）のトレードオフ。実用では学習進行で ε を減衰。
PROBLEM 12
Actor-Critic アーキテクチャは何の組合せか。
① 教師あり+教師なし　 ② 方策（Actor）+ 価値関数（Critic）　 ③ クラスタ+回帰　 ④ DQN+SARSA
ANSWER ② Actor が行動、Critic が状態を評価。分散低減で安定学習。A2C/A3C/PPO/SAC の基礎。
PROBLEM 13
『マルコフ性』とは。
① 過去履歴を全て参照　 ② 次状態は現在状態と行動のみに依存（過去履歴不要）　 ③ 必ず連続　 ④ 線形
ANSWER ② P(s_{t+1}|s_t, a_t) = P(s_{t+1}|s_t, a_t, history)。POMDP（部分観測）では緩和される。
PROBLEM 14
スパース報酬問題への対処として正しいのは。
① 学習率を上げる　 ② Reward Shaping、Curiosity-driven exploration（RND, ICM）、HER　 ③ 教師あり化　 ④ クラスタ
ANSWER ② 報酬がほとんど0のタスク（Montezuma's Revenge等）で重要。内発的動機付け（好奇心）、後付け目標（HER）で対処。
PROBLEM 15
モデルベース RL とモデルフリー RL の違いは。
① 同じ　 ② モデルベースは環境モデル(P, R)を学習・利用、モデルフリーはサンプルから直接学習　 ③ 教師あり vs 教師なし　 ④ 速度
ANSWER ② モデルベース（MuZero、Dreamer）はサンプル効率が良いがモデル誤差影響。モデルフリー（DQN、PPO）はシンプル、サンプル多。
PROBLEM 16
『継続的なエピソードでは γ < 1 が必要』な理由は。
① 計算簡単化　 ② 無限和の収束保証、近未来優先　 ③ ランダム性　 ④ 単純化のみ
ANSWER ② γ=1 だと累積和が発散。γ<1で幾何級数として収束。0.9〜0.999が典型。
PROBLEM 17
MuZero（DeepMind 2020）の特徴は。
① 教師あり　 ② 環境モデル自体も学習、ルール未知のAtariでもMCTS可能　 ③ Q-table　 ④ ランダム
ANSWER ② AlphaZero は環境ルール既知、MuZero はルールも学習。汎用性大幅向上。
PROBLEM 18
SAC（Soft Actor-Critic）が連続行動空間で強い理由は。
① 離散のみ　 ② 最大エントロピー原理で探索促進、決定的方策より安定　 ③ Q-tableのみ　 ④ 教師あり
ANSWER ② Haarnoja 2018。報酬最大化＋エントロピー最大化で探索維持。ロボット制御などで人気。
PROBLEM 19
ChatGPT のような LLM チャットボットの応答品質を人間嗜好に合わせる手法は。
① 単なる教師あり　 ② RLHF（または DPO）　 ③ クラスタリング　 ④ 線形回帰
ANSWER ② RLHF（InstructGPT 2022）。人間嗜好を反映する強化学習で、有害性削減・有用性向上。最近は DPO で簡略化。
PROBLEM 20
強化学習の典型的応用として誤りはどれか。
① ゲーム（AlphaGo, AlphaStar）　 ② ロボット制御　 ③ LLM の人間嗜好調整　 ④ 静的な画像分類
ANSWER ④ 画像分類は典型的に教師あり。RLは『連続的意思決定』『報酬最大化』が要。ゲーム、ロボット、自動運転、推薦、RLHFが代表。
