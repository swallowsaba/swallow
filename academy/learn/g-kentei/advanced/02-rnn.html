章14 — RNN・LSTMChapter 14 — Recurrent Neural Networks

画像はCNN、では系列データ（文章、音声、時系列）は？ ── 答えは RNN。隠れ状態を時間方向に伝播させることで、文脈を保持しながら系列を処理します。本章では、素朴RNNから、BPTT、勾配消失問題、それを解決した LSTM・GRU、双方向RNN、Seq2Seq、Attention（次章Transformerへの架け橋）、Beam Search、NMT までを5部構成で扱います。

G-KENTEI · ADVANCED · CHAPTER 14

時間を、ニューラルネットに
組み込む方法。

RNNは『隠れ状態 h_t』を導入することで時系列を処理します。h_t は『過去の要約』、各時刻で更新され次に伝播する。素朴な構造は勾配消失で長距離依存に弱く、LSTMが『ゲート』機構でこれを解決。やがてAttention/Transformerに主役を譲りますが、その思想は今も現代AIに息づいています。

SECTION ONE

基本

RNN、BPTT、勾配消失、LSTM、GRU、BiRNN、Seq2Seq、Attention、Beam Search、NMTの10節。

基本

§1.1 RNN — 隠れ状態で時間を扱う

h_t = tanh(W_h h_{t-1} + W_x x_t + b)
y_t = W_y h_t

すべての時刻で同じパラメータを共有
任意長の系列を扱える
FIG 14.1 ／ RNNを時間方向に展開した図。同じ重みパラメータを各時刻で共有する。隠れ状態 h が『過去の要約』として伝播する。

§1.2 BPTT — 時間方向の逆伝播

Backpropagation Through Time。RNNを展開した計算グラフ上で連鎖律で勾配を計算。系列長 T が長いほど計算・メモリが増加。実装では Truncated BPTT（一定長で切る）が使われる。

§1.3 勾配消失・爆発の問題

∂L/∂h₁ ∝ Π_{t=1}^{T-1} (∂h_{t+1}/∂h_t) ∝ W^T (tanh' の積)

W のスペクトル半径<1：指数的に減衰（勾配消失）→ 長期依存学べない
W のスペクトル半径>1：指数的に増大（勾配爆発）→ NaN

対処：Gradient Clipping、LSTM/GRU、直交初期化、層正規化

§1.4 LSTM — ゲート機構の発明

Hochreiter & Schmidhuber 1997。セル状態 c_t を別途持ち、3つのゲートで情報の流れを制御：

忘却ゲート f_t = σ(W_f [h_{t-1}, x_t] + b_f)
入力ゲート i_t = σ(W_i [h_{t-1}, x_t] + b_i)
出力ゲート o_t = σ(W_o [h_{t-1}, x_t] + b_o)
候補値 c̃_t = tanh(W_c [h_{t-1}, x_t] + b_c)
セル状態 c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t
隠れ状態 h_t = o_t ⊙ tanh(c_t)

セル状態の伝播は加算ベースで勾配消失が緩和。長期依存（数百ステップ）を扱える。

§1.5 GRU — LSTMの簡略版

Cho 2014。リセットゲートと更新ゲートの2つだけ。LSTMより少ないパラメータで同程度の性能を出すことが多い。

§1.6 双方向RNN（BiRNN/BiLSTM）

前向きと後向きの隠れ状態を結合。文脈を両方向から取得。分類・タグ付けでは効果絶大、ただし生成タスクは未来情報が使えないので不可。

§1.7 Seq2Seq — 系列→系列の生成

Sutskever 2014、Google翻訳の基礎。エンコーダ（入力系列を固定長ベクトルに）+ デコーダ（そのベクトルから出力系列を生成）。機械翻訳、要約、対話の枠組み。

§1.8 Attention — RNN時代の革命

Bahdanau 2015。エンコーダのすべての時刻 h_i にデコーダがその時々で重み α_t,i を付けて参照。固定長ベクトル経由のボトルネックを解消し、長文翻訳の品質が劇的向上。Transformer の基盤となる発想。

§1.9 Beam Search

系列生成で『一番確率高い1個』を取る Greedy より、『上位 k 候補を保持して並列展開』する Beam Search が翻訳・要約で標準。BLEU で2〜3ポイント改善。

§1.10 ニューラル機械翻訳（NMT）

2014〜2016：RNN+Attention の NMT が統計的機械翻訳を圧倒、Google 翻訳が全面採用（2016）。2017 で Transformer に置換。

SECTION TWO

応用

系列ラベリング、時系列予測、音声認識、現代的位置の4節。

応用

§2.1 系列ラベリング（NER、POS）

固有表現抽出、品詞タグ付けで Bi-LSTM + CRF が長く標準。各単語に対しタグを予測。BERT 登場までは最強。

§2.2 時系列予測

株価、需要、気象予測。LSTM/GRU、最近は Temporal Fusion Transformer、N-BEATS、Informer、PatchTST 等の Transformer 系。古典統計（ARIMA）との比較も重要。

§2.3 音声認識

2010s 後半まで DeepSpeech（RNN+CTC）が主流。2020 以降 Conformer（Transformer + Conv）、Whisper（OpenAI 2022）が SOTA に。

§2.4 RNN の現代的位置

NLPでは Transformer に座を譲った。しかし、(a) リアルタイム逐次処理（音声、ストリーミング）、(b) 線形メモリ（Mamba/SSM）、(c) 軽量モデル（モバイル、IoT）では RNN/SSM 系統が復活。2023〜2024 の Mamba、RWKV 等は『RNN の系譜』。

SECTION THREE

ハンズオン

PyTorch RNN/LSTM、文字生成、機械翻訳、時系列、Bi-LSTMの5節。

ハンズオン

§3.1 PyTorch で LSTM

import torch.nn as nn class LSTMClassifier(nn.Module): def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes): super().__init__() self.embed = nn.Embedding(vocab_size, embed_dim) self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True) self.fc = nn.Linear(hidden_dim, num_classes) def forward(self, x): x = self.embed(x) _, (h, _) = self.lstm(x) return self.fc(h[-1])

§3.2 文字レベル言語モデル

class CharLM(nn.Module): def __init__(self, vocab_size, hidden=256): super().__init__() self.embed = nn.Embedding(vocab_size, hidden) self.lstm = nn.LSTM(hidden, hidden, num_layers=2, batch_first=True) self.fc = nn.Linear(hidden, vocab_size) def forward(self, x, hidden=None): x = self.embed(x) out, hidden = self.lstm(x, hidden) return self.fc(out), hidden # 学習：次の文字予測（自己教師あり） # 生成：開始文字から逐次サンプリング

§3.3 Seq2Seq機械翻訳

class Seq2Seq(nn.Module): def __init__(self, src_vocab, tgt_vocab, hidden=256): super().__init__() self.encoder = nn.LSTM(hidden, hidden, batch_first=True) self.src_embed = nn.Embedding(src_vocab, hidden) self.tgt_embed = nn.Embedding(tgt_vocab, hidden) self.decoder = nn.LSTM(hidden, hidden, batch_first=True) self.fc = nn.Linear(hidden, tgt_vocab) def forward(self, src, tgt): _, (h, c) = self.encoder(self.src_embed(src)) out, _ = self.decoder(self.tgt_embed(tgt), (h, c)) return self.fc(out)

§3.4 時系列予測（株価風）

class TSPredictor(nn.Module): def __init__(self, n_features, hidden=64): super().__init__() self.lstm = nn.LSTM(n_features, hidden, batch_first=True) self.fc = nn.Linear(hidden, 1) def forward(self, x): out, _ = self.lstm(x) return self.fc(out[:, -1]) # 最終時刻のみ予測

§3.5 双方向LSTM

# BiLSTM：bidirectional=True で前後両方向 bilstm = nn.LSTM(128, 64, bidirectional=True, batch_first=True) # 出力は hidden*2 = 128 次元 # NER, POS タグ付け、文分類で標準（BERT 以前）
SECTION FOUR

シミュレーション

勾配消失/爆発、LSTMゲート、Attentionの3節。

シミュレーション

SIM 1 — RNN の勾配消失

>> 100ステップRNN、最終損失から入力側への勾配ノルム

[Vanilla RNN]
t=100:　 ||grad|| = 0.42
t= 80:　 ||grad|| = 0.024
t= 60:　 ||grad|| = 0.00103
t= 40:　 ||grad|| = 3.8e-5
t= 20:　 ||grad|| = 1.1e-6 ← ほぼ0
t= 1:　 ||grad|| = 2.3e-8
>> 入力側の情報がほぼ伝わらない

[LSTM]
t=100:　 ||grad|| = 0.51
t= 50:　 ||grad|| = 0.087
t= 1:　 ||grad|| = 0.0042
>> セル状態経由で勾配が保存される

SIM 2 — LSTMゲートの役割

>> 文字列「I am from Japan and I speak ___」での予測

入力 'I am':　 　 入力ゲート開、忘却ゲート閉
入力 'from Japan':忘却ゲート部分閉、入力ゲート開
　　　　　　　　 → 'Japan' を強く記憶
入力 'and I speak':忘却ゲート開（古い文脈忘れ気味）
出力時点:　　　 'Japan' の情報から → 'Japanese' を予測

>> ゲートが学習で『何を覚えて何を忘れるか』を獲得

SIM 3 — Attentionの効果

>> 50語の英文を仏訳、固定ベクトル経由 vs Attention

[Vanilla Seq2Seq（固定ベクトル）]
　 BLEU: 18.3
　 長文で品質激減（情報ボトルネック）

[Seq2Seq + Attention]
　 BLEU: 27.5　 ← +50%
　 長文でも品質維持
　 attention map で対応関係を可視化可能

>> Attentionが固定ベクトルボトルネックを解消
>> やがてRNN自体が不要に → Transformer
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
RNNが MLP より系列データに適する理由は。
① 計算高速　 ② 隠れ状態 h で過去の文脈を保持、可変長系列を扱える　 ③ パラメータ多い　 ④ 教師なし
ANSWER ② h_t = f(h_{t-1}, x_t) で過去情報を伝播、同じパラメータを全時刻で共有。
PROBLEM 2
BPTT（Backprop Through Time）の問題は。
① 不要　 ② 長系列で計算・メモリ増、勾配消失/爆発　 ③ 教師なし限定　 ④ 学習しない
ANSWER ② 系列長Tに比例して計算が増え、勾配が指数的に減衰または爆発。Truncated BPTT、Gradient Clipping、LSTM/GRU で対処。
PROBLEM 3
勾配消失問題で『記憶できる時間』の典型は。
① 1ステップ　 ② Vanilla RNNでは 10〜20ステップが限界　 ③ 無限　 ④ 100万ステップ
ANSWER ② tanh の勾配積が指数減衰。LSTM/GRU でこれを数百ステップまで拡大。
PROBLEM 4
LSTM のセル状態 c_t の特徴は。
① 隠れ状態と同じ　 ② 加算更新で勾配が保存され、長期依存を学べる　 ③ 出力のみ　 ④ ノイズ
ANSWER ② c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t の加算ベース更新が勾配消失を緩和。これが LSTM の核心。
PROBLEM 5
LSTMの3つのゲートとして正しいのは。
① 入力、Conv、Pool　 ② 忘却（forget）、入力（input）、出力（output）　 ③ 順、逆、双方向　 ④ 学習、推論、評価
ANSWER ② Hochreiter & Schmidhuber 1997。Sigmoidゲートで情報の流れを制御。
PROBLEM 6
GRU がLSTMより優れる点は。
① 必ず高精度　 ② パラメータが少なく、計算高速、性能はほぼ同等　 ③ ゲートなし　 ④ 双方向のみ
ANSWER ② Cho 2014。リセットゲートと更新ゲートだけで2ゲート構造。実装簡素、性能はLSTMと拮抗。
PROBLEM 7
双方向RNN（Bi-LSTM）が使えない典型タスクは。
① 文分類　 ② NER　 ③ 言語モデル（次トークン予測、未来情報禁止）　 ④ 感情分析
ANSWER ③ 生成・予測タスクは未来情報を使えない（カンニングになる）。分類・タグ付けには Bi-LSTM が長く標準。
PROBLEM 8
Seq2Seq モデルの構造は。
① 1個のRNN　 ② エンコーダ（入力→ベクトル）+ デコーダ（ベクトル→出力）　 ③ CNN のみ　 ④ MLP のみ
ANSWER ② Sutskever 2014。機械翻訳、要約、対話の基本枠組み。固定長ベクトルがボトルネック、Attentionで解消。
PROBLEM 9
Attention（Bahdanau 2015）が解決した Seq2Seq の課題は。
① 学習速度　 ② 固定長ベクトルのボトルネック、長文で性能劣化　 ③ メモリ消費　 ④ 過学習
ANSWER ② エンコーダ全時刻の隠れ状態にデコーダが重み付き参照。長文翻訳の品質が劇的改善。Transformer の母。
PROBLEM 10
Beam Search の役割は。
① 学習加速　 ② 系列生成で上位 k 候補を保持して探索、Greedy より高品質　 ③ 正則化　 ④ クラスタリング
ANSWER ② 翻訳・要約で標準。beam size 4〜8 で BLEU 2〜3 ポイント改善。LLM 生成では nucleus sampling 等の方が人気。
PROBLEM 11
Truncated BPTT の目的は。
① 精度向上　 ② 長系列で計算・メモリを節約、一定ステップで切って勾配計算　 ③ 学習なし　 ④ 過学習防止
ANSWER ② 全系列のBPTTは不可能なので、典型 35〜200 ステップで切る。長期依存は LSTM/GRU の力で。
PROBLEM 12
勾配爆発の対処として有効なのは。
① 学習率を上げる　 ② Gradient Clipping（勾配ノルムを閾値で制限）　 ③ パラメータ削除　 ④ Sigmoid を使う
ANSWER ② torch.nn.utils.clip_grad_norm_ で実装。閾値1.0〜5.0が典型。Transformer 学習でも常用。
PROBLEM 13
音声認識で 2020〜2026 の標準は。
① 古典GMM-HMM　 ② Whisper、Conformer など Transformer 系　 ③ Bi-LSTM のみ　 ④ K-means
ANSWER ② Whisper（OpenAI 2022）が多言語SOTA。Conformer（Google）はTransformer+Convの組合せでオンライン認識に強い。
PROBLEM 14
時系列予測で Transformer 系の代表例は。
① ResNet　 ② Informer、PatchTST、Temporal Fusion Transformer　 ③ K-NN　 ④ ARIMA
ANSWER ② 長系列・多変量を効率良く扱える。最近の研究では『単純MLPでもTransformerと同等』との指摘も（DLinear等）。
PROBLEM 15
2023 年以降、『RNN の系譜』を継ぐ新世代モデルは。
① CNN　 ② Mamba（SSM）、RWKV、Linear Attention　 ③ MLP のみ　 ④ K-NN
ANSWER ② Selective SSM (Mamba 2023) が Transformer に対抗、線形メモリ・長文脈で優位。RWKV は RNN ライクな並列学習。
PROBLEM 16
『Teacher Forcing』の意味は。
① 強制学習　 ② Seq2Seq学習時に、デコーダへの入力に過去のモデル出力ではなく『正解』を与える手法　 ③ 教師なし　 ④ クラスタ
ANSWER ② 訓練効率を上げるが、推論時の自己回帰生成と乖離する『Exposure Bias』問題も。Scheduled Sampling が緩和策。
PROBLEM 17
BiLSTM-CRF が長く標準だったタスクは。
① 画像分類　 ② NER（固有表現抽出）、品詞タグ付けなどの系列ラベリング　 ③ 強化学習　 ④ K-means
ANSWER ② BiLSTM が文脈、CRF がラベル間依存関係を学習。BERT 登場（2018）まで最強。今もリソース制約環境で実用。
PROBLEM 18
機械翻訳の評価指標として標準的なのは。
① Accuracy　 ② BLEU、METEOR、chrF、最近は COMET、BERTScore　 ③ MSE　 ④ R²
ANSWER ② BLEU が古典標準、ただし表面文字列マッチで限界あり。LLM 時代は BERTScore、COMET 等の埋め込みベース指標が好まれる。
PROBLEM 19
2014〜2017 の機械翻訳の主流は。
① ルールベース　 ② RNN+Attentionのニューラル機械翻訳（NMT）、Google翻訳が全面採用（2016）　 ③ Transformer　 ④ LLM
ANSWER ② 2017 年に Transformer 登場し急速に置換。現在は LLM が機械翻訳でも事実上の SOTA。
PROBLEM 20
CTC（Connectionist Temporal Classification）の用途は。
① 画像分類　 ② 音声認識・手書き認識など、入力と出力の長さが異なる・整列がないタスクの損失関数　 ③ 強化学習　 ④ クラスタ
ANSWER ② Graves 2006。アラインメントを暗黙にマージしながら学習。RNN+CTC（DeepSpeech）が音声認識の標準だった。

章のまとめ

RNN、BPTT、勾配消失/爆発、LSTM、GRU、Bi-RNN、Seq2Seq、Attention、Beam Search、NMT、CTC を扱いました。応用で系列ラベリング、時系列予測、音声認識、現代的位置（Mamba/RWKV の復活）。ハンズオンで PyTorch LSTM、文字レベル LM、Seq2Seq、時系列予測、Bi-LSTM。シミュレーションで勾配消失、ゲート役割、Attention効果を確認。次章は Transformer — 現代AIを定義した革命的アーキテクチャへ。
