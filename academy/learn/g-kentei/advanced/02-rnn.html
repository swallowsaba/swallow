応用 章2 — RNN・LSTM・GRUSequence Models

言語、音声、時系列などの 順序のあるデータ を扱うために設計されたNN。Transformer 登場前の標準であり、いまも軽量モデルや組込み用途で現役。勾配消失問題とゲート機構による克服 がこの章の核。

§1 / WHY系列データと NN

言語・音声・時系列・株価などは 順序が意味を持つ データ。MLP に「I love you」を 3 単語の独立な入力として与えると順序情報が失われる。CNN は局所相関は捉えるが 可変長 に弱い。

解決策：過去の情報を内部状態として持ち越す NN ＝ RNN。

§2 / RNNRNN の基本

各時刻 t で、入力 xₜ と前時刻の 隠れ状態 hₜ₋₁ から、新しい隠れ状態 hₜ を計算：

hₜ = tanh(W_xh · xₜ + W_hh · hₜ₋₁ + b)
yₜ = W_hy · hₜ + c

重要なのは 同じ重み行列を全時刻で共有 すること。これにより可変長系列を扱える。

FIG 2.1 RNN の時間方向展開図。各時刻の RNN ユニットは同じ重みを使う。隠れ状態 h が「過去の記憶」として時系列に流れていく。

§3 / UNROLL時間方向の展開と BPTT

RNN を時間方向に展開すると、巨大な MLP に見える。BPTT（Backpropagation Through Time）= 普通の逆伝播を時間方向にも適用する。

§4 / VANISH勾配消失問題（再訪）

長い系列だと、勾配が時間方向に何十回も掛け算される。tanh の微分は ≤ 1、Sigmoid は ≤ 0.25。指数的に減衰して下層（過去）に伝わらない。

これが「長期依存性が学べない」という RNN の根本問題。例：「私は フランス で生まれ育ち、… 数十単語後 …、流暢に フランス語 を話す」のような長距離依存。

§5 / LSTMLSTM とゲート機構

1997年、Hochreiter & Schmidhuber が考案。セル状態（cell state）c と 3つのゲートで、情報の流れを学習的に制御する。

FIG 2.2 LSTM の構造（簡略）。セル状態（赤い線）が「高速道路」として情報を流す。3つのゲート（忘却・入力・出力）が情報の出入りを制御する。

4つの式：

fₜ = σ(W_f · [hₜ₋₁, xₜ])  # 忘却ゲート
iₜ = σ(W_i · [hₜ₋₁, xₜ])  # 入力ゲート
c̃ₜ = tanh(W_c · [hₜ₋₁, xₜ])  # 候補値
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ  # セル更新
oₜ = σ(W_o · [hₜ₋₁, xₜ])  # 出力ゲート
hₜ = oₜ ⊙ tanh(cₜ)

なぜ勾配消失を緩和できるか：セル状態 cₜ = fₜ ⊙ cₜ₋₁ + ... という形は、ResNet の残差接続と同様、勾配がセル状態を介して直接過去に流れる 経路を提供する。

§6 / GRUGRU（Gated Recurrent Unit）

2014年、Cho らが提案。LSTM の簡略版。ゲートを 3 つから 2 つに削減、セル状態と隠れ状態を統合。

  • 更新ゲート（Update gate）
  • リセットゲート（Reset gate）

パラメータ数が LSTM より少なく、計算が速い。実務では LSTM とほぼ同等の性能 なので、軽量化のため GRU を使うことも多い。

§7 / SEQ2SEQSeq2Seq とアテンション

Seq2Seq（Sutskever et al., 2014）：エンコーダ RNN で入力系列を1つの固定長ベクトルに圧縮し、デコーダ RNN で出力系列を生成。機械翻訳のEnd-to-Endを実現。

問題：長い入力を1つのベクトルに圧縮するボトルネック → 性能が頭打ち。

アテンション機構（Bahdanau, 2014）：デコーダの各時刻で、エンコーダの全時刻の出力を加重平均して参照。これが後の Transformer の中核となる。

§8 / SUMMARY章のまとめ

1系列データは順序が重要 → 隠れ状態を持ち越す RNN
2RNN は全時刻で同じ重みを共有（可変長対応）
3BPTT で時間方向にも逆伝播
4勾配消失問題が長期依存性を阻害
5LSTM：セル状態と3ゲート（忘却・入力・出力）
6セル状態経由で勾配が直接流れる → 長期依存学習可能
7GRU：LSTM の簡略版、2ゲート、軽量
8Seq2Seq：エンコーダ・デコーダで機械翻訳を End-to-End
9アテンション機構が後の Transformer に繋がる
