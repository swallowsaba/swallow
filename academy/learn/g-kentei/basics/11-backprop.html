章11 — バックプロパゲーションChapter 11 — Backpropagation

バックプロパゲーションは 多層NNの学習を可能にした 最重要アルゴリズムです。連鎖律を計算グラフ上で系統的に適用するだけ。PyTorch・TensorFlow・JAX の自動微分の中核でもあります。本章で「勾配がどう流れるか」が見えるようにします。

勾配を、

バックプロパゲーション（誤差逆伝播法）は1986年、Rumelhart, Hinton, Williams によって NN 研究にもたらされました（独立に複数の研究者が発見）。その本質は「連鎖律を計算グラフ上で系統的に適用する」だけ。

§1 / GRAPH計算グラフ

NN を 「ノード（演算）」と「エッジ（テンソル）」のグラフ として表現します。

FIG 11.1 2層 MLP の計算グラフ。順伝播で値を計算、逆伝播で勾配を計算。中間値（h, z₁, ŷ）は両方で再利用される。

§2 / FORWARDフォワードパス（順伝播）

入力 x から損失 L まで、順に値を計算。途中の中間値（活性化）はすべて保存（バックワードで使うため）。

例：2層 MLP の順伝播
z₁ = W₁ x + b₁
h = ReLU(z₁) = max(0, z₁)
z₂ = W₂ h + b₂
ŷ = σ(z₂)（出力 sigmoid）
L = -[y log ŷ + (1-y) log(1-ŷ)]（CE 損失）

§3 / BACKバックワードパス（逆伝播）

出力側から入力側へ、各パラメータについて勾配を計算。連鎖律を機械的に適用：

∂L/∂W₂ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂W₂
∂L/∂W₁ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂h · ∂h/∂z₁ · ∂z₁/∂W₁

共通部分（左から右に並ぶ最初の項）を 再利用 するのが鍵。これにより：

  • 勾配計算コスト = 順伝播の数倍程度
  • パラメータ数が数十億あっても全て一回で計算できる

これがなければ深層学習は計算量的に不可能でした。

3.1 アルゴリズムの本質

  1. 順伝播：入力から出力へ。すべての中間値を保存
  2. 損失計算：L = L(ŷ, y)
  3. 逆伝播：出力側から入力側へ ∂L/∂... を計算（連鎖律）
  4. 更新：θ ← θ - η · ∂L/∂θ

§4 / AUTODIFF自動微分（AD）

バックプロパゲーションを「計算グラフから勾配を機械的に計算する」一般的アルゴリズムにしたもの。PyTorch・TensorFlow・JAX の中核。

  • 順モード AD：入力1つに対する出力多数の場合に効率的
  • 逆モード AD（バックプロパゲーションと同じ）：出力1つ（損失）に対する入力多数（パラメータ）の場合に効率的。深層学習はこちら

§5 / VANISH勾配消失・勾配爆発

FIG 11.2 勾配消失（小さい数を多数かけ続ける）と勾配爆発（大きい数を多数かけ続ける）。どちらも深い NN の学習を妨げる。

原因と対策：

対策効果
ReLU 系活性化正領域で勾配 1。Sigmoid/Tanh のような飽和を回避
He 初期化（ReLU用）／Xavier 初期化（Tanh用）初期勾配のスケールを適切に
残差接続（ResNet）勾配を直接下層に伝える
BatchNorm / LayerNorm各層の入力分布を安定化
勾配クリッピング勾配のノルムを閾値で抑制（特にRNN）

§6 / RESNETResNet：残差接続

2015年、He らが提案した 残差接続 は、深層学習を実質的に「使い物にした」ブレークスルー。

y = F(x) + x

F(x) は普通の層変換。入力 x を出力に直接加える。

FIG 11.3 ResNet の残差ブロック。y = F(x) + x という単純な式が、勾配を F を通らずに直接下層へ 伝える経路を作る。これにより152層、最終的に1000層超の NN が学習可能に。

残差接続による勾配の流れ：

∂L/∂x = ∂L/∂y · ∂y/∂x = ∂L/∂y · (∂F/∂x + 1)

「+ 1」 の項により、F の勾配がゼロに近くても勾配が消えない。これが ResNet を「最重要発明の一つ」と呼ばせる理由。

§7 / TOOLSPyTorch / TensorFlow / JAX

これらのフレームワークの核心は 自動微分エンジン。ユーザは順伝播を書くだけで、逆伝播は自動的に計算されます。

PyTorch動的計算グラフ（define-by-run）。研究で支配的。Meta 開発
TensorFlow静的グラフ（v1）→ Eager Execution（v2）。Google 開発、産業で広く使用
JAX関数型・XLAコンパイル・自動微分。研究で台頭。Google 開発

§8 / SUMMARY章のまとめ

1計算グラフ：演算をノード、テンソルをエッジで表現
2順伝播：入力 → 損失。中間値はすべて保存
3逆伝播：損失 → パラメータ。連鎖律を系統的に適用
4共通部分の再利用で計算コスト = 順伝播の定数倍
5自動微分（AD）：逆モードが深層学習の基盤
6勾配消失：層が深いと小さい数を掛け続けて勾配 → 0
7勾配爆発：大きい数を掛け続けて勾配 → ∞
8対策：ReLU、適切な初期化、ResNet、Normalization、勾配クリップ
9ResNet の残差接続 y = F(x) + x が深層化を可能に
10PyTorch/TF/JAX の核心は自動微分エンジン
