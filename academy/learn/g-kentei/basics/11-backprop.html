G検定 基本編 — 章11

連鎖律を、計算機に。
深層学習の心臓部。

100万のパラメータを持つNNの、各パラメータに対する損失の偏微分を、どうやって計算するのでしょう。素朴に有限差分で計算すれば100万回の順伝播が必要 ── 実用不可能です。逆伝播は、たった1回の順伝播と1回の逆伝播で全パラメータの勾配を一気に計算します。この『計算量を1次オーダーに圧縮する』アイデアが、現代AIを可能にしました。

SECTION ONE

基本

連鎖律・計算グラフ・逆伝播・全結合層・活性化関数・Softmax+CE・SGD・自動微分・勾配消失・計算量の10節。

基本

§1.1 連鎖律 — 微分の核心

y = f(g(h(x))) のとき　 dy/dx = (df/dg)·(dg/dh)·(dh/dx)

NN は f∘g∘h∘... の合成関数。連鎖律で各パラメータへの勾配を計算。
▎ 簡単な例
L = (Wx + b - y_target)² で W に対する勾配は
∂L/∂W = 2(Wx + b - y_target) · x　 (連鎖律の適用)

§1.2 計算グラフ — 順伝播の可視化

各演算をノード、データの流れをエッジとする有向グラフ。順伝播は forward 方向で値計算、逆伝播は backward 方向で勾配計算。

FIG 11.1 ／ 計算グラフ。順伝播で各ノードの値を計算、逆伝播で出力側から連鎖律により各エッジの勾配を計算。これがPyTorch/TensorFlowの内部動作。

§1.3 逆伝播の基本原理

入力 x、出力 L、ある中間量 z があるとき：
　 ∂L/∂x = (∂L/∂z) · (∂z/∂x)

各層について『下流から流れてきた勾配』に『局所微分』を掛けて『上流に流す』

§1.4 全結合層の逆伝播

順伝播：z = Wx + b （x: dᵢₙ, W: dₒᵤₜ×dᵢₙ, z: dₒᵤₜ）

逆伝播：上流勾配 ∂L/∂z = δ が与えられたとき
　 ∂L/∂W = δ · xᵀ　 (外積、形状: dₒᵤₜ×dᵢₙ)
　 ∂L/∂b = δ　　 (そのまま)
　 ∂L/∂x = Wᵀ · δ　 (下流へ流す)

§1.5 活性化関数の逆伝播

関数微分
Sigmoid σ(x)σ(x)(1-σ(x))
Tanh1 - tanh²(x)
ReLUx>0 で 1、x≤0 で 0
Leaky ReLU(α)x>0 で 1、x≤0 で α
GELUΦ(x) + x·φ(x)
SoftmaxJacobian J_ij = pᵢ(δᵢⱼ - pⱼ)

§1.6 Softmax+CE の合成勾配 — 美しい単純化

順伝播：p = softmax(z),　 L = -Σ yᵢ log pᵢ

逆伝播（logit z に対する勾配）：
　 ∂L/∂z = p - y　 (Softmax+CE 合体の場合)

※ Softmax と CE を別々に微分するより圧倒的に簡潔・安定
　 これが NN 分類の計算効率の根拠

§1.7 パラメータ更新（SGD）

θ ← θ - η · ∂L/∂θ

SGD：単純な勾配降下、学習率 η
Momentum：v ← β v + ∂L/∂θ, θ ← θ - η v
Adam：1次・2次モーメントの推定（章4で詳細）
AdamW：Adam + weight decay 分離（LLM学習標準）

§1.8 自動微分（Auto-diff）

モード計算量用途
Forward modeO(入力数)少入力・多出力（Jacobian-vector）
Reverse modeO(出力数)多入力・少出力（NN学習、損失1つ）

NN では reverse mode が標準（パラメータ数 ≫ 出力数 1）。PyTorch, TensorFlow, JAX が実装。

§1.9 勾配消失・勾配爆発

深層NN の連鎖律で、L 層の勾配は
　 ∂L/∂x = (∂z_L/∂x) = J_L · J_{L-1} · ... · J_1

J の固有値が < 1 → 勾配消失（深層で 0 に）
J の固有値が > 1 → 勾配爆発（深層で発散）

対策　 ReLU、適切な初期化、BatchNorm、残差接続、Gradient Clipping、LSTM のセル

§1.10 計算量とメモリ

  • 計算量：順伝播 O(N)、逆伝播 O(N)。合計でも O(N) パラメータ数の定数倍。
  • メモリ：順伝播の中間活性化を全て保存（O(L·B·D)：層数×バッチ×次元）。これが大規模NN のメモリ消費の主因。
SECTION TWO

応用

計算グラフ最適化、Gradient Checkpointing、Mixed Precision、Distributed Training の4節。

応用

§2.1 計算グラフの最適化（Operator Fusion）

『LayerNorm → Linear → GELU』のような連続演算を1つのカーネルに融合（Fused operations）。メモリ転送回数を削減、GPU 計算効率↑。FlashAttention などの最新Transformer実装で重要。

§2.2 Gradient Checkpointing — メモリと計算のトレードオフ

順伝播の活性化を一部のみ保存、逆伝播時に再計算。メモリ O(L)→O(√L)、計算は1.5倍程度。LLM 学習で必須。

§2.3 Mixed Precision（FP16/BF16）

計算は FP16/BF16、重みコピーは FP32 で精度を保つ。GPU メモリ半減、計算 2〜3 倍速。Loss Scaling で勾配のアンダーフロー対策。NVIDIA Apex、PyTorch AMP で標準。

§2.4 分散学習

並列化説明
Data Parallel各GPUに同じモデル、データを分割
Model Parallelモデルを層で分割
Tensor Parallel各層の行列を分割（Megatron）
Pipeline Parallel層をパイプライン実行（GPipe）
ZeROオプティマイザ状態を分散（DeepSpeed）
SECTION THREE

ハンズオン

NumPy逆伝播、PyTorch自動微分、カスタムbackward、勾配チェック、勾配可視化の5節。

ハンズオン

§3.1 NumPyでスクラッチ逆伝播

import numpy as np # 2層NN: x → Linear → ReLU → Linear → Softmax+CE def forward(x, y, W1, b1, W2, b2): z1 = x @ W1 + b1 a1 = np.maximum(0, z1) # ReLU z2 = a1 @ W2 + b2 e = np.exp(z2 - z2.max(axis=1, keepdims=True)) p = e / e.sum(axis=1, keepdims=True) # Softmax loss = -np.log(p[range(len(y)), y]).mean() cache = (x, z1, a1, p, y) return loss, cache def backward(cache): x, z1, a1, p, y = cache N = len(y) # Softmax+CE: dL/dz2 = (p - one_hot(y)) / N dz2 = p.copy() dz2[range(N), y] -= 1 dz2 /= N dW2 = a1.T @ dz2 db2 = dz2.sum(axis=0) da1 = dz2 @ W2.T dz1 = da1 * (z1 > 0) # ReLU derivative dW1 = x.T @ dz1 db1 = dz1.sum(axis=0) return dW1, db1, dW2, db2

§3.2 PyTorch 自動微分

import torch x = torch.tensor([2.0], requires_grad=True) y = x ** 3 + 2 * x ** 2 + 1 # y = x³ + 2x² + 1 y.backward() print(x.grad) # 3·4 + 2·2·2 = 12 + 8 = 20 ✓ # NN学習でも同じ仕組み W = torch.randn(3, requires_grad=True) x = torch.tensor([1., 2., 3.]) y_target = torch.tensor(5.) loss = ((W @ x) - y_target) ** 2 loss.backward() print(f"勾配 W: {W.grad}")

§3.3 カスタム backward 実装

class MyReLU(torch.autograd.Function): @staticmethod def forward(ctx, x): ctx.save_for_backward(x) return x.clamp(min=0) @staticmethod def backward(ctx, grad_output): x, = ctx.saved_tensors grad_input = grad_output.clone() grad_input[x < 0] = 0 return grad_input # 使用 x = torch.randn(5, requires_grad=True) y = MyReLU.apply(x).sum() y.backward()

§3.4 勾配チェック（Gradient Check）

# 解析勾配 vs 数値微分 def numerical_grad(f, x, eps=1e-5): grad = np.zeros_like(x) for i in range(x.size): idx = np.unravel_index(i, x.shape) x[idx] += eps; fp = f(x) x[idx] -= 2*eps; fm = f(x) x[idx] += eps grad[idx] = (fp - fm) / (2*eps) return grad # 解析勾配と比較。差が 1e-6 以下なら正しい analytical = my_backward(x) numerical = numerical_grad(my_forward, x) error = np.max(np.abs(analytical - numerical)) print(f"勾配誤差: {error}")

§3.5 勾配の可視化

# 各層の勾配ノルムを記録 def log_grad_norms(model): return {n: p.grad.norm().item() for n, p in model.named_parameters() if p.grad is not None} # 学習中の勾配ノルムをプロット # 入力側で勾配が小さければ勾配消失、大きければ勾配爆発
SECTION FOUR

シミュレーション

連鎖律手計算、勾配消失観察、Adamの効果の3節。

シミュレーション

SIM 1 — 連鎖律の手計算

>> y = (x² + 3x)² で x=2 のときの dy/dx

合成関数：g(x) = x² + 3x、 y = g²
　 dy/dg = 2g = 2(4+6) = 20
　 dg/dx = 2x + 3 = 7
　 dy/dx = (dy/dg)·(dg/dx) = 20 · 7 = 140

[PyTorch で確認]
　 x = torch.tensor(2.0, requires_grad=True)
　 y = (x**2 + 3*x)**2
　 y.backward()
　 x.grad → 140.0 ✓

>> 連鎖律は単純な掛け算、これを自動化するのが auto-diff

SIM 2 — 勾配消失の観察

>> 20層NN、Sigmoid活性化で各層の勾配ノルム

Layer 20 (出力)　: ||grad|| = 0.952
Layer 18:　　　　: ||grad|| = 0.180
Layer 15:　　　　: ||grad|| = 0.0078
Layer 12:　　　　: ||grad|| = 0.00012
Layer 9:　　　　: ||grad|| = 1.8e-6
Layer 6:　　　　: ||grad|| = 3.0e-8
Layer 3:　　　　: ||grad|| = 5.1e-10
Layer 1 (入力)　: ||grad|| = 8.5e-12

>> 出力から入力で勾配が10億倍小さく
>> ReLU + BatchNorm + 残差接続で解決

SIM 3 — オプティマイザ比較（収束速度）

>> MNIST、3層MLP、20エポック学習

SGD (lr=0.01):　　 Final acc=92.3% (収束遅い)
SGD+Mom (lr=0.01): Final acc=94.8%
Adam (lr=1e-3):　　 Final acc=97.2% (早期収束)
AdamW (lr=1e-3):　 Final acc=97.5% (汎化やや↑)
LION (lr=1e-4):　　 Final acc=97.0% (メモリ半分)

>> Adam系が NN 学習のデフォルト
>> LLM は AdamW、画像は SGD+Momentum も健在
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
逆伝播の数学的核心は。
① 線形回帰　 ② 連鎖律（chain rule）を計算グラフ上で系統的に適用　 ③ ランダム　 ④ クラスタリング
解答・解説を見る
ANSWER ② dy/dx = (dy/du)(du/dx) を多層に適用、計算グラフのbackwardで自動化。
PROBLEM 2
逆伝播を実用化した1986年の論文の著者は。
① Turing　 ② Rumelhart, Hinton, Williams　 ③ Bengio　 ④ LeCun
解答・解説を見る
ANSWER ② Nature論文『Learning representations by back-propagating errors』。多層NNの実用化を可能にした歴史的論文。
PROBLEM 3
100万パラメータNNで全勾配を計算するのに必要な順伝播・逆伝播の回数は。
① 100万回　 ② 順伝播1回 + 逆伝播1回（合計でも O(N) パラメータ数）　 ③ 1000回　 ④ ゼロ
解答・解説を見る
ANSWER ② これが逆伝播の真価。素朴な有限差分なら100万倍のコスト。reverse-mode auto-diff の効率性。
PROBLEM 4
Sigmoid 関数の微分は。
① σ(x)　 ② σ(x)·(1-σ(x))　 ③ 1-σ(x)　 ④ x
解答・解説を見る
ANSWER ② 美しい形。最大値は σ(0)·(1-σ(0)) = 0.25。これが勾配消失の主因。
PROBLEM 5
ReLU の微分は。
① 常に1　 ② x>0で1、x≤0で0　 ③ x　 ④ ランダム
解答・解説を見る
ANSWER ② x=0で厳密には微分不可能だが、実装で0または1を採用。シンプルさが計算効率に貢献。
PROBLEM 6
Softmax + CE の合成勾配が美しい形は。
① p+y　 ② p-y　 ③ p²　 ④ y²
解答・解説を見る
ANSWER ② logit z への ∂L/∂z = (p - y_onehot) / N。シンプル＆数値安定。NN分類の計算効率の根拠。
PROBLEM 7
全結合層 z = Wx + b で、上流勾配 δ = ∂L/∂z から得られる ∂L/∂W は。
① δ・x　 ② δ・xᵀ（外積）　 ③ Wδ　 ④ δ
解答・解説を見る
ANSWER ② 形状 dₒᵤₜ × dᵢₙ。バッチでは Σ δᵢ·xᵢᵀ。∂L/∂b = δ, ∂L/∂x = Wᵀδ。
PROBLEM 8
NN学習で reverse-mode 自動微分が使われる理由は。
① パラメータ数 ≫ 出力数（損失1つ）なので、forward-modeより遥かに効率的　 ② 唯一の方法　 ③ ランダム　 ④ メモリ少ない
解答・解説を見る
ANSWER ① Forward は入力数に、Reverse は出力数に計算量が比例。NN は損失1つなので Reverse 一択。
PROBLEM 9
勾配消失の主因は。
① 学習率　 ② 活性化関数の微分の積が層数で指数的に減衰（Sigmoid/Tanh）　 ③ パラメータ数　 ④ データ量
解答・解説を見る
ANSWER ② Sigmoid 微分最大 0.25、L層で 0.25^L。10層で 1e-6。ReLU、BN、ResNet で解決。
PROBLEM 10
勾配爆発の対処は。
① 学習率を上げる　 ② Gradient Clipping（勾配ノルムを上限でカット）　 ③ 何もしない　 ④ パラメータ増加
解答・解説を見る
ANSWER ② RNN/Transformer で標準。||g|| > threshold なら g ← g · threshold/||g||。閾値 1.0 が典型。
PROBLEM 11
SGD の更新式は。
① θ ← θ + η・∂L/∂θ　 ② θ ← θ - η・∂L/∂θ　 ③ θ ← ∂L/∂θ　 ④ θ ← η・θ
解答・解説を見る
ANSWER ② 勾配方向の逆（descent）に η（学習率）だけ動く。シンプルだが汎化が良いことがある。
PROBLEM 12
逆伝播でメモリを最も消費するのは。
① パラメータ　 ② オプティマイザ状態　 ③ 順伝播の中間活性化（逆伝播で必要）　 ④ 損失値
解答・解説を見る
ANSWER ③ 各層の出力（B × D）を逆伝播のために保存。L 層では O(L·B·D)。これが大規模NN のメモリ消費の主因。
PROBLEM 13
Gradient Checkpointing の効果は。
① 計算高速化　 ② 中間活性化を一部だけ保存し逆伝播で再計算、メモリ O(L)→O(√L)　 ③ 精度向上　 ④ パラメータ削減
解答・解説を見る
ANSWER ② メモリと計算のトレードオフ（計算1.5倍）。LLM 学習で必須。PyTorch torch.utils.checkpoint で実装。
PROBLEM 14
Mixed Precision Training（FP16/BF16）の主な利点は。
① 精度向上　 ② GPU メモリ半減＆計算2〜3倍速、Tensor Coreを活用　 ③ パラメータ削減　 ④ ランダム性
解答・解説を見る
ANSWER ② FP16計算 + FP32重みコピー + Loss Scaling。BF16はダイナミックレンジが広く Loss Scaling 不要。LLM学習で標準。
PROBLEM 15
Data Parallel の動作は。
① 1GPUだけ使う　 ② 各GPUに同じモデルコピー、ミニバッチを分割、勾配をAllReduceで集約　 ③ モデルを分割　 ④ ランダム
解答・解説を見る
ANSWER ② 最も基本的な並列化。PyTorch DDP, Horovod。モデルがGPUに載るサイズなら最初の選択肢。
PROBLEM 16
勾配チェック（Gradient Check）の用途は。
① 学習加速　 ② 解析的に計算した勾配と数値微分を比較し、実装バグを検出　 ③ パラメータ削減　 ④ 必須学習
解答・解説を見る
ANSWER ② カスタム backward を書いたとき必須。差が 1e-7〜1e-5 オーダーなら正しい。学習中は遅いので無効化。
PROBLEM 17
PyTorch で勾配計算を有効化するには。
① 自動で有効　 ② requires_grad=True で作成、または .requires_grad_() で設定　 ③ 不可能　 ④ ランダム
解答・解説を見る
ANSWER ② requires_grad=True の Tensor は計算グラフに含まれ、.backward() で勾配が記録される。.grad 属性で取得。
PROBLEM 18
ZeRO（DeepSpeed）が分散するのは。
① モデルのみ　 ② オプティマイザ状態、勾配、パラメータを分散保持しメモリ削減　 ③ データのみ　 ④ ランダム
解答・解説を見る
ANSWER ② ZeRO-1：オプティマイザ状態、ZeRO-2：+勾配、ZeRO-3：+パラメータも分散。LLM学習で必須。
PROBLEM 19
Operator Fusion（Fused operations）の効果は。
① 精度向上　 ② 連続した演算を1カーネルに融合、メモリ転送削減でGPU効率↑　 ③ パラメータ削減　 ④ ランダム
解答・解説を見る
ANSWER ② FlashAttention の核心。HBM ↔ SRAM 転送を減らす。Triton, CUDA Graph で実装。
PROBLEM 20
『Stop Gradient』の用途は。
① 学習停止　 ② 計算グラフの一部で勾配を遮断（GAN、対照学習、強化学習の Target Network 等で必須）　 ③ パラメータ削除　 ④ ランダム
解答・解説を見る
ANSWER ② .detach() で実装。Target を更新せず参照だけしたいときに使う。BYOL、DQN、PPO で必須テクニック。
