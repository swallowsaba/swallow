G検定 基本編 — 章12

学習を、安定させる。
汎化を、引き出す。

パーセプトロンが提案されて 65 年。『なぜ深層学習が動くか』は今でも完全に理解されていません。しかし数多くの『工学的工夫』が経験的に発見され、深層学習の実用化を支えています。本章では、それらを系統的に学びます。

SECTION ONE

基本

L1/L2正則化・Dropout・BatchNorm・LayerNorm・初期化・Early Stopping・データ拡張・Label Smoothing・Mixup・Gradient Clipping の10節。

基本

§1.1 正則化（L1, L2, Weight Decay）

L2正則化（Ridge）　 L_total = L + λ ||θ||²
　 大きな重みを罰する、滑らかな解

L1正則化（Lasso）　 L_total = L + λ ||θ||₁
　 多くの重みが正確に 0 → スパースモデル

Weight Decay　 NNでは SGD/AdamW の更新時に θ ← θ - η(∇L + λθ)
　 AdamW は減衰を Adam の更新式から分離して正しく動く

§1.2 Dropout — ニューロンをランダムに無効化

Hinton 2012 が提案。訓練時に 各ニューロンを確率 p（典型 0.5）で 0 に。

学習時：h ← Bernoulli(p) ⊙ h
推論時：h ← (1-p) · h 　(scale 補正、または学習時に 1/(1-p) で逆補正)

効果：(a) 大規模アンサンブルの近似、(b) ニューロン間の共依存防止、(c) 暗黙的正則化。ただし Batch Normalization と併用すると効果が薄れることが多い。

§1.3 Batch Normalization

Ioffe & Szegedy 2015。ミニバッチごとに各層の出力を平均0・分散1に正規化：

　 μ_B = (1/B) Σ h
　 σ²_B = (1/B) Σ (h - μ_B)²
　 ĥ = (h - μ_B) / √(σ²_B + ε)
　 y = γ ĥ + β　 (γ, β は学習可能)

推論時：訓練時の移動平均統計を使用

効果：(a) 内部共変量シフト緩和、(b) 大きな学習率使用可能、(c) 初期化への感度低下、(d) 正則化効果。CNN で標準。

§1.4 Layer Normalization と Group Normalization

  • BatchNorm：バッチ方向で正規化。バッチサイズ大が必要、RNN/Transformer に不向き。
  • LayerNorm：単一サンプル内の特徴次元で正規化。バッチサイズ非依存、RNN・Transformer で標準。
  • GroupNorm：チャネルをグループに分けて正規化。小バッチCNN向け。
  • RMSNorm：平均を引かず、二乗平均だけで正規化。LLaMA 等で採用、計算効率↑。

§1.5 重みの初期化

手法分散適性
Xavier/Glorot1/n_in や 2/(n_in+n_out)Sigmoid, tanh
He/Kaiming2/n_inReLU, Leaky ReLU
直交初期化直交行列RNN, 深層 NN
LeCun1/n_inSELU

§1.6 Early Stopping

検証損失が改善しなくなったら学習を停止。Patience（数エポック改善なしで停止）で実装。最も簡単で効果的な正則化。

§1.7 データ拡張

対象手法
画像反転、回転、クロップ、色変化、Cutout、RandAugment、AutoAugment
音声時間伸縮、ピッチシフト、ノイズ付加、SpecAugment
テキスト類義語置換、Back Translation、EDA、SimCLR系の対照
表形式SMOTE、特徴量ノイズ、Mixup

§1.8 Label Smoothing

one-hot ラベル [0,1,0,0] を [ε/(K-1), 1-ε, ε/(K-1), ε/(K-1)]（典型 ε=0.1）に置換。過信を防ぎ汎化を改善。情報論的にはエントロピーの底上げ。

§1.9 Mixup と CutMix

  • Mixup：2サンプルを λ:1-λ で線形混合、ラベルも同様に混合。
  • CutMix：1枚の画像の一部を別画像で置換、ラベルも面積比で混合。
  • MixCut：上記の組合せ。CIFAR・ImageNet で大幅性能向上。

§1.10 Gradient Clipping

勾配ノルムが閾値を超えたらスケーリング。RNN や Transformer の学習で標準（勾配爆発防止）：

g ← g · min(1, threshold / ||g||)　 (典型 threshold=1.0 〜 5.0)
SECTION TWO

応用

Stochastic Depth、Knowledge Distillation、Mixed Precision、Gradient Checkpointing の4節。

応用

§2.1 Stochastic Depth

残差ブロックを確率的にスキップ（恒等写像で置換）。Dropout の深さ版。100層超の深いResNet で標準的に使われる。

§2.2 Knowledge Distillation（蒸留）

大きな教師モデルの出力（soft target）で小さな生徒モデルを学習。Hinton 2015。LLM の小型化（DistilBERT、TinyLLaMA）で必須。

§2.3 Mixed Precision Training

主に FP16 で計算しつつ、損失スケーリングと FP32 重みコピーで精度を保つ。GPU メモリ半減、計算速度2〜3倍。最近は BF16 が主流（FP16より広いdynamic range）。NVIDIA Hopper 以降は FP8 も。

§2.4 Gradient Checkpointing（Activation Checkpointing）

順伝播の活性化を全部保存せず、一部だけ。逆伝播時に再計算。メモリを √L 倍に削減、計算は 1.3〜2倍。LLM 学習で必須。

SECTION THREE

ハンズオン

Dropout、BN、Data Aug、LR Schedule、Early Stoppingの5節。

ハンズオン

§3.1 PyTorch で Dropout

import torch.nn as nn class MLPDropout(nn.Module): def __init__(self): super().__init__() self.fc1 = nn.Linear(784, 256) self.drop1 = nn.Dropout(0.3) self.fc2 = nn.Linear(256, 256) self.drop2 = nn.Dropout(0.3) self.fc3 = nn.Linear(256, 10) def forward(self, x): x = self.drop1(torch.relu(self.fc1(x))) x = self.drop2(torch.relu(self.fc2(x))) return self.fc3(x) # model.train() でドロップアウト有効、model.eval() で無効

§3.2 Batch Normalization 適用

class MLPBN(nn.Module): def __init__(self): super().__init__() self.net = nn.Sequential( nn.Linear(784, 256), nn.BatchNorm1d(256), nn.ReLU(), nn.Linear(256, 256), nn.BatchNorm1d(256), nn.ReLU(), nn.Linear(256, 10) ) def forward(self, x): return self.net(x) # CNN なら nn.BatchNorm2d、Transformer なら nn.LayerNorm

§3.3 データ拡張（画像）

from torchvision import transforms train_transform = transforms.Compose([ transforms.RandomCrop(32, padding=4), transforms.RandomHorizontalFlip(), transforms.ColorJitter(brightness=0.2, contrast=0.2), transforms.RandAugment(num_ops=2, magnitude=9), # 強力 transforms.ToTensor(), transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]) ]) # test_transform はノーマライズと ToTensor のみ

§3.4 学習率スケジューリング（Cosine Annealing + Warmup）

from torch.optim.lr_scheduler import CosineAnnealingLR, LinearLR, SequentialLR opt = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01) # 最初の 1000 ステップで線形 Warmup、その後 Cosine 減衰 warmup = LinearLR(opt, start_factor=0.01, total_iters=1000) cosine = CosineAnnealingLR(opt, T_max=100000) scheduler = SequentialLR(opt, [warmup, cosine], milestones=[1000]) for step in range(num_steps): # ... 学習 ... opt.step() scheduler.step()

§3.5 Early Stopping 実装

best_val = float('inf') patience = 10 counter = 0 for epoch in range(num_epochs): # 訓練ループ train_loss = train_one_epoch() val_loss = validate() if val_loss < best_val: best_val = val_loss counter = 0 torch.save(model.state_dict(), 'best.pt') # 最良時保存 else: counter += 1 if counter >= patience: print(f"Early stopping at epoch {epoch}") break
SECTION FOUR

シミュレーション

Dropout、BN、データ拡張の3つの効果実験。

シミュレーション

SIM 1 — Dropout の効果（CIFAR-10）

>> 同じ CNN、Dropout 率だけ変える

Dropout 0.0: train=99.5%, test=82.1%　 ← 過学習
Dropout 0.2: train=97.8%, test=85.3%
Dropout 0.5: train=92.4%, test=87.2%　 ← バランス良い
Dropout 0.7: train=85.1%, test=84.7%　 ← 過抑制、未学習気味

>> Dropout 0.3〜0.5 が典型
>> BNと併用すると効果が薄れる傾向（互いに似た役割）

SIM 2 — Batch Normalization の効果

>> 同じ深層 CNN、BN の有無だけ変える

[BN なし]
Epoch 1: loss=2.30, accuracy=10%　 ← 学習進まず
Epoch 10: loss=1.85, accuracy=42%　 ← 遅い
Epoch 50: loss=0.95, accuracy=68%

[BN あり]
Epoch 1: loss=0.71, accuracy=82%　 ← 一気に進む
Epoch 10: loss=0.21, accuracy=92%
Epoch 50: loss=0.05, accuracy=95%

>> BN で学習が安定・高速化、大きな学習率も使える
>> ResNet50 でlr=0.1 でも収束、なしでは困難

SIM 3 — データ拡張の威力

>> CIFAR-10、ResNet18

[拡張なし]　　　 test=85.2%
[Random Crop+Flip] test=92.1%　 ← 標準的な拡張で 7%↑
[+ ColorJitter] test=92.8%
[+ RandAugment] test=94.3%　 ← 強力な自動拡張
[+ Mixup]　　　 test=95.1%
[+ CutMix]　　　 test=95.6%　 ← さらなる改善
[+ Label Smoothing] test=95.9%

>> データ拡張の積み重ねで 10%以上の改善
>> 現代の SOTA はこれら『ベストプラクティス』の集積
SECTION FIVE

問題

G検定形式の演習問題 20問。

問題

PROBLEM 1
L2 正則化（Weight Decay）が抑制するのは。
① 学習速度　 ② 大きな重みの値、過学習　 ③ 計算量　 ④ メモリ
解答・解説を見る
ANSWER ② L_total = L + λ||θ||² で大きな重みを罰し、汎化を改善。MAP推定でガウス事前に対応。
PROBLEM 2
Dropout の典型的なドロップ率は。
① 0.01　 ② 0.1〜0.5　 ③ 0.9　 ④ 1.0
解答・解説を見る
ANSWER ② 一般に 0.1〜0.5。隠れ層で 0.5、入力層で 0.2 が古典的。BN と併用するときは小さめにする傾向。
PROBLEM 3
Dropout の動作として正しいのは。
① 訓練時のみニューロンを確率的に無効化、推論時はすべて使用　 ② 常に無効化　 ③ 推論でのみ動作　 ④ パラメータ削減
解答・解説を見る
ANSWER ① 訓練時のみドロップ、推論時は通常通り（スケール補正あり）。アンサンブル近似と解釈できる。
PROBLEM 4
Batch Normalization の正規化対象は。
① 単一サンプル内　 ② ミニバッチ内の各チャネルごと　 ③ パラメータ自体　 ④ 学習率
解答・解説を見る
ANSWER ② ミニバッチ内で各特徴/チャネルごとに平均・分散を計算し正規化。バッチサイズ依存、Tiny バッチでは不安定。
PROBLEM 5
Transformer で標準的な正規化は。
① BatchNorm　 ② LayerNorm（バッチサイズ非依存、系列長変化OK）　 ③ なし　 ④ Dropout
解答・解説を見る
ANSWER ② LayerNorm は単一サンプル内の特徴次元で正規化。可変長の系列、小バッチでも安定。最近は RMSNorm も（LLaMA 等）。
PROBLEM 6
He 初期化が適している活性化関数は。
① Sigmoid　 ② tanh　 ③ ReLU 系（Leaky ReLU、GELU を含む）　 ④ Step
解答・解説を見る
ANSWER ③ He（2015）は分散 2/n_in で、ReLU の片側 0 の性質を考慮。Xavier は Sigmoid/tanh 向け。LeCun は SELU 向け。
PROBLEM 7
Early Stopping の動作は。
① 訓練を必ず最終エポックまで　 ② 検証損失が改善しなくなったら停止（過学習開始前）　 ③ ランダム停止　 ④ 学習しない
解答・解説を見る
ANSWER ② Patience（数エポック改善なし）で停止。最も簡単で効果的な正則化。最良時のモデルを保存して使う。
PROBLEM 8
画像認識でよく使われる強力な自動データ拡張は。
① 反転のみ　 ② RandAugment、AutoAugment、TrivialAugment　 ③ ノイズのみ　 ④ なし
解答・解説を見る
ANSWER ② RandAugment は2つの操作を強度 9 でランダム適用。AutoAugment は RL で最適化された方策。深層学習の SOTA で標準。
PROBLEM 9
Label Smoothing が変更する対象は。
① 学習率　 ② one-hot ラベルを [ε/(K-1), 1-ε,...] に置換、過信抑制　 ③ パラメータ数　 ④ バッチサイズ
解答・解説を見る
ANSWER ② 真ラベルの確信度を 1.0 から 1-ε（典型 0.9）に下げる。汎化改善＋過信抑制。ImageNet, NMT などで標準。
PROBLEM 10
Mixup の動作は。
① 単一サンプル　 ② 2サンプルを λ:1-λ で線形混合、ラベルも同様に混合　 ③ ノイズ加算　 ④ パラメータ平均
解答・解説を見る
ANSWER ② Zhang 2018。画像とラベルを線形補間。決定境界を滑らかにし、汎化と頑健性を改善。CutMix は領域置換版。
PROBLEM 11
Gradient Clipping の目的は。
① 過学習防止　 ② 勾配爆発防止（特に RNN/Transformer 学習）　 ③ 計算高速化　 ④ メモリ削減
解答・解説を見る
ANSWER ② ||g|| が閾値超過時にスケーリング。RNN の長系列・Transformer の大規模学習で標準（threshold=1.0 が典型）。
PROBLEM 12
Knowledge Distillation（蒸留）の目的は。
① パラメータ増加　 ② 大きな教師モデルの知識を小さな生徒モデルに転送　 ③ ランダム学習　 ④ クラスタリング
解答・解説を見る
ANSWER ② Hinton 2015。教師の softmax 出力（soft target）で生徒を学習。DistilBERT、TinyLLaMA、MobileBERT 等で軽量化に必須。
PROBLEM 13
Mixed Precision Training で主に使う型は。
① FP32 のみ　 ② FP16 や BF16 と FP32 重みコピーの組合せ　 ③ INT8 のみ　 ④ INT4 のみ
解答・解説を見る
ANSWER ② 計算は FP16/BF16、重みは FP32 で保持、損失スケーリングで underflow 防止。GPU メモリ半減、計算2-3倍速。BF16 はダイナミックレンジが広く実用的。
PROBLEM 14
Gradient Checkpointing（Activation Checkpointing）の効果は。
① 計算高速化　 ② 順伝播の活性化を一部のみ保存し、逆伝播で再計算。メモリO(L) → O(√L)に　 ③ 精度向上　 ④ 学習率自動調整
解答・解説を見る
ANSWER ② メモリと計算のトレードオフ。LLM 学習では巨大モデルを限られた GPU で訓練するために必須。
PROBLEM 15
『Warmup』（学習率ウォームアップ）が必要な代表的状況は。
① 全ての状況　 ② Transformer 学習、大バッチ学習　 ③ 線形回帰　 ④ K-means
解答・解説を見る
ANSWER ② 初期の勾配が大きく不安定なため、η を 0 から徐々に上げる（数千ステップ）。Transformer、ViT、LLM 学習で必須。Cosine Annealing と組合せが標準。
PROBLEM 16
深層 CNN で勾配消失を抑制する主要技術として誤りはどれか。
① ReLU　 ② 残差接続　 ③ Batch Normalization　 ④ Sigmoid 活性化
解答・解説を見る
ANSWER ④ Sigmoid は勾配消失の主要原因の一つ。ReLU、残差接続、BN、適切な初期化が勾配消失緩和の四天王。
PROBLEM 17
Stochastic Depth の動作は。
① パラメータをランダムドロップ　 ② 残差ブロックを確率的にスキップ（恒等写像に）　 ③ 学習率変化　 ④ ノイズ加算
解答・解説を見る
ANSWER ② Huang 2016。残差NNの深さをランダム化することで正則化。100+ 層 ResNet で標準的に使われる。
PROBLEM 18
推論時に Dropout を有効にして複数回サンプリングする手法は何の推定に使うか。
① 高速化　 ② 不確実性推定（Monte Carlo Dropout, Gal 2016）　 ③ 学習継続　 ④ 正則化
解答・解説を見る
ANSWER ② Yarin Gal が示した。推論時 Dropout で N 回サンプリング、出力の分散が予測不確実性の近似となる。ベイズ NN の安価な近似。
PROBLEM 19
『Cosine Annealing』学習率スケジューラの形状は。
① 直線的減少　 ② 余弦関数で滑らかに減衰（T_max でゼロ近くに）　 ③ 階段状　 ④ 一定
解答・解説を見る
ANSWER ② lr = lr_max · (1+cos(π·t/T))/2 で滑らかに減衰。Warmup と組み合わせが現代学習の標準。SGDR で再起動版もある。
PROBLEM 20
AdamW が Adam と異なる主要な点は。
① Momentumなし　 ② 重み減衰を更新式の最後で適用（Adam の正則化と分離）、L2 正則化と等価に動作　 ③ 学習率を増やす　 ④ Bias 補正削除
解答・解説を見る
ANSWER ② 元の Adam で L2 を勾配に足すと2次モーメントとの相互作用で減衰効果が薄れる。AdamW は θ ← θ - η(m̂/√v̂ + ε) - η λ θ と最後で減衰、L2 と一致。LLM学習の事実上の標準。
