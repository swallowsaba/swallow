G検定 基本編 — 章7

学び方を、選ぶ。
過学習と、闘う。

『どのアルゴリズムを使うか』より、どの学習パラダイムを選ぶかの方が結果を大きく左右します。ラベル付きデータが豊富なら教師あり、構造を発見したいなら教師なし、ラベル収集が高コストなら自己教師あり、最適化問題なら強化学習。さらに、データを 訓練/検証/テスト に分け、過学習をモニタし、ハイパラを系統的に探索する ── これらの『学習を運営する』スキルが、本章の中身です。

SECTION ONE

基本

5つの学習パラダイム、データ分割、バイアス-バリアンス、過学習、ハイパラ、NFL定理の10節。

基本

§1.1 教師あり学習 — ラベル付きデータからの学習

(x, y) ペアから関数 f: x → y を学習。回帰（y が連続）と 分類（y が離散カテゴリ）に分かれる。

タスク例代表手法損失関数
回帰住宅価格、需要予測線形回帰、SVR、勾配ブースティングMSE, MAE, Huber
二値分類スパム/ham、疾病有無ロジ回帰、SVM、ランダムフォレストBCE, Hinge
多クラス分類画像認識、文書カテゴリSoftmax + NN, GBDTCE
多ラベル分類マルチタグ付けSigmoid × K独立BCE
順序回帰レーティング予測Ordinal Logistic順序対応CE

§1.2 教師なし学習 — 構造の発見

ラベルなしのデータからパターンや構造を見出す：

  • クラスタリング：K-means、階層型、DBSCAN、GMM
  • 次元削減：PCA、t-SNE、UMAP、Autoencoder
  • 密度推定：KDE、GMM、Normalizing Flow
  • 異常検知：Isolation Forest、One-Class SVM、Autoencoder再構成誤差
  • アソシエーション：Aprioriアルゴリズム、買い物カゴ分析

§1.3 半教師あり学習

少量のラベル付き＋大量のラベルなしから学習。Self-Training、Pseudo-Labeling、Co-Training、グラフベース手法など。実用：医療画像、Webラベル分類（収集コスト高）。

§1.4 自己教師あり学習 — データ自身が教師

ラベルなしデータから人工的に教師信号を作る。NLP の Masked Language Model（BERT）、Next Token Prediction（GPT）、画像の対照学習（SimCLR、MoCo）、Masked Image Modeling（MAE）。これが大規模事前学習の核で、現代の基盤モデル時代を支える。

§1.5 強化学習 — 報酬からの行動学習

エージェント-環境ループ
　 状態 s_t → 行動 a_t → 報酬 r_t → 次状態 s_{t+1}
　 累積報酬 Σ γt r_t を最大化する方策 π(a|s) を学習

主要アルゴリズム
　 Q-Learning, SARSA, DQN, A3C, PPO, SAC
応用　 AlphaGo, AlphaZero, ChatGPT(RLHF), 自動運転, ロボティクス
FIG 7.1 ／ 5つの主要な学習パラダイム。データに含まれる『教師信号』の質と量で分類される。実用では複数を組み合わせることが多い（自己教師あり事前学習 + 教師あり微調整など）。

§1.6 訓練・検証・テストデータ

データを3分割するのが標準（典型 60/20/20、または 80/10/10）：

分割用途注意
訓練データパラメータの学習最大の比率
検証データハイパラ選択、Early Stoppingこれに過適合させない
テストデータ最終評価のみ（1回）本番性能の推定。複数回見ると信頼性失う
▎ データリーク（Data Leakage）
テストデータの情報が訓練に漏れる現象。典型例：(a)時系列データを単純シャッフルで分割、(b)正規化を全データで実行してから分割、(c)同一患者の複数画像が訓練と検証両方に、(d)目的変数を間接的に含む特徴量。本番性能を大幅に過大評価する致命的なバグ。

§1.7 汎化誤差とバイアス・バリアンス分解

期待汎化誤差の分解（MSE の場合）
　 E[(y - f̂(x))²] = (Bias)² + Variance + Irreducible Noise

　 バイアス　 E[f̂(x)] - f(x)　 モデルの『間違い方の偏り』
　 バリアンス　 E[(f̂(x) - E[f̂(x)])²]　 訓練データへの感度
　 削減不可能ノイズ　 ラベル自体のばらつき
FIG 7.2 ／ 古典的なバイアス-バリアンスのトレードオフ。モデルが単純すぎるとBias大（未学習）、複雑すぎるとVariance大（過学習）。中間に最適点がある。深層学習では『Double Descent』現象でこの古典像が更新された。

§1.8 過学習と未学習

状態訓練誤差検証誤差対処
未学習（Under-fitting）大大モデル複雑化、特徴量追加、正則化緩める
適切小小このまま運用
過学習（Over-fitting）小大正則化、データ追加、Early Stopping、Dropout

§1.9 ハイパーパラメータ調整

手法説明用途
Grid Searchすべての組合せを試す少数の離散ハイパラ
Random Searchランダムサンプル高次元（Bergstra 2012 が標準推奨）
Bayesian Optimizationガウス過程で次の候補予測1試行が高コスト（NN学習等）
Hyperband / ASHA悪い候補を早期打切り大規模並列、Population Based Training
Optuna / Ray TuneTPE + 早期打切り実用的標準

§1.10 No Free Lunch 定理

NFL定理（Wolpert 1996）：『全ての可能な問題分布で平均した性能は、全アルゴリズムで等しい』。実用的含意：万能アルゴリズムは存在せず、問題に合った手法を選ぶ必要。深層学習が万能に見えるのは、実用問題が『画像・言語など特定の構造』を持つから。

SECTION TWO

応用

各パラダイムの代表手法、転移学習、マルチタスク、メタ学習の4節。

応用

§2.1 各パラダイムの代表アルゴリズム

教師あり：線形回帰、ロジ回帰、SVM、決定木、Random Forest、XGBoost、LightGBM、NN。教師なし：K-means、DBSCAN、GMM、PCA、t-SNE、Autoencoder。半教師あり：Pseudo-Labeling、MixMatch、FixMatch。自己教師あり：BERT MLM、GPT NTP、SimCLR、MAE。強化学習：DQN、PPO、SAC、AlphaZero。

§2.2 転移学習 — 事前学習＋微調整

大規模データで学習したモデルを、別タスクに転用。BERT/GPT で標準化された手法：(1)事前学習（巨大コーパスで自己教師あり）、(2)微調整（少量のタスク特化データで教師あり）。LoRAなど低ランク手法でメモリ削減も。

§2.3 マルチタスク学習

複数タスクを同時に解くNNを学習。タスク間の知識共有で各タスク単独より高性能になることがある。GPT-4 が翻訳・要約・コード生成を統一的に扱えるのもマルチタスク的。

§2.4 メタ学習・Few-shot Learning

『学び方を学ぶ』。MAML（Model-Agnostic Meta-Learning）はパラメータ初期値を最適化し、新タスクに数ステップで適応できるようにする。GPT-4 の In-Context Learning は事実上のメタ学習で、プロンプトの例から新タスクを推論する。

SECTION THREE

ハンズオン

データ分割、ハイパラ探索、学習曲線、過学習対処、K-fold の5節。

ハンズオン

§3.1 train_test_split と Cross-Validation

from sklearn.model_selection import train_test_split, cross_val_score, KFold from sklearn.ensemble import RandomForestClassifier from sklearn.datasets import load_iris X, y = load_iris(return_X_y=True) # 単純な分割 X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.2, stratify=y, random_state=42) # 5-fold Cross Validation clf = RandomForestClassifier(n_estimators=100, random_state=42) scores = cross_val_score(clf, X, y, cv=5, scoring='accuracy') print(f"CV平均: {scores.mean():.4f} ± {scores.std():.4f}")

§3.2 Optuna でハイパラ探索

import optuna from sklearn.model_selection import cross_val_score def objective(trial): params = { 'n_estimators': trial.suggest_int('n_estimators', 50, 500), 'max_depth': trial.suggest_int('max_depth', 3, 20), 'min_samples_split': trial.suggest_int('min_samples_split', 2, 20), } clf = RandomForestClassifier(**params, random_state=42) score = cross_val_score(clf, X, y, cv=5).mean() return score study = optuna.create_study(direction='maximize') study.optimize(objective, n_trials=50) print("Best params:", study.best_params) print("Best score:", study.best_value)

§3.3 学習曲線で過学習を診断

from sklearn.model_selection import learning_curve import matplotlib.pyplot as plt import numpy as np sizes, train_scores, val_scores = learning_curve( RandomForestClassifier(n_estimators=100), X, y, train_sizes=np.linspace(0.1, 1.0, 10), cv=5) plt.plot(sizes, train_scores.mean(axis=1), label='Train') plt.plot(sizes, val_scores.mean(axis=1), label='Validation') plt.xlabel('Training Size'); plt.ylabel('Accuracy'); plt.legend() # Train と Val のギャップが大 → 過学習 # 両方とも低 → 未学習

§3.4 過学習対策の効果

from sklearn.tree import DecisionTreeClassifier # 無制限の決定木：高過学習リスク clf_unlimited = DecisionTreeClassifier(random_state=42) clf_unlimited.fit(X_train, y_train) print(f"Train: {clf_unlimited.score(X_train, y_train):.4f}") print(f"Test: {clf_unlimited.score(X_test, y_test):.4f}") # 深さ制限で正則化 clf_limited = DecisionTreeClassifier(max_depth=3, random_state=42) clf_limited.fit(X_train, y_train) print(f"Train (depth=3): {clf_limited.score(X_train, y_train):.4f}") print(f"Test (depth=3): {clf_limited.score(X_test, y_test):.4f}")

§3.5 Stratified K-Fold のスクラッチ

from sklearn.model_selection import StratifiedKFold skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42) fold_scores = [] for fold, (train_idx, val_idx) in enumerate(skf.split(X, y)): X_tr, X_v = X[train_idx], X[val_idx] y_tr, y_v = y[train_idx], y[val_idx] clf = RandomForestClassifier(n_estimators=100, random_state=42) clf.fit(X_tr, y_tr) s = clf.score(X_v, y_v) fold_scores.append(s) print(f"Fold {fold+1}: {s:.4f}") print(f"平均: {np.mean(fold_scores):.4f} ± {np.std(fold_scores):.4f}")
SECTION FOUR

シミュレーション

バイアス-バリアンス分解、過学習、NFL の3つの実験。

シミュレーション

SIM 1 — バイアス-バリアンス分解の実験

>> 真の関数 y = sin(x) + ε にデータ生成、多項式回帰でフィット
>> 100回の独立試行で予測の平均（Bias）と分散（Variance）を測定

次数 1: Bias²=0.412, Var=0.011, 合計=0.423 ← 未学習（高Bias）
次数 3: Bias²=0.045, Var=0.038, 合計=0.083 ← バランス良い
次数 5: Bias²=0.012, Var=0.087, 合計=0.099
次数 9: Bias²=0.003, Var=0.241, 合計=0.244 ← 過学習（高Var）

>> 古典的Uカーブ：次数3前後が最適

SIM 2 — 過学習の検出

>> 5層NN、MNIST、過学習を意図的に起こす

Epoch 1: train_loss=0.512, val_loss=0.478, gap=-0.034
Epoch 5: train_loss=0.123, val_loss=0.156, gap= 0.033
Epoch 10: train_loss=0.047, val_loss=0.142, gap= 0.095
Epoch 20: train_loss=0.011, val_loss=0.198, gap= 0.187 ← gap拡大
Epoch 50: train_loss=0.001, val_loss=0.342, gap= 0.341 ← 明らかな過学習

>> Early Stopping を Epoch 10 で適用すべき
>> あるいは Dropout、L2正則化、データ拡張を導入

SIM 3 — NFL定理の意味

>> 仮想的に『すべての可能な問題分布』で評価

データセットA（画像分類風）: CNN=92%, RF=78%, k-NN=65%
データセットB（表形式）　 : CNN=72%, RF=89%, k-NN=84%
データセットC（時系列）　 : CNN=68%, RF=72%, RNN=85%
データセットD（小サンプル）: CNN=55%, RF=68%, k-NN=72%

>> どのモデルも『あらゆる問題に強い』わけではない
>> 問題特性に合った選択が必要（NFL定理の経験的確認）
SECTION FIVE

問題

G検定形式の演習問題 20問。

問題

PROBLEM 1
教師あり学習の定義として正しいのは。
① ラベルなしから学ぶ　 ② (入力, 正解) ペアからマッピングを学ぶ　 ③ 報酬から学ぶ　 ④ 学習しない
ANSWER ② 教師あり学習は (x, y) ペアから関数 y=f(x) を学習。分類と回帰が代表。
PROBLEM 2
K-means や PCA はどの学習パラダイムか。
① 教師あり　 ② 教師なし　 ③ 強化学習　 ④ 自己教師あり
ANSWER ② ラベルを使わずデータ構造（クラスタ、主成分）を見出す教師なし学習の代表。
PROBLEM 3
BERT の MLM や GPT の Next Token Prediction はどのパラダイムか。
① 教師あり　 ② 教師なし　 ③ 自己教師あり（データから教師信号を作る）　 ④ 強化学習
ANSWER ③ 大量のラベルなしテキストから、自分自身でマスク埋めや次トークン予測という『教師信号』を作って学習。現代基盤モデルの核。
PROBLEM 4
強化学習の中核要素は。
① ラベル　 ② 状態・行動・報酬・方策　 ③ クラスタ　 ④ ベクトル化
ANSWER ② エージェントが状態s_tで行動a_tを取り、報酬r_tを得て次状態s_{t+1}へ。累積報酬を最大化する方策 π(a|s) を学習。AlphaGo, RLHF の根幹。
PROBLEM 5
訓練・検証・テストデータの主な役割として正しい組合せは。
① 訓練=学習、検証=ハイパラ選択、テスト=最終評価　 ② すべて同じ　 ③ テストで学習　 ④ 検証はランダム
ANSWER ① テストデータは『最終評価1回のみ』。検証で繰り返しチューニングしたあとに、初めて見るデータで本番性能を推定。
PROBLEM 6
『データリーク』が起きる典型例は。
① 時系列データを単純シャッフルで分割、全データで正規化、目的変数情報の混入　 ② 大きすぎるバッチ　 ③ 学習率小さい　 ④ パラメータ少ない
ANSWER ① いずれもテスト情報の漏洩。本番性能を大幅に過大評価する致命的バグ。時系列は時間順分割、正規化は訓練データで計算してテストに適用。
PROBLEM 7
過学習（overfitting）の典型的サインは。
① 訓練・検証とも誤差大　 ② 訓練誤差小、検証誤差大　 ③ 両方小　 ④ ランダム
ANSWER ② 過学習は『訓練データを暗記して、未見データで失敗』。対処はEarly Stopping、Dropout、L2、データ拡張、Augmentation。
PROBLEM 8
バイアス-バリアンス分解で『バイアス大』を表す状況は。
① モデルが複雑すぎ　 ② モデルが単純すぎて関数を表現できない（未学習）　 ③ データ多すぎ　 ④ 正則化なし
ANSWER ② Bias は『系統的な誤差』、モデルの表現力不足が原因。対処はモデル複雑化、特徴量追加、正則化緩和。
PROBLEM 9
5-fold Cross-Validation の目的は。
① データ量を増やす　 ② モデル評価の分散を減らし安定化、ハイパラ選択の信頼性　 ③ 計算を減らす　 ④ 正則化
ANSWER ② データを5分割し各回1つを検証に。5回の評価の平均で評価分散を抑える。小データで特に重要。
PROBLEM 10
高次元ハイパラ探索で Grid Search より Random Search が推奨される理由は。
① Bergstra 2012：少数の重要パラメータを効率的に探索できる　 ② 計算量が同じ　 ③ ランダムは常に最適　 ④ コードが短い
ANSWER ① 多くの場合、性能を支配するハイパラは少数。Grid は全格子点を試すが、無関係次元の試行が無駄。Random は重要次元を多くサンプリングできる。
PROBLEM 11
Bayesian Optimization の特徴は。
① 過去の試行結果からガウス過程で次の候補を予測、試行数を最小化　 ② ランダム　 ③ 厳密な数学なし　 ④ Optuna は使えない
ANSWER ① GP やTPE で予測平均と分散を考慮し『有望かつ未探索な点』を選ぶ。NN の学習など1試行が高コストな場合に強力。Optuna は TPE ベースで実装。
PROBLEM 12
転移学習（Transfer Learning）の典型的フローは。
① 事前学習（大規模データで自己教師あり）→ ファインチューニング（少量タスクデータ）　 ② ゼロから訓練のみ　 ③ ハイパラ調整のみ　 ④ ランダム
ANSWER ① BERT/GPT/ViT すべてこの流れ。LLM ではIn-Context Learning, LoRA, RLHFなど発展形が多数。
PROBLEM 13
No Free Lunch（NFL）定理が示唆することは。
① あるアルゴリズムが万能　 ② すべての可能な問題分布で平均すると、どのアルゴリズムも同性能 → 問題ごとに適した手法を選ぶ必要　 ③ 学習不要　 ④ 深層学習が最強
ANSWER ② Wolpert 1996。実用問題が特定構造（画像・言語の局所性、自然法則）を持つから NN が機能するが、すべての問題に最強なアルゴリズムは数学的に存在しない。
PROBLEM 14
層化抽出（Stratified Sampling）が必要な典型ケースは。
① クラス不均衡データの分類　 ② 完全に均一なデータ　 ③ 連続値のみ　 ④ 時系列のみ
ANSWER ① クラス比が偏った（陽性 1% など）データで単純ランダム分割すると検証セットに陽性が含まれない可能性も。層化で各分割のクラス比を均一に保つ。
PROBLEM 15
『Early Stopping』が機能する原理は。
① 訓練を最後まで続ける　 ② 検証損失が悪化し始める前に学習を止めて過学習を防ぐ　 ③ パラメータ削減　 ④ データ拡張
ANSWER ② 訓練損失は単調減少するが検証損失は U字。最低点で止めると過学習せず汎化性能最大化。Patience（数エポック改善なしで停止）で実装。
PROBLEM 16
Few-shot Learning の典型は。
① 大量データで学習　 ② 数〜数十サンプルで新タスクに適応（MAML, In-Context Learning）　 ③ ゼロデータ　 ④ パラメータ調整なし
ANSWER ② 限られたサンプルで新タスクに適応する手法。GPT-4 のIn-Context Learning はプロンプトの例から推論する強力な Few-shot 手法。
PROBLEM 17
マルチタスク学習の利点として誤りはどれか。
① タスク間で知識共有　 ② 各タスクで Bias を下げる効果　 ③ 正則化効果でVariance↓　 ④ 全てのタスクが互いに干渉して性能が必ず下がる
ANSWER ④ 実際にはタスク間の知識共有でしばしば性能↑。ただし関連性のないタスク同時学習は『負の転移』を起こすことがある。
PROBLEM 18
『Double Descent』現象（深層学習で観察）の主張は。
① モデル複雑度を上げ続けると、過学習を超えて再び汎化が改善する　 ② U字カーブのみ　 ③ 学習しない　 ④ 必ず単調減少
ANSWER ① 古典的 U字 の後に二度目の下降。過パラメータ化された NN が経験的に汎化することの一つの説明。Belkin 2019。
PROBLEM 19
半教師あり学習が威力を発揮する状況は。
① ラベル付け不可　 ② ラベル付きデータが少ないがラベルなしデータは大量にある（医療画像など）　 ③ 全データラベル付き　 ④ データなし
ANSWER ② ラベル収集コストが高い領域で、ラベルなしデータを活用して学習を強化。Pseudo-Labeling, FixMatch, MixMatch などの手法。
PROBLEM 20
『時系列データ』を扱う際の正しい分割方法は。
① ランダム k-fold　 ② 時系列順に分割：過去で訓練、未来で検証・テスト（Time Series Split）　 ③ シャッフル　 ④ 階層化
ANSWER ② 時系列の単純シャッフルは『未来を見て過去を予測』するデータリークの典型。常に時間順で分割。Expanding window / Rolling window などの手法がある。

章のまとめ

5つの学習パラダイム（教師あり、教師なし、半教師あり、自己教師あり、強化学習）と、訓練・検証・テスト分割、バイアス-バリアンス分解、過学習・未学習、ハイパラ調整（Grid/Random/Bayesian/Optuna）、NFL 定理を扱いました。応用は転移学習、マルチタスク、メタ学習。ハンズオンで train_test_split、CV、Optuna、学習曲線、Stratified K-Fold。シミュレーションでバイアス-バリアンス、過学習検出、NFL の意味。次章は古典機械学習（線形回帰、ロジ回帰、SVM、決定木、Random Forest、GBDT、K-means、PCA など）で、これらのパラダイムの具体的な実装を扱います。
