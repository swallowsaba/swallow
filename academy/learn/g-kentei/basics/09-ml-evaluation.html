G検定 基本編 — 章9

数字で性能を語る。
正しい数字を選ぶ。

モデルが本番でどれだけ役立つかは、評価指標で決まります。クラス不均衡、コスト非対称、ランキング、確率出力 ── 状況により適切な指標は異なる。Accuracy だけ見ていてはたどり着けない正しい意思決定への鍵が、本章にあります。

SECTION ONE

基本

混同行列・Accuracy・Precision/Recall・F1・ROC-AUC・PR-AUC・Log Loss・MAP@k・回帰指標・多クラスの10節。

基本

§1.1 混同行列

予測：陽性予測：陰性
実際：陽性TP（真陽性）FN（偽陰性）
実際：陰性FP（偽陽性）TN（真陰性）

§1.2 Accuracy（正解率）

Accuracy = (TP + TN) / 全体

クラス均衡なら有用。不均衡（陽性 1%）では誤解を生む典型。
▎ Accuracyの罠
陽性率1%のスパム検知で『全て陰性』と予測 → Accuracy 99% だが完全に無価値。不均衡では F1/PR-AUC を使う。

§1.3 Precision と Recall

Precision = TP / (TP + FP)　 陽性予測のうち実際に陽性の割合
Recall（感度） = TP / (TP + FN)　 実際の陽性のうち検出できた割合
Specificity = TN / (TN + FP)　 実際の陰性のうち正しく陰性判定の割合

使い分け
　 医療：Recall重視（見逃し=死）
　 スパム：Precision重視（誤検出は迷惑）

§1.4 F1スコア

F1 = 2·P·R / (P+R)　 調和平均、両方高くないと高くならない
F_β：β>1でRecall重視、β<1でPrecision重視

§1.5 ROC曲線とAUC

ROC：閾値を変化させて (FPR, TPR) をプロット
AUC：曲線下面積、ランダム=0.5、完全=1.0
解釈：陽性-陰性ペアで陽性が高スコアになる確率
FIG 9.1 ／ ROC曲線。閾値変化で(FPR, TPR)をプロット。AUCで分類性能を一値に集約。

§1.6 PR曲線とPR-AUC

Precision-Recall曲線。極端不均衡では ROC より PR-AUC が有意義。情報検索、異常検知、推薦で標準。

§1.7 Log Loss と Brier Score

Log Loss = -Σ y log p̂ + (1-y) log(1-p̂)
Brier = (1/N)Σ(p̂-y)²
ハードラベルでなく『出力確率の信頼性』を評価

§1.8 MAP@k と NDCG（ランキング）

Precision@k：上位k個の精度。MAP：APの平均。NDCG：上位ほど重みを大きく、正規化。推薦・検索・NERで標準。

§1.9 回帰指標

指標式特徴
MSE(1/N)Σ(y-ŷ)²外れ値に敏感、学習で最頻用
RMSE√MSE単位がyと同じ、解釈しやすい
MAE(1/N)Σ|y-ŷ|外れ値に頑健
R²1 - SSE/SST分散説明割合、線形回帰標準
MAPE(1/N)Σ|y-ŷ|/|y|%スケール、y=0で爆発
HuberMSE+MAE混合外れ値頑健、滑らか

§1.10 多クラス分類指標

平均計算特徴
Macro各クラスF1を単純平均少数クラスを保護
Micro全TP/FP/FN集計後F1多数クラス支配
Weightedクラス頻度で重み付け頻度バランス考慮
SECTION TWO

応用

クラス不均衡対応、コスト感度学習、Calibration、統計検定の4節。

応用

§2.1 クラス不均衡対応

(a)サンプリング：Oversampling (SMOTE)、Undersampling、(b)損失重み付け：class_weight、Focal Loss、(c)閾値調整：0.5を最適化、(d)指標選択：F1、PR-AUC、(e)生成モデル：GAN/VAEで少数クラス生成。

§2.2 コスト感度学習

FPとFNのコストが非対称（医療、不正検知）な場合、コスト行列を組み込んだ損失で学習、または閾値を期待コスト最小で選ぶ。

§2.3 Calibration（確率較正）

現代NNは過信気味。Platt Scaling（シグモイド）、Isotonic Regression、Temperature Scalingで較正。評価はECE（Expected Calibration Error）。

§2.4 統計検定でモデル比較

2モデルの差が偶然か検定：McNemar検定（同データ予測比較）、対応のあるt検定（CVスコア）、Wilcoxon符号付き順位。Kaggle微差競争で重要。

SECTION THREE

ハンズオン

sklearn metrics、混同行列、ROC、閾値最適化、CVの5節。

ハンズオン

§3.1 sklearn metrics 総合使用

from sklearn.metrics import ( accuracy_score, precision_score, recall_score, f1_score, roc_auc_score, log_loss, confusion_matrix, classification_report) y_true = [0, 0, 1, 1, 0, 1, 1, 0] y_pred = [0, 1, 1, 1, 0, 0, 1, 0] y_prob = [0.1, 0.6, 0.8, 0.9, 0.2, 0.4, 0.7, 0.3] print(f"Accuracy: {accuracy_score(y_true, y_pred):.4f}") print(f"Precision: {precision_score(y_true, y_pred):.4f}") print(f"Recall: {recall_score(y_true, y_pred):.4f}") print(f"F1: {f1_score(y_true, y_pred):.4f}") print(f"ROC-AUC: {roc_auc_score(y_true, y_prob):.4f}") print(f"Log Loss: {log_loss(y_true, y_prob):.4f}")

§3.2 ROC曲線描画

from sklearn.metrics import roc_curve, auc import matplotlib.pyplot as plt fpr, tpr, _ = roc_curve(y_true, y_prob) roc_auc = auc(fpr, tpr) plt.plot(fpr, tpr, label=f'ROC (AUC={roc_auc:.3f})') plt.plot([0,1], [0,1], '--', label='Random') plt.legend(); plt.savefig('roc.png')

§3.3 閾値最適化（F1最大化）

import numpy as np from sklearn.metrics import precision_recall_curve p, r, t = precision_recall_curve(y_true, y_prob) f1 = 2*p*r/(p+r+1e-10) best = np.argmax(f1) print(f"最適閾値={t[best]:.4f}, F1={f1[best]:.4f}")

§3.4 CV自前実装

from sklearn.model_selection import StratifiedKFold from sklearn.linear_model import LogisticRegression skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42) metrics = {'acc':[], 'auc':[], 'f1':[]} for tr, va in skf.split(X, y): clf = LogisticRegression(max_iter=1000).fit(X[tr], y[tr]) pred = clf.predict(X[va]); prob = clf.predict_proba(X[va])[:,1] metrics['acc'].append(accuracy_score(y[va], pred)) metrics['auc'].append(roc_auc_score(y[va], prob)) metrics['f1'].append(f1_score(y[va], pred)) for k,v in metrics.items(): print(f"{k}: {np.mean(v):.4f}±{np.std(v):.4f}")

§3.5 Calibration の効果

from sklearn.calibration import CalibratedClassifierCV from sklearn.metrics import brier_score_loss base = RandomForestClassifier(n_estimators=100, random_state=42) cal = CalibratedClassifierCV(base, method='isotonic', cv=5).fit(X_tr, y_tr) base.fit(X_tr, y_tr) print(f"Base: {brier_score_loss(y_te, base.predict_proba(X_te)[:,1]):.4f}") print(f"Cal: {brier_score_loss(y_te, cal.predict_proba(X_te)[:,1]):.4f}")
SECTION FOUR

シミュレーション

不均衡データの罠、閾値とPR、ROC vs PRの3節。

シミュレーション

SIM 1 — 不均衡データの Accuracy の罠

>> スパム検知：陽性率1%、10000サンプル

[全て陰性と予測（無価値）]
　 TP=0, FP=0, FN=100, TN=9900
　 Accuracy=99.00% ← 高く見える
　 Recall=0%, F1=0, AUC=0.5

[まともなモデル]
　 TP=80, FP=20, FN=20, TN=9880
　 Accuracy=99.60%, F1=0.800, AUC=0.89

>> Accuracy差はわずか0.6%でも、F1/AUCで実体が明白

SIM 2 — 閾値と Precision-Recall

>> 同モデルで閾値変化

閾値 0.3: P=0.45, R=0.95 ← Recall重視
閾値 0.5: P=0.78, R=0.82
閾値 0.7: P=0.92, R=0.65
閾値 0.9: P=0.98, R=0.32 ← Precision重視

>> トレードオフ、ビジネス要件で決定

SIM 3 — ROC vs PR-AUC、不均衡

>> 陽性率を変えて同モデル評価

陽性率 50%: ROC=0.91, PR=0.92
陽性率 10%: ROC=0.91, PR=0.65
陽性率 1%: ROC=0.90, PR=0.28
陽性率0.1%: ROC=0.89, PR=0.08

>> ROCは不均衡に頑健、PRは真の困難度を示す
SECTION FIVE

問題

G検定形式 20問。

問題

PROBLEM 1
混同行列の TP は何を意味するか。
① 真陽性（陽性を陽性と予測）　 ② 偽陽性　 ③ 偽陰性　 ④ 真陰性
解答・解説を見る
ANSWER ① TP = True Positive。
PROBLEM 2
Precision の定義は。
① TP/(TP+FN)　 ② TP/(TP+FP)　 ③ TN/(TN+FP)　 ④ (TP+TN)/全体
解答・解説を見る
ANSWER ② 陽性予測のうち実際に陽性の割合。FPが増えると下がる。
PROBLEM 3
Recall（感度）の定義は。
① TP/(TP+FN)　 ② TP/(TP+FP)　 ③ TN/(TN+FP)　 ④ (TP+TN)/全体
解答・解説を見る
ANSWER ① 実際の陽性のうち検出できた割合。FN（見逃し）が増えると下がる。医療診断で重視。
PROBLEM 4
F1スコアの定義は。
① 算術平均　 ② P×R　 ③ PとRの調和平均　 ④ P/R
解答・解説を見る
ANSWER ③ F1=2PR/(P+R)。両方が高くないと高くならない。
PROBLEM 5
スパム検知で『全て非スパム』予測で Accuracy 99% のモデルは。
① 良い　 ② 完全に無価値（Recall=0、F1=0）　 ③ 完全分類器　 ④ ベスト
解答・解説を見る
ANSWER ② 不均衡データでの Accuracy の典型的な罠。F1・PR-AUC は0。
PROBLEM 6
ROC-AUC の意味として正しいのは。
① 訓練時間　 ② 陽性-陰性ペアで陽性のスコアが高い確率　 ③ Precisionの平均　 ④ パラメータ数
解答・解説を見る
ANSWER ② AUCの確率的解釈。0.5=ランダム、1.0=完全。
PROBLEM 7
極端な不均衡（陽性率 0.1%）で適切な指標は。
① Accuracy　 ② PR-AUC　 ③ MSE　 ④ R²
解答・解説を見る
ANSWER ② ROC-AUCは不均衡に頑健すぎて高く出るがPR-AUCは真の困難度を反映。
PROBLEM 8
医療診断で重視すべき指標は。
① Precision　 ② Recall　 ③ Specificity　 ④ MSE
解答・解説を見る
ANSWER ② Recall でFN（見逃し）を最小化。
PROBLEM 9
スパムフィルタで重視すべきは。
① Precision　 ② Recall　 ③ 両方　 ④ Accuracy
解答・解説を見る
ANSWER ① Precision でFP（重要メール誤検出）を最小化。
PROBLEM 10
回帰指標で外れ値に頑健なのは。
① MSE　 ② RMSE　 ③ MAE　 ④ MAPE
解答・解説を見る
ANSWER ③ MAE は絶対値、外れ値の影響が線形。MSE は二乗で敏感。中間に Huber Loss。
PROBLEM 11
R²（決定係数）の意味は。
① 必ず0〜1　 ② 1-SSE/SST、モデルが説明する分散の割合　 ③ パラメータ数　 ④ Accuracy
解答・解説を見る
ANSWER ② テストで負になることもある。
PROBLEM 12
NNの予測確率を真の頻度に合わせる手法は。
① 正則化　 ② Calibration（Platt、Isotonic、Temperature）　 ③ 標準化　 ④ Augmentation
解答・解説を見る
ANSWER ② 現代NNは過信気味。Calibrationで補正、ECEで評価。
PROBLEM 13
多クラス F1 で macro と weighted の違いは。
① 同じ　 ② macro は各クラス等値、weighted はクラス頻度で重み付け　 ③ macroはスパース　 ④ weightedは教師あり
解答・解説を見る
ANSWER ② 不均衡では macro が公平、頻度反映なら weighted。
PROBLEM 14
推薦システムでの MAP@k の意味は。
① 全体精度　 ② 上位k個のAverage Precisionの平均、ランキング品質指標　 ③ MSE　 ④ R²
解答・解説を見る
ANSWER ② NDCG とともに推薦・検索で標準。
PROBLEM 15
Log Loss と Accuracy の違いは。
① 同じ　 ② Log Lossは予測確率の質を評価（自信度を考慮）、Accuracyはハードラベルのみ　 ③ Log Lossは離散　 ④ Accuracyは確率
解答・解説を見る
ANSWER ② 自信を持って間違えるほどLog Lossが高くなる。
PROBLEM 16
McNemar検定の用途は。
① t検定の代替　 ② 同一データに対する2モデル予測比較で差が有意か判定　 ③ クラスタリング評価　 ④ 回帰分析
解答・解説を見る
ANSWER ② 同データ・異モデルの予測一致／不一致を表で集計し検定。
PROBLEM 17
SMOTE が行うことは。
① 少数クラスを単純複製　 ② 少数クラスのk近傍間に合成サンプルを補間生成　 ③ 多数クラスを削除のみ　 ④ ランダム
解答・解説を見る
ANSWER ② 単純複製と違い、合成データで多様性を保つ。imbalanced-learnで実装。
PROBLEM 18
Cross-Validation の主目的は。
① データ拡張　 ② 評価分散を減らしハイパラ選択の信頼性向上　 ③ 計算高速化　 ④ ラベル予測
解答・解説を見る
ANSWER ② 1回分割では運次第。k-foldで多数評価の平均で安定化。
PROBLEM 19
Focal Loss が解決する問題は。
① 過学習　 ② クラス不均衡＋簡単例の過剰寄与（物体検出由来）　 ③ 学習率　 ④ 正則化
解答・解説を見る
ANSWER ② -(1-p)^γ log p で容易例の損失を下げ、難しい例（少数クラス）に集中。RetinaNet で導入。
PROBLEM 20
分類閾値の標準値 0.5 を変える理由として正しいのは。
① 必ず0.5　 ② 業務要件、不均衡、コスト非対称性に応じて最適化　 ③ 変えてはいけない　 ④ モデル変更必要
解答・解説を見る
ANSWER ② 0.5は単なる慣例値。F1最大化、期待コスト最小化などで決定。
