ニューラルネットの『学習』とは、何でしょうか。本質は 損失関数 L(θ) を最小化するパラメータ θ を見つけること。10億次元の谷に投げ込まれた小石が、最も低い場所へ転がる過程です。それを記述するのが微分。勾配が『どちらが下り坂か』を教え、ヘッセ行列が『カーブの曲がり具合』を教える。これに『慣性』（Momentum）と『適応的歩幅』（Adam）を加えれば、現代の深層学習オプティマイザが完成します。本章で、その下山の数学を完全に習得しましょう。

SECTION ONE

基本

微分・勾配・連鎖律・ヘッセ行列・テイラー展開・凸最適化・勾配降下法・SGD・Adam・ラグランジュの10節。

基本

§1.1 微分・偏微分 — 変化率を測る

1変数関数 f(x) の 微分 df/dx は、点 x における関数の 瞬間の変化率。グラフ上では 接線の傾き。多変数関数 f(x₁, x₂, ..., x_n) では、変数1つだけを動かして他を固定した変化率を 偏微分 ∂f/∂xᵢ と呼びます。

微分の定義　 df/dx = limh→0 (f(x+h) - f(x)) / h

偏微分　 ∂f/∂xᵢ = limh→0 (f(..., xᵢ+h, ...) - f(..., xᵢ, ...)) / h

機械学習における微分
　・ニューロン活性化関数 σ(z) の微分 σ'(z)
　・損失関数 L の パラメータ θ に関する偏微分 ∂L/∂θᵢ
　・これらが 逆伝播 で使われる

機械学習で頻出の微分公式：

関数微分備考
xnn xn-1多項式の基本
exex変化しない、softmax の核
log(x)1/xクロスエントロピー、対数尤度
sigmoid σ(x) = 1/(1+e-x)σ(x)(1-σ(x))出力で書ける、計算効率高
tanh(x)1 - tanh²(x)これも出力で書ける
ReLU(x) = max(0, x)1 (x>0), 0 (x<0)x=0 で不連続だが実用上問題なし
softmax(x)ipi(δij - pj)クロスエントロピーと組み合わせると ŷ - y
FIG 4.1 ／ 微分は『局所の傾き』。f'(x₀) が正なら関数は増加中、負なら減少中。最小化問題では、傾きの反対方向に進めば値が下がる。これが勾配降下法の根本原理。

§1.2 勾配ベクトル ∇f — 多変数版の傾き

多変数関数 f(x₁, ..., x_n) の 勾配ベクトル ∇f は、すべての偏微分を集めた n 次元ベクトル：

勾配ベクトル
　 ∇f(x) = (∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xn)

重要な性質
　① ∇f は 関数値が最も急に増加する方向 を指す
　② -∇f は 最も急に減少する方向（最急降下方向）
　③ ||∇f|| はその方向への 増加の急さ
　④ 等高線（contour）に 垂直 である
FIG 4.2 ／ 2変数損失関数の等高線図と勾配ベクトル。∇L はその点での最急上昇方向、-∇L が最急降下方向。等高線に垂直になる幾何学的事実は、勾配降下法の収束性能を理解する基本。

§1.3 連鎖律 — 合成関数の微分、バックプロパの心臓

合成関数 y = f(g(x)) の微分は、内側と外側の微分の積：

1変数の連鎖律　 dy/dx = (df/dg) · (dg/dx)

多変数の連鎖律（z が y₁, y₂, ..., yₖ の関数で、各 yᵢ が x の関数）
　 dz/dx = Σᵢ (∂z/∂yᵢ) · (dyᵢ/dx)

NN の例　 L = loss(σ(Wx + b), y_target)
　 ∂L/∂W = (∂L/∂σ) · (∂σ/∂z) · (∂z/∂W)　 ← 3段の連鎖

連鎖律は 逆伝播（バックプロパゲーション）の本質。NN は層の合成 f = f_L ∘ f_{L-1} ∘ ... ∘ f_1 ですから、損失の入力に関する微分は連鎖律で次々と計算できます。これを 計算グラフ で可視化すると分かりやすい。

FIG 4.3 ／ 計算グラフ上の連鎖律。順伝播で各中間量を計算しておき、逆伝播では上流の勾配に局所微分を掛けながら入力へ伝播させる。PyTorch・TensorFlow の自動微分はこれを機械的に実行する仕組み。

§1.4 ヘッセ行列 — 2階微分の集合体

多変数関数 f の ヘッセ行列 H は、すべての2階偏微分を並べた行列：

Hij = ∂²f / (∂xᵢ ∂xⱼ)

性質
　・常に対称（連続性の下で）
　・固有値の符号で局所形状が決まる
　　 - 全て正 → 正定値 → 極小
　　 - 全て負 → 負定値 → 極大
　　 - 正負混在 → 不定値 → 鞍点
　　 - 0を含む → 縮退（より高次が必要）
FIG 4.4 ／ ヘッセ行列の固有値による臨界点の3タイプ。高次元（NN）では鞍点が圧倒的に多く、SGDのノイズがその脱出に役立つ。
▎ NNにおけるヘッセ行列の罠
理論的には Newton 法（H⁻¹∇L で更新）が高速収束するが、深層学習では パラメータ数が 10⁹〜10¹² のため H の保存も逆行列計算も不可能。Adam 等の適応学習率はヘッセ行列の 対角近似 と見なせる。完全な2次最適化は K-FAC、Sophia、Shampoo 等の研究で部分的に実用化。

§1.5 テイラー展開 — 局所の多項式近似

関数 f を点 x₀ の周りで多項式展開する：

1変数テイラー展開
　 f(x₀+Δx) = f(x₀) + f'(x₀)Δx + (1/2)f''(x₀)Δx² + (1/6)f'''(x₀)Δx³ + ...

多変数版（2次まで）
　 f(x₀+Δ) ≈ f(x₀) + ∇f(x₀)TΔ + (1/2)ΔTH Δ

機械学習での用途
　・1次近似 → 勾配降下法の根拠
　・2次近似 → Newton 法、信頼領域法、XGBoost の更新
　・誤差解析、収束証明

§1.6 凸関数と凸最適化 — 解析可能な最適化

関数 f が 凸 とは、任意の x, y と t∈[0,1] で f(tx + (1-t)y) ≤ tf(x) + (1-t)f(y) が成立すること。直観的には『グラフが線分の下』。重要性質：

凸関数の性質
　・局所最小値 = 大域最小値
　・1次条件：∇f(x*) = 0 ⇒ x* は大域最適
　・2次条件：H が半正定値 ⇔ f は凸
　・任意の学習率で勾配降下が大域最適に収束（適切に小さければ）

代表的凸問題
　・線形回帰（OLS）
　・ロジスティック回帰
　・SVM
　・Ridge / Lasso

非凸の代表
　・ニューラルネット（多数の局所最適と鞍点を持つ）

NN は非凸ですが、経験的に SGD のノイズが鞍点を脱出させ、結果としてフラットな良いミニマに収束することが多い。これが深層学習が機能する『謎』の一部。

§1.7 勾配降下法 — 最も単純な最適化

パラメータ θ を勾配の反対方向に更新する反復法：

勾配降下法（GD）
　 θt+1 = θt − η ∇L(θt)

学習率 η の選び方
　・大きすぎ → 振動・発散
　・小さすぎ → 収束が極めて遅い
　・凸関数で Lipschitz 連続な勾配なら、η ≤ 1/L で収束保証

収束率
　・一般凸：O(1/T)（T=反復回数）
　・強凸：O(exp(-T/κ))（κ=条件数）
　・Nesterov 加速：O(1/T²)
FIG 4.5 ／ 学習率 η の効果。適切なら滑らかに最小へ、大きすぎると振動・発散、小さすぎると遅い。学習率の選択は深層学習で最も重要なハイパーパラメータの一つ。

§1.8 確率的勾配降下（SGD） — ミニバッチで爆速化

全データの勾配 ∇L(θ) = (1/N) Σ ∇Lᵢ(θ) は N が巨大だと毎回計算不可能。SGD（Stochastic Gradient Descent） はランダムに選んだ ミニバッチ（典型 B=32〜512）の勾配で更新する：

SGD
　 θt+1 = θt − η · (1/B) Σi∈batch ∇Lᵢ(θt)

長所
　・反復が高速（GPU並列化が容易）
　・ノイズが鞍点・浅い局所最適を脱出させる
　・暗黙的正則化効果（フラットなミニマへ）

短所
　・収束がノイジー（最後にぴたっと止まりにくい）
　・大バッチではノイズが減り、汎化性能が下がることも

§1.9 Momentum と Adam — 慣性と適応学習率

オプティマイザ核となる更新式特徴
SGDθ ← θ − η g最もシンプル。学習率調整で SOTA に届くことも
Momentumv ← βv + g, θ ← θ − η v慣性で振動抑制、収束加速（β=0.9 典型）
Nesterov Momentumθ_lookahead で勾配計算先読みで凸最適化の最適収束率
AdaGradθ ← θ − η g/√Σg²学習率を要素別に減衰、過去全て記憶
RMSPropg²の指数移動平均AdaGradの『過去を忘れる版』
AdamMomentum + RMSProp + バイアス補正NNのデファクト、頑健
AdamWAdam + 重み減衰分離LLM 学習の事実上の標準
LIONMomentum のみ、符号で更新2023 Google、メモリ効率
FIG 4.6 ／ SGD・Momentum・Adam の軌跡比較。細長い谷で SGD は振動するが、Momentum は慣性で進み、Adam は各方向で適応的に歩幅を変えて素早く収束する。
Adam の更新式（典型 β₁=0.9, β₂=0.999, ε=10⁻⁸）
　 mt = β₁ mt-1 + (1-β₁) gt　 ← 1次モーメント（Momentum 相当）
　 vt = β₂ vt-1 + (1-β₂) gt²　 ← 2次モーメント
　 m̂t = mt / (1-β₁t)　 ← バイアス補正
　 v̂t = vt / (1-β₂t)
　 θt+1 = θt − η · m̂t / (√v̂t + ε)

§1.10 制約付き最適化とラグランジュ未定乗数法

等式制約 g(x) = 0 のもとで f(x) を最適化する場合、ラグランジュ関数 を作る：

ラグランジアン　 L(x, λ) = f(x) − λ g(x)

停留点条件
　 ∂L/∂x = ∇f − λ∇g = 0
　 ∂L/∂λ = -g(x) = 0

不等式制約 h(x) ≤ 0 を含む場合 → KKT 条件
　① 停留性：∇f - λ∇g - μ∇h = 0
　② 主実行可能性：g=0, h≤0
　③ 双対実行可能性：μ ≥ 0
　④ 相補性：μh(x) = 0

応用：SVM（マージン最大化＋制約）、エントロピー最大化（指数分布族の導出）、Constrained Policy Optimization（安全強化学習）など。

SECTION TWO

応用

NNのバックプロパ、学習率スケジューリング、2次最適化、鞍点問題の4節。

応用

§2.1 ニューラルネットの逆伝播 — 連鎖律の機械的適用

L 層 NN の損失勾配を、出力側から入力側へ連鎖律で計算するアルゴリズム。1986年に Rumelhart, Hinton, Williams が Nature 論文で発表し、第2次AIブームを牽引しました。

順伝播
　 a⁽⁰⁾ = x
　 z⁽ˡ⁾ = W⁽ˡ⁾ a⁽ˡ⁻¹⁾ + b⁽ˡ⁾
　 a⁽ˡ⁾ = σ(z⁽ˡ⁾)
　 L = loss(a⁽ᴸ⁾, y)

逆伝播
　 δ⁽ᴸ⁾ = ∂L/∂a⁽ᴸ⁾ ⊙ σ'(z⁽ᴸ⁾)
　 δ⁽ˡ⁾ = ((W⁽ˡ⁺¹⁾)T δ⁽ˡ⁺¹⁾) ⊙ σ'(z⁽ˡ⁾)
　 ∂L/∂W⁽ˡ⁾ = δ⁽ˡ⁾ (a⁽ˡ⁻¹⁾)T
　 ∂L/∂b⁽ˡ⁾ = δ⁽ˡ⁾

計算量は順伝播の数倍、メモリは中間量 a⁽ˡ⁾ の保存に O(L) 必要（→ Gradient Checkpointing で削減）。詳細は章11 で。

§2.2 学習率スケジューリング — 動的な歩幅調整

固定 η より、学習中に動的に変える方が一般に高性能：

  • Step Decay：一定エポックごとに η を 1/10 に
  • Exponential Decay：ηt = η₀ · γt（γ < 1）
  • Cosine Annealing：余弦関数でなめらかに減衰、SGDR・現代LLM学習で人気
  • One Cycle Policy：η を一旦上げて下げる、Fast.ai が普及
  • ReduceLROnPlateau：検証損失が改善しないと η を下げる
  • Warmup：最初の数千ステップで η を 0 から線形に上げる。Transformer 学習で必須

§2.3 2次最適化と準ニュートン法

ヘッセ情報を使う Newton 法 θ ← θ − H⁻¹∇L は局所2次収束で強力だが、深層学習では H が巨大で実装困難。準ニュートン法（BFGS、L-BFGS） はヘッセを近似で更新し、メモリ効率を改善。最近の研究：K-FAC（ブロック対角近似）、Shampoo（テンソル分解）、Sophia（2023年 LLM 用2次オプティマイザ）。

§2.4 鞍点問題と高次元最適化

低次元での直感とは異なり、高次元では鞍点が圧倒的に多い（Dauphin et al. 2014）。理由：n 次元では H の固有値が全て同符号になる確率が指数的に小さい。SGD のノイズが鞍点脱出に貢献するため、深層学習では SGD/Adam が経験的に有効。完全な Newton 法は鞍点に直接落ちる可能性があり、深層 NN には不向き。

FIG 4.7 ／ 高次元損失曲面のイメージ。NNでは鞍点が指数的に多く、SGDの確率的ノイズがその脱出に役立つ。経験的に SGD/Adam が見つけるフラットなミニマは汎化性能が良い。

§3.1 数値微分 — 定義から計算する

import numpy as np # 中心差分による数値微分（前進差分より精度高い） def numerical_gradient(f, x, h=1e-4): grad = np.zeros_like(x) for i in range(len(x)): tmp = x[i] x[i] = tmp + h f_plus = f(x) x[i] = tmp - h f_minus = f(x) grad[i] = (f_plus - f_minus) / (2 * h) x[i] = tmp # 元に戻す return grad # f(x, y) = x² + 2y² の (1, 2) での勾配を計算 def f(x): return x[0]**2 + 2 * x[1]**2 x = np.array([1.0, 2.0]) grad = numerical_gradient(f, x) print("数値勾配:", grad) # [2.0, 8.0] print("理論勾配:", [2*x[0], 4*x[1]]) # [2.0, 8.0]
OUTPUT
数値勾配: [2.00000002 8.00000010]
理論勾配: [2.0, 8.0]

§3.2 PyTorch の自動微分

import torch # requires_grad=True で自動微分の対象に x = torch.tensor([1.0, 2.0], requires_grad=True) # 順伝播 y = x[0]**2 + 2 * x[1]**2 # 逆伝播 y.backward() print("x.grad =", x.grad) # [2.0, 8.0] # NN の簡単な学習 W = torch.randn(3, 2, requires_grad=True) b = torch.zeros(3, requires_grad=True) xin = torch.tensor([1.0, 2.0]) target = torch.tensor([1.0, 0.0, 0.0]) for step in range(100): out = W @ xin + b loss = ((out - target)**2).sum() loss.backward() with torch.no_grad(): W -= 0.05 * W.grad b -= 0.05 * b.grad W.grad.zero_() b.grad.zero_() if step % 20 == 0: print(f"step={step}, loss={loss.item():.4f}")

§3.3 勾配降下法のスクラッチ実装

# 2次関数 f(x, y) = (x-3)² + (y+2)² を最小化 # 真の最小は (3, -2) def loss(params): x, y = params return (x - 3)**2 + (y + 2)**2 def grad(params): x, y = params return np.array([2*(x-3), 2*(y+2)]) params = np.array([0.0, 0.0]) # 初期値 lr = 0.1 history = [params.copy()] for step in range(50): g = grad(params) params = params - lr * g history.append(params.copy()) if step % 10 == 0: print(f"step={step:3d}, params=({params[0]:6.3f}, {params[1]:6.3f}), loss={loss(params):.4f}") print(f"最終: ({params[0]:.4f}, {params[1]:.4f})")
OUTPUT
step= 0, params=( 0.600, -0.400), loss=8.3200
step= 10, params=( 2.678, -1.785), loss=0.1493
step= 20, params=( 2.965, -1.977), loss=0.0017
step= 30, params=( 2.996, -1.998), loss=0.0000
step= 40, params=( 3.000, -2.000), loss=0.0000
最終: (3.0000, -2.0000)

§3.4 Adam のスクラッチ実装

class Adam: def __init__(self, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8): self.lr = lr self.beta1 = beta1 self.beta2 = beta2 self.eps = eps self.m = None # 1次モーメント self.v = None # 2次モーメント self.t = 0 # タイムステップ def step(self, params, grads): if self.m is None: self.m = np.zeros_like(params) self.v = np.zeros_like(params) self.t += 1 self.m = self.beta1 * self.m + (1 - self.beta1) * grads self.v = self.beta2 * self.v + (1 - self.beta2) * grads**2 m_hat = self.m / (1 - self.beta1**self.t) v_hat = self.v / (1 - self.beta2**self.t) return params - self.lr * m_hat / (np.sqrt(v_hat) + self.eps) opt = Adam(lr=0.1) params = np.array([0.0, 0.0]) for step in range(100): params = opt.step(params, grad(params)) print(f"Adam 最終: ({params[0]:.4f}, {params[1]:.4f})")
OUTPUT
Adam 最終: (3.0000, -2.0000)
（同じ問題で同じ最適解に到達。実用ではノイズ・非凸性で差が出る）

§3.5 損失曲面の可視化

SECTION FOUR

シミュレーション

学習率の効果、鞍点問題、損失曲面地形の3つの思考実験。

シミュレーション

SIM 1 — 学習率と発散・収束

f(x) = x² の最小化を学習率を変えて実行。η の選択が結果を完全に決定する。
>> 初期値 x₀ = 10、勾配 g = 2x、更新 x ← x - η·2x = (1-2η)x

η=0.1（適切）: 10 → 8.00 → 6.40 → 5.12 → ... → 0.000　【収束】
η=0.5（境界）: 10 → 0.00 → 0.00　　　　　　　　　　【1ステップで到達】
η=0.9（やや大）: 10 → -8.0 → 6.40 → -5.1 → ... → 0.000　【振動しつつ収束】
η=1.0（限界）: 10 → -10 → 10 → -10 → ...　　　　　　【無限振動】
η=1.5（大）: 10 → -20 → 40 → -80 → ...　　　　　　【発散】

>> 凸関数で Lipschitz 係数 L=2 なので、η < 1/L = 0.5 が収束保証境界

SIM 2 — 鞍点問題

f(x, y) = x² - y² の鞍点 (0, 0) 付近での挙動。SGD のノイズの効果。
>> 初期値 (1, 0.001)、勾配 = (2x, -2y)

[純粋GD] step→ (0.80, 0.0008) → (0.64, 0.00064) → ...
    y方向に勾配ほぼ0 → 鞍点に張り付き　【脱出できない】

[SGD（ノイズ追加）]
  step → (0.81, 0.043) → (0.65, 0.121) → (0.52, 0.298)
  　　 → (0.42, 0.605) → (0.33, 1.21) → 発散
    y方向のノイズが拡大 → 鞍点から離脱

>> SGD のノイズが高次元最適化で必須な理由

SIM 3 — Adam vs SGD on Rosenbrock 関数

最適化のテスト関数として有名な Rosenbrock 関数 f(x,y) = (1-x)² + 100(y-x²)² の収束を比較。
>> 真の最小：(1, 1)、損失 0
>> 初期値：(-1, 1)、平坦な細長い谷を持つ

SGD (η=0.001):
  100 steps:　 loss = 4.39
  500 steps:　 loss = 1.83
  1000 steps:　loss = 0.62
  5000 steps:　loss = 0.034

Adam (lr=0.01):
  100 steps:　 loss = 1.21
  500 steps:　 loss = 0.012
  1000 steps:　loss = 5.3e-6
  5000 steps:　loss = 1.1e-12 ← ほぼ完全収束

>> Adam が SGD より遥かに高速、特に細長い谷で効果絶大
SECTION FIVE

問題

G検定形式の演習問題 20問。微分・最適化のすべての概念を試験レベルで確認する。

問題

PROBLEM 1
f(x) = x³ - 2x + 1 の x=2 における微分値は。
① 4　 ② 8　 ③ 10　 ④ 12
ANSWER ③ f'(x) = 3x² - 2、f'(2) = 12 - 2 = 10。
PROBLEM 2
勾配ベクトル ∇f(x) が指す方向は。
① 最も急に減少する方向　 ② 最も急に増加する方向　 ③ 関数の零点　 ④ 接平面の法線
ANSWER ② 勾配は『最急上昇方向』を指す。最小化問題では負の勾配方向 -∇f に進む（最急降下法・SGD・Adam）。等高線に垂直になる幾何学的事実も重要。
PROBLEM 3
合成関数 y = f(g(x)) の微分（連鎖律）として正しいのは。
① dy/dx = f'(g(x)) · g'(x)　 ② dy/dx = f'(x) · g'(x)　 ③ dy/dx = f'(x) + g'(x)　 ④ dy/dx = f(g'(x))
ANSWER ① 連鎖律：外側の関数の微分（g(x) で評価）と内側の微分の積。NNの逆伝播は連鎖律を計算グラフ上で機械的に適用したもの。
PROBLEM 4
ヘッセ行列 H の固有値がすべて正のとき、その臨界点は。
① 極大　 ② 極小　 ③ 鞍点　 ④ 縮退
ANSWER ② H 正定値 → 周囲が下に凸 → 極小。負定値なら極大、固有値に正負混在なら鞍点。
PROBLEM 5
テイラー展開を機械学習で用いる典型例は。
① 決定木の分岐　 ② Newton 法、XGBoost の更新方向決定　 ③ データ拡張　 ④ ハッシュ化
ANSWER ② テイラー展開は局所近似で、1次近似→勾配降下、2次近似→Newton 法・XGBoost。誤差解析にも頻出。
PROBLEM 6
凸関数の重要な性質として正しいのは。
① 局所最小と大域最小が一致する　 ② 必ず微分不可能　 ③ 学習が遅い　 ④ 制約付き最適化に使えない
ANSWER ① 凸関数では局所最小＝大域最小。これが線形回帰、ロジスティック回帰、SVM などが理論的に扱いやすい根拠。NN は非凸なので局所最小が複数存在しうる。
PROBLEM 7
勾配降下法の更新式 θ ← θ − η ∇L における η が大きすぎるとき起きうる現象は。
① 過学習　 ② 振動・発散　 ③ 計算量増　 ④ パラメータ削減
ANSWER ② η が大きすぎると更新が最小を行き過ぎ、振動・発散。一般に Lipschitz 連続な勾配なら η ≤ 1/L が収束保証境界。
PROBLEM 8
SGD（確率的勾配降下）の利点として正しくないものは。
① 反復が高速　 ② ノイズが鞍点脱出に貢献　 ③ メモリ効率　 ④ 必ず大域最適に収束
ANSWER ④ SGDは局所最適や鞍点に陥る可能性があり、大域最適への収束保証はない（凸関数では確率収束）。それでも実用上は非常に有効、特に NN ではノイズが正則化として機能。
PROBLEM 9
Momentum オプティマイザが解決する問題は。
① 学習率自動調整　 ② 勾配ノイズによる振動と遅い収束　 ③ 正則化　 ④ データ拡張
ANSWER ② Momentum は勾配の指数移動平均を慣性として保持し、振動を抑制しつつ収束を加速する。物理的には『谷を転がるボール』のアナロジー。
PROBLEM 10
Adam オプティマイザの特徴として正しいのは。
① Momentum と RMSProp（適応学習率）の組み合わせ＋バイアス補正　 ② 2次オプティマイザ　 ③ 学習率不要　 ④ NN以外でしか動かない
ANSWER ① Adam は1次モーメント（Momentum）と2次モーメント（RMSProp）を併用し、初期段階のバイアス補正を加えた1次オプティマイザ。深層学習のデファクト。
PROBLEM 11
AdamW が通常の Adam と異なる点は。
① 重み減衰を勾配計算から分離した　 ② Momentum を削除した　 ③ 学習率を固定した　 ④ 全く同じ
ANSWER ① Adam で L2 正則化を素朴に加えると2次モーメントとの相互作用で正則化が弱まる。AdamW は重み減衰 λθ を更新式の最後で直接引くことで、SGD と同じ正則化効果を実現。LLM 学習の事実上の標準。
PROBLEM 12
ラグランジュ未定乗数法が解く問題のタイプは。
① 無制約最適化　 ② 等式制約付き最適化　 ③ 線形回帰　 ④ クラスタリング
ANSWER ② ラグランジュ関数 L(x, λ) = f(x) - λg(x) で等式制約 g(x)=0 のもとでの f(x) の最適化を扱う。不等式制約まで拡張すると KKT 条件。SVM、エントロピー最大化で核心。
PROBLEM 13
バックプロパゲーション（逆伝播）の理論的根拠は。
① 連鎖律　 ② 中心極限定理　 ③ 大数の法則　 ④ ベイズの定理
ANSWER ① 逆伝播は連鎖律を計算グラフ上で機械的に適用するアルゴリズム。1986年に Rumelhart, Hinton, Williams が普及させた。これがなければ深層学習は実用化できなかった。
PROBLEM 14
Newton 法と勾配降下法の主な違いは。
① Newton 法は1次情報のみ使用　 ② Newton 法はヘッセ行列の逆を使う2次最適化、局所的に高速　 ③ 全く同じ　 ④ 勾配降下法は教師あり
ANSWER ② Newton 法 θ ← θ − H⁻¹∇L は2次情報を使い、凸問題で局所的に2次収束。深層学習では H が巨大で実装困難なため、Adam 等の対角近似が実用的妥協。
PROBLEM 15
学習率 Warmup が必要な典型的状況は。
① 線形回帰　 ② 大バッチ学習や Transformer 学習の初期段階　 ③ 決定木　 ④ K-means
ANSWER ② 大バッチや Transformer では、初期の勾配が大きく不安定なため、η を 0 から徐々に上げる Warmup（数千ステップ）が学習安定化に必須。Cosine Annealing と組み合わせるのが現代標準。
PROBLEM 16
高次元損失曲面で最も多く存在するのは。
① 局所最小　 ② 鞍点　 ③ 極大　 ④ 平坦領域のみ
ANSWER ② Dauphin et al. 2014 が示したように、高次元ではヘッセ行列のすべての固有値が同符号になる確率が指数的に小さく、鞍点が圧倒的に多い。SGDのノイズが脱出に貢献。
PROBLEM 17
活性化関数 sigmoid σ(x) = 1/(1+e^{-x}) の微分は。
① 1　 ② σ(x)　 ③ σ(x)(1 - σ(x))　 ④ e^{-x}
ANSWER ③ σ'(x) = σ(x)(1 - σ(x))。出力値で書ける（再利用効率高い）。最大値は σ(0)=0.5 の点で 0.25 と小さく、深層NNで勾配消失の原因に。これがReLU普及の理由の一つ。
PROBLEM 18
バッチサイズを大きくした際の典型的なトレードオフは。
① 並列効率↑、汎化性能やや低下（シャープミニマへ）　 ② 全てが向上　 ③ 全てが悪化　 ④ メモリは変わらない
ANSWER ① 大バッチは並列効率・メモリ効率↑だが、勾配ノイズ↓で『シャープなミニマ』に収束しやすく汎化が劣化する傾向。Linear Scaling Rule＋Warmup で部分的補正。
PROBLEM 19
Gradient Checkpointing が解決する問題は。
① 計算速度の遅さ　 ② 学習中の活性化保存に必要なメモリの大きさ　 ③ ハイパーパラメータ選択　 ④ データの不均衡
ANSWER ② 通常の逆伝播は順伝播の全活性化を保存（O(L) メモリ）。Checkpointing は一部だけ保存し、必要時に再計算でメモリを O(√L) に。計算は 1.3〜2 倍に増えるが、巨大モデル学習で必須。
PROBLEM 20
KKT 条件が登場する典型問題は。
① 不等式制約付き最適化、SVM など　 ② クラスタリング　 ③ 単純線形回帰　 ④ K-NN
ANSWER ① KKT（Karush-Kuhn-Tucker）条件は不等式制約付き最適化で必要十分（凸の場合）の最適性条件。停留性・主実行可能性・双対実行可能性・相補性の4つ。SVMの双対形式、エントロピー最大化など。

章のまとめ

本章では、機械学習の学習過程を支える微分・最適化の全体像を扱いました。微分・偏微分 から 勾配ベクトル（最急上昇方向）、連鎖律（バックプロパゲーションの心臓）、ヘッセ行列（曲率の判定）、テイラー展開（局所近似）、凸関数と凸最適化（解析可能性）、勾配降下法（最も基本）、SGD（実用化）、Momentum と Adam（現代オプティマイザ）、ラグランジュ未定乗数法（制約付き最適化）。応用では NNの逆伝播、学習率スケジューリング、2次最適化、鞍点問題を、ハンズオンでは数値微分・PyTorch自動微分・GD/Adamスクラッチ実装・損失曲面可視化を、シミュレーションでは学習率・鞍点・Rosenbrock関数を扱いました。次章は確率・統計、その後の章で機械学習・NNと進みます。すべての『学習』は、ここで扱った『下山の数学』に基づいています。
