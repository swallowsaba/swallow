章6 — 情報理論Chapter 6 — Information Theory

1948年、Claude Shannon は『通信の数学的理論』で 情報を厳密に定義しました。本章では、自己情報量から、エントロピー、相互情報量、KLダイバージェンス、クロスエントロピー、情報源符号化定理までを、基本・応用・ハンズオン・シミュレーション・問題の5部構成で解説します。NN のクロスエントロピー損失、決定木の情報利得、VAEのELBO、t-SNE/UMAP の KL 最小化 ── 機械学習の損失関数の意味は、すべて情報理論で説明されます。

G-KENTEI · BASICS · CHAPTER 6

情報を、ビットで測る。
そして機械を学ばせる。

『1ビットの情報』とは何でしょう。Shannonの答えは：『等確率の2択を1つに絞る情報量』。この発想から、エントロピー（不確実性）、相互情報量（共有された情報）、KLダイバージェンス（2つの分布の隔たり）が定義され、これらが現代の機械学習における 損失関数の正体 を与えます。クロスエントロピー損失、情報利得（決定木）、ELBO（VAE）、KL 最小化（t-SNE, UMAP）── すべて本章の言葉です。

SECTION ONE

基本

自己情報量・エントロピー・結合エントロピー・条件付きエントロピー・相互情報量・KL/JS/CE・微分エントロピー・Shannon符号化定理の10節。

基本

§1.1 自己情報量 — 1つの事象が運ぶ情報

確率 p で起こる事象が運ぶ情報量：

自己情報量　 h(x) = -log P(x)　 (単位：底2なら bit、e なら nat)

性質
　・p=1 → h=0　 確実な事象は情報なし
　・p→0 → h→∞　 珍しい事象ほど情報量大
　・独立な複数事象 → 情報量は和（log の性質）
▎ 直観
『明日太陽が昇る』（p≈1）→ ほぼ情報なし。『隕石が落ちる』（p≈0）→ 巨大な情報。情報量は『珍しさ』の対数尺度。

§1.2 エントロピー H(X) — 不確実性の平均

シャノンエントロピー　 H(X) = -Σx P(x) log P(x) = E[h(X)]

性質
　・H(X) ≥ 0（常に非負）
　・K個の値をとる確率変数で、最大値は log K（一様分布のとき）
　・最小値 0（決定的なとき、つまり1つの値の確率が1）
FIG 6.1 ／ バイナリエントロピーは p=0.5 でピーク 1bit。決定木の情報利得、二値分類のCE損失で頻出。

§1.3 結合エントロピーと §1.4 条件付きエントロピー

結合エントロピー　 H(X, Y) = -Σ P(x, y) log P(x, y)

条件付きエントロピー　 H(Y|X) = EX[H(Y|X=x)]
　　　　　　　　　　 = -Σ P(x, y) log P(y|x)

連鎖律　 H(X, Y) = H(X) + H(Y|X) = H(Y) + H(X|Y)

独立なら　 H(X, Y) = H(X) + H(Y)　 (H(Y|X) = H(Y))

§1.5 相互情報量 — XとYが共有する情報

相互情報量
　 I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)
　 　　　 = H(X) + H(Y) - H(X, Y)
　 　　　 = Σ P(x, y) log [P(x, y) / (P(x)P(y))]

性質　 I(X; Y) ≥ 0、 I(X; Y) = 0 ⇔ X, Y 独立
　 任意の依存関係（非線形含む）を捉える
FIG 6.2 ／ エントロピーのベン図。I(X;Y) は X と Y が共有する情報、H(X|Y) は Y を知った後の X の残りの不確実性。

§1.6 KLダイバージェンス — 2つの分布の隔たり

KLダイバージェンス　 KL(p || q) = Σ p(x) log [p(x) / q(x)]

性質
　・KL ≥ 0（Gibbs の不等式）、 KL = 0 ⇔ p = q
　・非対称： KL(p||q) ≠ KL(q||p)（だから『距離』ではない）
　・q(x)=0 で p(x)≠0 の点があると KL = ∞

2つの方向の意味
　 KL(p_data || p_model)　 forward KL → モード被覆（mode covering）
　 KL(p_model || p_data)　 reverse KL → モード探索（mode seeking）、VAE で使う

§1.7 JSダイバージェンス — 対称化された KL

JSダイバージェンス　 JS(p, q) = (1/2)KL(p || m) + (1/2)KL(q || m)　 (m = (p+q)/2)

性質　 対称、有界（log 2 が上限）、平方根は距離関数
応用　 GAN の損失（原論文）、t-SNE のクラスタ評価

§1.8 クロスエントロピー — 学習の損失関数

クロスエントロピー　 H(p, q) = -Σ p(x) log q(x)

分解　 H(p, q) = H(p) + KL(p || q)
　 H(p) は真の分布のエントロピー（固定）→ H(p, q) 最小化 ⇔ KL(p||q) 最小化

機械学習　 p = 真のラベル分布（典型 one-hot）、q = モデルの予測
　 二値分類　 CE = -[y log p + (1-y) log(1-p)]
　 多クラス　 CE = -Σk yk log pk
FIG 6.3 ／ CE = H(p) + KL(p||q)。NN学習で CE を最小化すると、自動的に予測分布 q が真の分布 p に近づく。

§1.9 微分エントロピー（連続変数）

微分エントロピー　 h(X) = -∫ f(x) log f(x) dx

注意　 離散と違い 負の値を取りうる。スケール不変でもない（h(aX) = h(X) + log|a|）

正規分布 N(μ, σ²) のエントロピー　 h = (1/2) log(2πe σ²)
固定分散下で正規分布が最大エントロピー　 → 最大エントロピー原理

§1.10 Shannonの情報源符号化定理

『1記号あたり平均 H(X) ビットで符号化できる』。これが下限で、それより短くは不可能。Huffman 符号、算術符号、近年の 言語モデルベース圧縮（DeepMindの 2023 年研究）まで、この定理が現実の圧縮アルゴリズムの理論的基礎。

SECTION TWO

応用

NNのCE損失、決定木の情報利得、VAE/ELBO、t-SNE/UMAPの4節。

応用

§2.1 NNのクロスエントロピー損失 — MLEと等価

Softmax 出力 + CE 損失は、確率モデルの MLE と数学的に等価：

多クラス分類の損失
　 L = -(1/N) Σi Σk yi,k log ŷi,k

勾配の美しい単純化
　 Softmax + CE の出力前の logit に対する勾配 = ŷ - y
　 これが NN 学習の計算効率の根拠

§2.2 決定木の情報利得

情報利得（IG）　 IG(D, A) = H(D) - H(D|A)
　 属性 A で分割したときのエントロピー減少量

ID3 / C4.5 / CART　 IG が最大の属性で分岐
ジニ係数　 1 - Σpᵢ²　 IG の代替、計算高速

§2.3 VAE の ELBO

VAE 損失 = -ELBO
　 = Eq(z|x)[-log p(x|z)] + KL(q(z|x) || p(z))
　 　 ↑ 再構成損失　　　　 　 ↑ 事前との隔たり（正則化）

意味　 表現 z で x を復元しつつ、潜在分布を事前 N(0, I) に近づける

§2.4 t-SNE / UMAP の KL 最小化

t-SNE（van der Maaten 2008）：高次元空間でのペア類似度 pij と低次元での qij の KL(p || q) を最小化することで、近い点は近く、遠い点は遠くに配置。UMAP も同様の情報論的目的関数を使う。可視化・次元削減の現代標準。

FIG 6.4 ／ 機械学習の主要目的関数はすべて情報理論で書ける。これが『学習を統一的に理解する』情報論的視座。
SECTION THREE

ハンズオン

エントロピー、KL、CE/Softmax、相互情報量、Huffman の5節。

ハンズオン

§3.1 エントロピー計算

import numpy as np def entropy(p, base=2): p = np.asarray(p) p = p[p > 0] # 0 log 0 = 0 return -np.sum(p * np.log(p) / np.log(base)) # 公平コイン print("公平コイン:", entropy([0.5, 0.5])) # 1.0 bit # 偏ったコイン print("p=0.9:", entropy([0.9, 0.1])) # 0.469 bit # 決定的 print("確定:", entropy([1.0, 0.0])) # 0.0 bit # 8面サイコロ print("8面サイコロ:", entropy([1/8]*8)) # 3.0 bit

§3.2 KLダイバージェンスの非対称性

def kl_div(p, q, base=2): p, q = np.asarray(p), np.asarray(q) mask = p > 0 return np.sum(p[mask] * np.log(p[mask] / q[mask]) / np.log(base)) p = [0.7, 0.2, 0.1] q = [0.3, 0.4, 0.3] print(f"KL(p||q) = {kl_div(p, q):.4f}") # 0.376 print(f"KL(q||p) = {kl_div(q, p):.4f}") # 0.408 ← 異なる！ print("非対称: KL(p||q) ≠ KL(q||p)")

§3.3 Softmax + クロスエントロピー

def softmax(z): z = z - z.max() # 数値安定のためのトリック e = np.exp(z) return e / e.sum() def cross_entropy(y_true, y_pred): y_pred = np.clip(y_pred, 1e-12, 1) # log(0) 回避 return -np.sum(y_true * np.log(y_pred)) # 真のラベル (one-hot)、予測 logit y_true = np.array([0, 1, 0]) # class 1 が正解 logits = np.array([1.0, 3.0, 0.5]) y_pred = softmax(logits) print("確率:", y_pred) print("CE 損失:", cross_entropy(y_true, y_pred)) # 0.146 — クラス1の予測確率が log(0.864) ≈ -0.146 # 勾配（Softmax+CE は logits に対し簡潔！） grad_logits = y_pred - y_true print("勾配 = ŷ - y:", grad_logits)

§3.4 相互情報量による特徴選択

from sklearn.feature_selection import mutual_info_classif from sklearn.datasets import load_iris X, y = load_iris(return_X_y=True) mi = mutual_info_classif(X, y, random_state=42) for i, m in enumerate(mi): print(f"特徴 {i}: MI = {m:.4f}") # 花弁の長さ・幅は分類に高 MI、萼片の特徴は相対的に低い

§3.5 Huffman符号化

import heapq from collections import Counter def huffman_codes(freq): heap = [[w, [sym, ""]] for sym, w in freq.items()] heapq.heapify(heap) while len(heap) > 1: lo = heapq.heappop(heap); hi = heapq.heappop(heap) for pair in lo[1:]: pair[1] = '0' + pair[1] for pair in hi[1:]: pair[1] = '1' + pair[1] heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:]) return dict((sym, code) for sym, code in heap[0][1:]) text = "abracadabra" freq = Counter(text) codes = huffman_codes(freq) print("符号:", codes) # 頻度高い 'a' は短く、'd' などは長い符号
SECTION FOUR

シミュレーション

エントロピー直観、KL非対称性、CE=MLEの3つの思考実験。

シミュレーション

SIM 1 — エントロピーの直観

>> 様々な確率分布のエントロピー比較

[0.5, 0.5]　　　 → H = 1.000 bit （最大、最も予測不能）
[0.7, 0.3]　　　 → H = 0.881 bit
[0.9, 0.1]　　　 → H = 0.469 bit
[0.99, 0.01]　　 → H = 0.081 bit
[1.0, 0.0]　　　 → H = 0.000 bit （確定的）

[0.25]×4 → H=2.000 bit （4面サイコロ）
[0.125]×8 → H=3.000 bit （8面）
log K のスケール、K が増えるほど不確実性最大値も増える

SIM 2 — KL の非対称性で何が起きるか

>> 2峰分布 p と単峰分布 q を考える
　 p: (0.4, 0.1, 0.4, 0.1) ※双峰
　 q: (0.25)×4　　　　　　　 ※一様

KL(p||q) = 0.193 （forward）
　Modes Covering: q は p のすべてのモードをカバーすべき
　→ q を p の全範囲に広げる
　→ 真の分布が複数モードを持つとき重要

KL(q||p) = 0.155 （reverse）
　Mode Seeking: q は p の高確率領域に集中すべき
　→ q が p のひとつのモードに張り付く
　→ VAEで使われる（複数モードのうち1つ近似でも可）

SIM 3 — CE 最小化 = MLE は同じこと

>> 多クラス分類で CE 損失を最小化することは、
>> 多項分布の負の対数尤度を最小化することに等しい

データ点 (x_i, y_i)、モデル q_θ(y|x)
　 L_CE = -(1/N) Σ_i log q_θ(y_i | x_i)
　 L_NLL = -(1/N) Σ_i log p(y_i | x_i; θ)
　 完全に同じ式！

>> だから NN 学習は MLE であり、確率モデル学習である
>> 正則化を加えると MAP 推定に
SECTION FIVE

問題

G検定形式の演習問題 20問。

問題

PROBLEM 1
確率 p で起こる事象の自己情報量は。
① p　 ② -p　 ③ -log p　 ④ 1-p
ANSWER ③ h(x) = -log P(x)。確率が小さいほど情報量が大きい。
PROBLEM 2
エントロピー H(X) が最大となるのは。
① 確定的　 ② 一様分布　 ③ 最も歪んだ分布　 ④ 連続のみ
ANSWER ② 離散K値の場合、一様分布で H = log K が最大。確定的は H=0 で最小。
PROBLEM 3
公平なコインのエントロピーは。
① 0 bit　 ② 0.5 bit　 ③ 1 bit　 ④ 2 bit
ANSWER ③ H = -(0.5 log₂0.5 + 0.5 log₂0.5) = 1 bit。公平コイン1回分の情報量＝1bit。
PROBLEM 4
相互情報量 I(X;Y) と H の関係で誤りはどれか。
① I(X;Y) = H(X) + H(Y) - H(X,Y)　 ② I(X;Y) = H(X) - H(X|Y)　 ③ I(X;Y) ≥ 0　 ④ I(X;Y) は負の値も取りうる
ANSWER ④ 相互情報量は常に非負（I≥0）。X,Y独立なら 0、強く依存するなら大きい値。
PROBLEM 5
KLダイバージェンス KL(p||q) について誤りはどれか。
① 常に非負　 ② p=q なら 0　 ③ 対称：KL(p||q) = KL(q||p)　 ④ 距離関数ではない
ANSWER ③ KL は非対称。これが距離関数でない理由。対称化したのが JS ダイバージェンス。
PROBLEM 6
クロスエントロピー H(p, q) と KL の関係は。
① H(p,q) = H(p) - KL(p||q)　 ② H(p,q) = H(p) + KL(p||q)　 ③ 関係なし　 ④ H(p,q) = KL(q||p)
ANSWER ② CE = H(p) + KL(p||q)。H(p) は学習データ固定なので、CE最小化 = KL最小化 = q を p に近づける。
PROBLEM 7
NN の分類で『Softmax + CE 損失』が標準的に使われる理由として最も適切なのは。
① 計算が遅い　 ② 多項分布のMLEと等価、勾配が ŷ-y で簡潔　 ③ パラメータ数を減らす　 ④ 正則化のため
ANSWER ② 多項分布の負の対数尤度であり、ロジットへの勾配が ŷ-y となる美しい性質を持つ。これが NN 分類学習の効率を支える。
PROBLEM 8
決定木の分岐基準『情報利得（IG）』の定義は。
① H(D) - H(D|A) （分割前後のエントロピー差）　 ② H(D) のみ　 ③ Var(D)　 ④ 平均値
ANSWER ① IG は属性Aで分割したときのエントロピー減少量。これが大きい属性を選ぶことで予測能力の高い分岐を作る。ID3/C4.5の基準。
PROBLEM 9
VAE の損失関数 -ELBO の構成として正しいのは。
① 再構成損失 + KL(q(z|x) || p(z))　 ② CE のみ　 ③ MSE のみ　 ④ 正則化なし
ANSWER ① -ELBO = 再構成損失 + KL正則化。エンコーダ出力が事前分布N(0,I)に近づくよう KL で正則化。
PROBLEM 10
t-SNE が最小化する量は。
① ユークリッド距離　 ② 高次元と低次元の分布の KL ダイバージェンス　 ③ MSE　 ④ Hinge損失
ANSWER ② 高次元での近傍類似度分布 P と低次元での Q の KL(P||Q) を最小化。クラスタを可視化する標準手法。
PROBLEM 11
『最大エントロピー原理』が示すのは。
① 与えられた制約条件下で最も不確実な分布を選ぶ　 ② 最頻分布を選ぶ　 ③ 最頻値　 ④ 中央値
ANSWER ① 既知の制約以外について余計な仮定をしない原則。例：平均と分散だけ既知 → 正規分布が最大エントロピー解。
PROBLEM 12
微分エントロピーが離散エントロピーと異なる点で正しいのは。
① 負の値も取りうる　 ② 必ず非負　 ③ 必ず整数　 ④ 必ず log K 以下
ANSWER ① 連続分布の微分エントロピーは負になりうる（例：分散が極小な正規分布）。スケール変換にも不変でない。
PROBLEM 13
Shannon の情報源符号化定理が示す下限は。
① 1記号あたり 1 bit　 ② 1記号あたり log K bit　 ③ 1記号あたりエントロピー H(X) bit　 ④ 上限なし
ANSWER ③ 平均符号長 ≥ H(X)。これがロスレス圧縮の理論的下限。Huffman 符号は H+1 までで実現。
PROBLEM 14
GAN（原論文）の損失関数が最小化する分布間距離は。
① KL（forward）　 ② JSダイバージェンス　 ③ MSE　 ④ ユークリッド
ANSWER ② Goodfellow の元のGANは JS ダイバージェンスを最小化することが示されている。Wasserstein GAN は別の距離（Wasserstein距離）を使う。
PROBLEM 15
相互情報量を使った特徴選択の利点は。
① 計算速度のみ　 ② 線形・非線形両方の依存関係を捉える　 ③ ピアソン相関の代替で線形のみ　 ④ ラベル不要
ANSWER ② Pearson相関は線形のみだが、MIは任意の関数関係（U字、二次、複雑な依存）を捉える。教師あり特徴選択で強力。
PROBLEM 16
エントロピーが大きい状況の例として最も適切なのは。
① 確定的シーン　 ② 完全にランダムなビット列　 ③ 単一値の連続　 ④ 既知の数列
ANSWER ② 完全にランダム＝予測不能＝エントロピー最大。圧縮も不可能（log K bit/記号）。
PROBLEM 17
InfoNCE（対照学習で使う）の役割は。
① CE損失と同じ　 ② 正サンプルと負サンプルの間で相互情報量の下限を推定して最大化　 ③ MSE　 ④ 正則化のみ
ANSWER ② InfoNCE は対照学習（SimCLR, MoCo）の損失。表現空間で正ペア類似度を負ペアより高くする、MI下限の最大化として解釈可能。
PROBLEM 18
2分類タスクで真ラベル y=1、予測確率 p=0.8 のとき、二値CE損失は。
① -log(0.8)　 ② -log(0.2)　 ③ 0.5　 ④ 1.0
ANSWER ① CE = -[y log p + (1-y) log(1-p)]、y=1 なら CE = -log p = -log(0.8) ≈ 0.223。
PROBLEM 19
『Label Smoothing』は何を変更するか。
① one-hotラベル y を [1-ε, ε/(K-1), ...] に置き換え、過信を防ぐ　 ② 損失関数を別物に変更　 ③ 学習率　 ④ バッチサイズ
ANSWER ① 真ラベルの確信度を 1.0 から 1-ε（典型 0.9）に下げる正則化。NNの過信を防ぎ、汎化を改善。情報論的にはエントロピーの底上げ。
PROBLEM 20
KL(p||q) と KL(q||p) が大きく違う典型的状況は。
① 単峰分布同士の中心ずれ　 ② 多峰分布 vs 単峰分布　 ③ 全く同じ分布　 ④ KLは常に対称
ANSWER ② 多峰の p を単峰の q で近似する場合、KL(p||q) は mode covering（q を広く）を要求し、KL(q||p) は mode seeking（q を一つに集中）を要求。VAEは reverse KL を使い結果として1つのモードに集中しやすい。

章のまとめ

自己情報量から始め、エントロピー（不確実性）、結合・条件付きエントロピー、相互情報量、KL/JSダイバージェンス、クロスエントロピー、微分エントロピー、Shannonの情報源符号化定理を扱いました。応用では NN の CE 損失（多項MLE）、決定木の情報利得、VAE の ELBO、t-SNE/UMAP の KL 最小化を確認。ハンズオンでエントロピー計算・KL 非対称性・Softmax+CE 勾配・MI 特徴選択・Huffman 符号化、シミュレーションでエントロピー直観・KL方向の意味・CE=MLE 等価性。次章は機械学習パラダイム。教師あり/なし/強化学習、ハイパラ調整、過学習対策へ進みます。
