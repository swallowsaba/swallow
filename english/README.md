# EigoMaster｜アメリカ英語 発音スタジオ

日本人がアメリカ英語を **ビジネスレベル（CEFR B2〜C1 / TOEIC 800+）** までマスターするための統合学習Webアプリです。
サーバー・データベース・有料APIを一切使わず、**GitHub Pages（完全静的）** で動作します。
学習進捗はすべてお使いの端末内（`localStorage`）に保存され、JSONで書き出し／読み込みできます。

> **実装状況：全モジュール実装済み（単語・文法・フォニックス・発音チェック・リンキング・リーディング・シャドーイング・カタカナ変換・レベル診断）**

---

## 8つの学習モジュール

| モジュール | 画面 | 内容 |
|---|---|---|
| 単語（SRS） | `#/vocab` | レベル別（A1〜C1）× 3モード（フラッシュカード／4択／タイピング）。SM-2簡易版の間隔反復で出題。 |
| 熟語・句動詞 | `#/idioms` | ビジネス頻出の句動詞・イディオムをレベル別に。フラッシュカード／4択、SRS連動。カタカナは自動生成。 |
| 文法 | `#/grammar` | つまずき順の6レッスン。解説→例文（音声）→並べ替え→穴埋め→英作文セルフチェック。 |
| フォニックス | `#/phonics` | 44音素の一覧（口の形・例語音声）＋ ミニマルペア聞き分けゲーム（R/L・B/V・S/TH 等）。 |
| 発音チェック | `#/pron-check` | 音声認識で目標文と単語単位で照合し一致率を表示。非対応環境は録音聞き比べにフォールバック。 |
| リンキング | `#/linking` | 英文を解析し、連結・フラップT・脱落・同化・弱形を色分け表示＋参考カタカナ。通常/ゆっくり再生。 |
| リーディング | `#/reading` | サンプル文／貼り付けテキストを文単位表示。文クリックで読み上げ、単語タップで辞書＋単語帳、WPM速読。 |
| シャドーイング | `#/video` | YouTube動画＋字幕で練習。速度0.5〜1.0・A-Bループ・カラオケ字幕・録音聞き比べ。音声なし練習も可。 |
| カタカナ変換 | `#/katakana` | 英文を「実際の聞こえ方」に近いカタカナへ変換（辞書＋ルール）。あくまで発音の補助輪。 |

さらに **レベル診断**（`#/diagnosis`：10問でCEFR目安を判定）、**単語帳**（`#/wordbook`：リーディングで保存した語の一覧・読み上げ・削除）、**ホームのダッシュボード**（連続日数・デイリーゴール・直近7日間グラフ・バッジ・累計）を備えます。

## 動かし方（ローカル）

`index.html` をブラウザで開くだけで起動します。データは `fetch` を使わず `<script>` で読み込むため、`file://` でも教材データまで含めて動作します。

- **そのまま開く（file://）**：ほぼ全機能が動作します。
  ※ Service Worker（オフライン動作）は `file://` では無効です（ブラウザ仕様。エラーにはなりません）。
- **ローカルサーバー推奨（http://）**：PWA・オフライン動作の確認や、音声認識を安定させたい場合はこちら。

```bash
# このフォルダ直下で
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

### ブラウザ別の対応状況
- **音声読み上げ（お手本）**：主要ブラウザで動作（`speechSynthesis`）。
- **音声認識（発音チェックの自動採点）**：Chrome系のみ。非対応時は録音して聞き比べる方式に自動で切り替わります。
- **録音**：`MediaRecorder` 対応ブラウザ。マイク許可が必要です。
- **YouTube**：再生にはネット接続が必要（IFrame Player API を利用）。

## GitHub Pages への公開手順

1. このフォルダの中身をリポジトリのルートに置いて push します（`index.html` がルートに来るようにします）。
2. GitHub リポジトリの **Settings → Pages** を開きます。
3. **Build and deployment** の Source を **Deploy from a branch** にし、Branch を `main`（フォルダは `/root`）に設定して保存します。
4. 数十秒〜数分で `https://<ユーザー名>.github.io/<リポジトリ名>/` に公開されます。

サブディレクトリ公開でも動くよう、リンク・スクリプトはすべて相対パスにしています。

## Netflix など埋め込み不可サービスでの使い方（トランスクリプト並走モード）

Netflix等は規約・技術仕様により動画を直接埋め込めません。次の運用を推奨します。

1. 動画はNetflixアプリ／サイト側でそのまま再生します。
2. EigoMaster は別ウィンドウ（スマホなら別アプリ）で開き、**シャドーイングの「音声なしで練習する」** を使います。
3. セリフ（自分で書き起こした字幕やスクリプト）を貼り付けると、英語／カタカナ／訳の3段表示と読み上げで並走練習できます。

YouTube動画はURLを貼るだけで、アプリ内での速度変更・A-Bループ・字幕同期が使えます。

## 字幕・スクリプトの形式

シャドーイング／リーディングは次を自動判別して読み込みます。

- **SRT**（`00:00:01,000 --> 00:00:04,000`）
- **VTT**（`WEBVTT` ヘッダ付き）
- **プレーンテキスト**（タイムスタンプ無し。改行または文末記号で文に分割）

3段表示の「訳」を出したい場合は、各行を `English | 日本語` のように `|`（またはタブ）で区切ると訳として表示されます。

> 著作権の都合で字幕データは同梱していません。ご自身で用意したテキストをお使いください。

## データについて（拡張方法）

教材は `data/` 配下の JavaScript ファイルで、`window.EigoData` に登録しています（`fetch` のCORS制約を避けるため）。
同じ形式で要素を追加するだけで語数・レッスンを増やせます。

| ファイル | 内容 |
|---|---|
| `data/words.js` | 単語（レベル・IPA・カタカナ・例文）。約90語のビジネス頻出スターターセット。 |
| `data/idioms.js` | 熟語・句動詞（意味・例文・レベル・種別）。36件。 |
| `data/grammar.js` | 文法レッスン（解説・例文・並べ替え・穴埋め・英作文）。 |
| `data/phonics.js` | 44音素とミニマルペア。 |
| `data/linking_rules.js` | リンキングのルールと練習例。 |
| `data/pron_dict.js` | 発音辞書（語→IPA→カタカナ）。カタカナ変換・辞書の基盤。 |
| `data/diagnosis.js` | レベル診断の問題。 |
| `data/reading.js` | リーディングのサンプル文（自作・権利フリー）。 |

> 同梱データは動作確認に十分な量のサンプルです。実運用では各ファイルを増補してください（例：単語を500語以上へ）。

## 設計メモ

- **発音の正直さ**：音声認識は雑音や訛りで揺れます。一致率はあくまで「目安」と明記しています。
- **カタカナは補助輪**：最初の取っかかり用です。慣れたらIPAと音だけで聞き取れるよう、徐々に頼らない設計思想です。
- **リンキング解析**：完全な音声解析ではなく、つづりベースのヒューリスティックによる近似です（UIにも明記）。
- **プライバシー**：通信は YouTube 再生時を除いて発生しません。学習データは端末内のみ。録音は保存・送信しません（その場での再生のみ）。

## 技術構成

- 依存ライブラリなし（Vanilla JS）。ES Modules は使わず、`file://` でも動くよう通常スクリプトで読み込み。
- `window.EM`（共有API：ルーティング・TTS・トースト・画面登録）に各モジュールが `EM.registerView()` で登録。
- 保存は `window.Storage`（スキーマ検証・破損耐性・エクスポート/インポート）。
- PWA：`manifest.json` ＋ `sw.js`（http(s) 時のみ Service Worker 登録）。

```
index.html
css/style.css
js/   storage.js app.js srs.js katakana.js linking.js subtitle.js
      vocab.js wordbook.js shadowing.js pronunciation.js phonics.js grammar.js reading.js diagnosis.js
data/ words.js grammar.js phonics.js linking_rules.js pron_dict.js diagnosis.js reading.js
icons/ manifest.json sw.js
```
